🛂 Authorization

🎯 What is Authorization?

Authorization determines what an authenticated user is allowed to do:

  • Can they access a particular URL?
  • Can they call a specific method?
  • Do they have the required role/permission?

👥 Roles and Authorities

  • Roles – High-level groups of permissions (e.g., ROLE_USER, ROLE_ADMIN)
  • Authorities – Fine-grained permissions (e.g., ORDER_READ, ORDER_WRITE)

Spring Security works with GrantedAuthority objects, which can represent either roles or permissions.

🌐 URL-based Authorization

Configure which URLs require which roles:

@Bean
SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
    http
        .authorizeHttpRequests(auth -> auth
            .requestMatchers("/public/**").permitAll()
            .requestMatchers("/admin/**").hasRole("ADMIN")
            .requestMatchers("/api/orders/**").hasAnyRole("USER", "ADMIN")
            .anyRequest().authenticated()
        )
        .httpBasic(Customizer.withDefaults());

    return http.build();
}

Note: By convention, roles are stored as ROLE_ADMIN internally, but you use hasRole("ADMIN") in configuration.

🔧 Method-level Security

Sometimes you want authorization checks directly on methods in your service layer.

  • @PreAuthorize – Check before method execution
  • @PostAuthorize – Check after method execution
  • @Secured – Simpler role-based checks

Enable method security:

@Configuration
@EnableMethodSecurity
public class MethodSecurityConfig {
}

Then use annotations:

@Service
public class OrderService {

    @PreAuthorize("hasRole('ADMIN')")
    public void cancelOrder(Long orderId) {
        // only admins can cancel orders
    }

    @PreAuthorize("hasAnyRole('USER', 'ADMIN')")
    public Order getOrder(Long orderId) {
        // users and admins can view orders
    }
}

📊 Expression-based Access Control

Spring Security's expression language allows powerful checks:

  • hasRole('ADMIN')
  • hasAuthority('ORDER_READ')
  • hasAnyAuthority('ORDER_READ','ORDER_WRITE')
  • principal.username == #username (ownership checks)

🧪 Example: Ownership-based Authorization

@PreAuthorize("#userId == authentication.principal.id or hasRole('ADMIN')")
public UserProfile getUserProfile(Long userId) {
    // user can view own profile, or admins can view any profile
}

🛡️ Combining Authentication & Authorization

A typical flow:

  1. User logs in and gets authenticated
  2. Their roles/authorities are loaded and stored in the Authentication object
  3. For each request, Spring Security checks URL or method rules to decide access

🎯 Summary

  • Authorization controls what authenticated users can do
  • Spring Security supports URL-based and method-level authorization
  • Use roles for coarse-grained access and authorities/expressions for fine-grained rules
