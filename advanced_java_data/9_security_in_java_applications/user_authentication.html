👤 User Authentication

🎯 Goal of Authentication

Authentication verifies a user's identity. In web applications, this usually means:

  • Collecting credentials (username/password, token, etc.)
  • Verifying them against a user store (database, LDAP, OAuth2 provider)
  • Establishing a logged-in session or issuing a token

🧱 Authentication Flow with Spring Security

  1. User sends credentials (form login, HTTP Basic header, token)
  2. Spring Security filters intercept the request
  3. An AuthenticationManager delegates to one or more AuthenticationProviders
  4. On success, an Authentication object is stored in the SecurityContext
  5. Subsequent requests reuse this identity (session, token, etc.)

📦 Built-in Authentication Mechanisms

  • HTTP Basic – Credentials sent in the Authorization header (Base64-encoded)
  • Form Login – Standard login page with username/password
  • Remember-me – Persistent login using cookies
  • OAuth2 / OpenID Connect – Login with Google, GitHub, etc.
  • JWT-based – Token-based stateless authentication (via custom configuration)

⚙️ Example: In-Memory Users (For Learning)

@Bean
UserDetailsService userDetailsService() {
    UserDetails user = User
        .withUsername("user")
        .password(passwordEncoder().encode("password"))
        .roles("USER")
        .build();

    UserDetails admin = User
        .withUsername("admin")
        .password(passwordEncoder().encode("admin123"))
        .roles("ADMIN")
        .build();

    return new InMemoryUserDetailsManager(user, admin);
}

@Bean
PasswordEncoder passwordEncoder() {
    return new BCryptPasswordEncoder();
}

This is useful for demos, but not for production.

💾 Database-backed Authentication

In real apps, you store users in a database.

  • Create a User entity/table with username, password hash, roles
  • Implement UserDetails and UserDetailsService to load users
  • Use a BCryptPasswordEncoder to store passwords securely

🔐 Password Hashing

Never store plain-text passwords.

  • Use strong hashing algorithms (e.g., BCrypt, Argon2, PBKDF2)
  • Spring Security's BCryptPasswordEncoder is a good default choice

🔑 Form Login Example

@Bean
SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
    http
        .csrf(csrf -> csrf.disable())
        .authorizeHttpRequests(auth -> auth
            .requestMatchers("/login", "/public/**").permitAll()
            .anyRequest().authenticated()
        )
        .formLogin(form -> form
            .loginPage("/login")
            .defaultSuccessUrl("/home", true)
            .permitAll()
        )
        .logout(logout -> logout.permitAll());

    return http.build();
}

Here you can customize the login page and success URL.

🧾 Token-based Authentication (JWT Overview)

For stateless APIs, you'll often use JSON Web Tokens (JWT):

  • User logs in with credentials
  • Server issues a signed token (JWT) containing user identity and roles
  • Client sends token with each request (Authorization header)
  • Server validates token signature and extracts identity without server-side session

Spring Security can be extended with custom filters to support JWT.

🎯 Summary

  • Authentication confirms the user's identity
  • Spring Security provides multiple mechanisms: HTTP Basic, form login, OAuth2, JWT integration
  • Always store hashed passwords and prefer database-backed user stores in real applications
