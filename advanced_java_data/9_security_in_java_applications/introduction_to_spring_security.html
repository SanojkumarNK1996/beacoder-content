🔐 Introduction to Spring Security

🎯 Why Application Security Matters

Modern applications handle sensitive data: user profiles, payments, confidential business information. A single vulnerability can lead to:

  • Data breaches and financial loss
  • Reputation damage
  • Legal and compliance issues (GDPR, PCI-DSS, etc.)

Security must be treated as a first-class concern, not an afterthought.

🧩 Authentication vs Authorization

  • Authentication – Who are you?
    Verifying the identity of a user (login with username/password, tokens, etc.).
  • Authorization – What are you allowed to do?
    Controlling access to resources and actions (roles, permissions).

Spring Security helps with both.

🌱 What is Spring Security?

Spring Security is a powerful and highly customizable authentication and access-control framework for Java/Spring applications.

Key capabilities:

  • Authentication (form login, HTTP Basic, OAuth2, JWT integration)
  • Authorization (URL-based, method-level)
  • Protection against common attacks (CSRF, session fixation)
  • Integration with Spring MVC and Spring Boot

🏗️ How Spring Security Works (High-Level)

  1. Incoming HTTP request reaches the Spring Security filter chain
  2. Filters perform authentication and authorization checks
  3. If authentication is successful, a SecurityContext is created for the user
  4. Authorization checks decide if the request is allowed
  5. If checks pass, the request reaches your controller

🧱 Spring Security Building Blocks

  • SecurityFilterChain – Configures how requests are secured (HTTP endpoints, login, logout)
  • Authentication – Represents the currently authenticated user
  • SecurityContext – Stores Authentication information
  • UserDetailsService – Loads user information from a data source
  • PasswordEncoder – Hashes passwords (e.g., BCrypt)

⚙️ Basic Spring Security Setup (Spring Boot)

Add the Spring Security starter:

<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-security</artifactId>
</dependency>

By default, Spring Boot will:

  • Secure all endpoints
  • Generate a default user with a random password (printed in logs)
  • Require login via a form-based login page

🧠 Security Configuration (Overview)

In modern Spring Security (Spring Boot 3+), you typically define a SecurityFilterChain bean:

@Bean
SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
    http
        .csrf(csrf -> csrf.disable())
        .authorizeHttpRequests(auth -> auth
            .requestMatchers("/public/**").permitAll()
            .anyRequest().authenticated()
        )
        .httpBasic(Customizer.withDefaults());
    return http.build();
}

This is just a taste; later sections will explore authentication and authorization in more detail.

🧪 Common Attack Types Spring Security Helps With

  • Brute-force login attempts – Strong authentication and password policies
  • CSRF (Cross-Site Request Forgery) – Built-in CSRF protection for state-changing operations
  • Session fixation – Secure session handling
  • Clickjacking – Security headers (X-Frame-Options)

🎯 Summary

  • Spring Security is the standard security framework for Spring applications
  • It provides robust support for authentication, authorization, and protection against common attacks
  • Understanding its core concepts (filter chain, SecurityContext, configuration) is essential for secure Java applications
