🏢 Enterprise Patterns

🎯 Goal

Enterprise patterns are higher-level patterns commonly used in business applications, especially in Java/Spring ecosystems.

📦 DAO (Data Access Object)

Encapsulates access to persistent storage (database) behind an interface.

  • Separates data access logic from business logic
  • Makes it easier to change the persistence technology

In Spring, Spring Data JPA repositories are a modern realization of DAO.

📨 DTO (Data Transfer Object)

A simple object used to transfer data between layers or over the network.

  • Decouples internal domain models from external API contracts
  • Prevents over-sharing of data
  • Helps with serialization and versioning

🎭 MVC (Model-View-Controller)

Separates application into three main concerns:

  • Model – Domain data and business logic
  • View – UI representation
  • Controller – Handles requests, coordinates Model and View

In Spring MVC:

  • Controllers are @Controller or @RestController classes
  • Models are Java objects/entities/DTOs
  • Views can be Thymeleaf templates, JSON, etc.

📚 Repository Pattern

Similar to DAO but focuses on the concept of a collection-like interface for accessing domain objects.

In Spring Data JPA:

public interface BookRepository extends JpaRepository<Book, Long> {
    List<Book> findByAuthor(String author);
}

🏛️ Layered Architecture

Common layers in enterprise Java:

  • Controller layer – HTTP endpoints / API
  • Service layer – Business logic
  • Repository/DAO layer – Data access

Each layer depends only on the layer below it, promoting separation of concerns and testability.

🎯 Summary

  • Enterprise patterns like DAO, DTO, MVC, and Repository are building blocks of clean, maintainable architectures
  • Spring and Spring Boot provide first-class support for these patterns
  • Combining GoF design patterns with enterprise patterns helps you design robust Java systems
