🧩 Structural Patterns

🎯 Goal

Structural design patterns are about how classes and objects are composed to form larger structures while keeping them flexible and efficient.

🔌 Adapter Pattern

Converts the interface of a class into another interface the clients expect. It lets classes work together that couldn't otherwise because of incompatible interfaces.

Example

Suppose your code expects a PaymentProcessor interface, but you need to integrate with a third-party library that exposes a different API.

public interface PaymentProcessor {
    void pay(double amount);
}

// Third-party class
public class StripeClient {
    public void makePayment(double amountInDollars) { /* ... */ }
}

// Adapter
public class StripePaymentAdapter implements PaymentProcessor {

    private final StripeClient stripeClient;

    public StripePaymentAdapter(StripeClient stripeClient) {
        this.stripeClient = stripeClient;
    }

    @Override
    public void pay(double amount) {
        stripeClient.makePayment(amount);
    }
}

🎨 Decorator Pattern

Adds responsibilities to objects dynamically without modifying their code. Often used for cross-cutting concerns like logging, caching, security.

public interface MessageService {
    void send(String message);
}

public class EmailService implements MessageService {
    public void send(String message) { /* send email */ }
}

public class LoggingMessageDecorator implements MessageService {

    private final MessageService delegate;

    public LoggingMessageDecorator(MessageService delegate) {
        this.delegate = delegate;
    }

    public void send(String message) {
        System.out.println("Logging: " + message);
        delegate.send(message);
    }
}

🏛️ Facade Pattern

Provides a simple interface to a complex subsystem.

Example: a OrderFacade that internally coordinates payment, inventory, and shipping services, exposing a single placeOrder() method.

🧱 Composite Pattern

Lets you treat individual objects and compositions of objects uniformly.

  • Tree structures: file systems, UI components, menus
  • Operations applied to both leaf nodes and composite nodes

🚪 Proxy Pattern

Provides a surrogate or placeholder for another object to control access to it.

  • Remote proxy – Access remote objects
  • Virtual proxy – Lazy initialization
  • Protection proxy – Access control

In Spring, proxies are heavily used for AOP, transactions, and security.

🎯 Summary

  • Structural patterns focus on class and object composition
  • Adapter and Facade help with integration and simplification
  • Decorator and Proxy are useful for adding cross-cutting behavior without changing core classes
