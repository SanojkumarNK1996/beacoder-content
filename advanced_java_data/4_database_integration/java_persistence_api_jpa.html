📚 Java Persistence API (JPA)

🎯 What is JPA?

JPA (Java Persistence API) is a Java specification that defines how to map Java objects to relational database tables. It's an abstraction layer over ORM implementations like Hibernate, EclipseLink, etc.

  • 📄 JPA: Standard API (interfaces, annotations)
  • ⚙️ Hibernate: Popular implementation of JPA

🌍 Language and Accent Analogy

JPA is like the English language (rules), and Hibernate is like a specific accent (implementation). You speak "JPA", and Hibernate "understands and executes" it.

🏗️ Basic Concepts

  • 🧱 Entity: A Java class mapped to a database table
  • 📌 Field / Property: Mapped to a table column
  • 🔑 Primary Key: Unique identifier for each row
  • 🔗 Relationships: @OneToOne, @OneToMany, @ManyToOne, @ManyToMany
  • 🧠 EntityManager: Interface to manage entity lifecycle (persist, find, remove)
  • 🧾 Persistence Context: First-level cache for managed entities

🧱 Defining a JPA Entity

import jakarta.persistence.*;  // or javax.persistence.* for older versions

@Entity                      // Marks this class as a JPA entity
@Table(name = "users")       // Optional - default is class name
public class User {

    @Id                      // Primary key
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false, length = 100)
    private String name;

    @Column(unique = true, nullable = false)
    private String email;

    private String role;

    // Default constructor (required by JPA)
    public User() {}

    public User(String name, String email, String role) {
        this.name = name;
        this.email = email;
        this.role = role;
    }

    // Getters and setters omitted for brevity
}

📌 Common Annotations

  • @Entity – Marks a class as a JPA entity
  • @Table – Customizes table name and schema
  • @Id – Marks primary key field
  • @GeneratedValue – Auto-generates primary key values
  • @Column – Customizes column properties (length, nullable, unique)

🔗 Mapping Relationships

1️⃣ One-to-Many / Many-to-One

Example: One Author has many Books.

@Entity
public class Author {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String name;

    @OneToMany(mappedBy = "author", cascade = CascadeType.ALL, orphanRemoval = true)
    private List<Book> books = new ArrayList<>();
}

@Entity
public class Book {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String title;

    @ManyToOne
    @JoinColumn(name = "author_id", nullable = false)
    private Author author;
}

2️⃣ One-to-One

Example: One User has one Profile.

@Entity
public class UserProfile {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String address;
    private String phone;

    @OneToOne
    @JoinColumn(name = "user_id")
    private User user;
}

3️⃣ Many-to-Many

Example: A Student can enroll in many Courses, and a Course can have many Students.

@Entity
public class Student {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String name;

    @ManyToMany
    @JoinTable(
        name = "student_course",
        joinColumns = @JoinColumn(name = "student_id"),
        inverseJoinColumns = @JoinColumn(name = "course_id")
    )
    private List<Course> courses = new ArrayList<>();
}

⚙️ EntityManager Basics (Plain JPA)

Without Spring Data, you use EntityManager to perform CRUD operations.

import jakarta.persistence.*;

@Repository
public class UserJpaRepository {

    @PersistenceContext
    private EntityManager entityManager;

    public void save(User user) {
        entityManager.persist(user);
    }

    public User findById(Long id) {
        return entityManager.find(User.class, id);
    }

    public void delete(User user) {
        entityManager.remove(user);
    }
}

In practice, with Spring Boot, you'll usually use Spring Data JPA instead of manually dealing with EntityManager, but understanding it helps you grasp how JPA works underneath.

🧠 Entity States & Lifecycle

  • ➕ New / Transient: Not associated with any persistence context or database row
  • ✅ Managed / Persistent: Attached to persistence context; changes are tracked
  • 🧾 Detached: Was managed but is no longer tracked
  • ❌ Removed: Marked for deletion

JPA automatically synchronizes managed entities with the database at flush time (often at transaction commit).

📄 JPQL (Java Persistence Query Language)

JPQL is a query language that operates on entities and fields, not tables and columns. It's similar to SQL but object-oriented.

// JPQL example
String jpql = "SELECT u FROM User u WHERE u.email = :email";

User user = entityManager.createQuery(jpql, User.class)
    .setParameter("email", "john@example.com")
    .getSingleResult();

🧪 Example: Mapping a Real Entity

@Entity
@Table(name = "orders")
public class Order {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false)
    private String orderNumber;

    @Column(nullable = false)
    private LocalDateTime orderDate;

    @Enumerated(EnumType.STRING)
    private OrderStatus status;

    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "customer_id")
    private Customer customer;

    private BigDecimal totalAmount;

    // getters and setters
}

public enum OrderStatus {
    CREATED, PAID, SHIPPED, DELIVERED, CANCELLED
}

📦 JPA in Spring Boot

Spring Boot makes JPA usage very simple via spring-boot-starter-data-jpa.

<!-- pom.xml -->
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-data-jpa</artifactId>
</dependency>

<dependency>
  <groupId>org.postgresql</groupId>
  <artifactId>postgresql</artifactId>
  <scope>runtime</scope>
</dependency>
  
# application.yml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/mydb
    username: myuser
    password: secret

  jpa:
    hibernate:
      ddl-auto: update   # create, update, validate, create-drop
    show-sql: true
    properties:
      hibernate:
        format_sql: true

With Spring Data JPA, you rarely need to use EntityManager directly; instead, you define repository interfaces (covered in the next subtopic).

🎯 Key Takeaway

JPA provides a standard way to map Java objects to relational tables, manage relationships, and query data using an object-oriented API. It is the foundation on which Spring Data JPA builds, enabling highly productive and type-safe data access in modern enterprise applications.
