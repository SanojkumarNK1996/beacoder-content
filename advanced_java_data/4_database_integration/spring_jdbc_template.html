🧪 Spring JDBC Template

🎯 Goal

Learn how Spring's JdbcTemplate simplifies working with relational databases using plain SQL while reducing boilerplate code compared to raw JDBC.

❌ Problem with Plain JDBC

With traditional JDBC, you must manually handle:

  • 🔌 Getting and closing connections
  • 📜 Preparing statements
  • 🔁 Iterating over ResultSets
  • 🧹 Cleaning up resources in finally blocks
  • ❗ Handling SQLExceptions everywhere
// Plain JDBC example (simplified)
String sql = "SELECT id, name, email FROM users";
List<User> users = new ArrayList<>();

try (Connection connection = dataSource.getConnection();
     PreparedStatement statement = connection.prepareStatement(sql);
     ResultSet rs = statement.executeQuery()) {

    while (rs.next()) {
        User user = new User();
        user.setId(rs.getLong("id"));
        user.setName(rs.getString("name"));
        user.setEmail(rs.getString("email"));
        users.add(user);
    }

} catch (SQLException e) {
    e.printStackTrace();
}

This works, but it's verbose, repetitive, and error-prone.

✅ What is JdbcTemplate?

JdbcTemplate is a core Spring class that:

  • 📦 Handles connection management and resource cleanup
  • 🧹 Wraps exceptions into runtime exceptions (no forced try-catch)
  • 🔁 Simplifies querying and updating data with convenient methods
  • 📄 Allows you to focus on SQL and mapping only

🍳 Cooking Analogy

Plain JDBC is like cooking from scratch: you chop, marinate, cook, clean. JdbcTemplate is like a meal kit: ingredients are prepped; you just follow simple steps.

⚙️ Setting Up JdbcTemplate in Spring Boot

1️⃣ Add Database Dependency

In a Spring Boot project, you typically use spring-boot-starter-jdbc and a database driver.

<!-- pom.xml -->
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-jdbc</artifactId>
</dependency>

<dependency>
  <groupId>org.postgresql</groupId>
  <artifactId>postgresql</artifactId>
  <scope>runtime</scope>
</dependency>

2️⃣ Configure DataSource

Spring Boot auto-configures a DataSource and JdbcTemplate when you provide database properties.

# application.yml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/mydb
    username: myuser
    password: secret
    driver-class-name: org.postgresql.Driver
  
  jdbc:
    template:
      fetch-size: 50
      max-rows: 1000

Now Spring Boot will automatically create a JdbcTemplate bean that you can inject.

🏗️ Basic Usage: Querying Data

🔹 Domain Model

public class User {
    private Long id;
    private String name;
    private String email;

    // getters and setters
}

🔹 RowMapper

A RowMapper<T> converts each row of the ResultSet into an object.

public class UserRowMapper implements RowMapper<User> {
    @Override
    public User mapRow(ResultSet rs, int rowNum) throws SQLException {
        User user = new User();
        user.setId(rs.getLong("id"));
        user.setName(rs.getString("name"));
        user.setEmail(rs.getString("email"));
        return user;
    }
}

🔹 Repository with JdbcTemplate

@Repository
public class UserRepository {

    private final JdbcTemplate jdbcTemplate;

    public UserRepository(JdbcTemplate jdbcTemplate) {
        this.jdbcTemplate = jdbcTemplate;
    }

    public List<User> findAll() {
        String sql = "SELECT id, name, email FROM users";
        return jdbcTemplate.query(sql, new UserRowMapper());
    }

    public User findById(Long id) {
        String sql = "SELECT id, name, email FROM users WHERE id = ?";
        return jdbcTemplate.queryForObject(sql, new UserRowMapper(), id);
    }
}

✏️ Inserting, Updating, Deleting Data

1️⃣ Insert (CREATE)

public int createUser(User user) {
    String sql = "INSERT INTO users (name, email) VALUES (?, ?)";
    return jdbcTemplate.update(sql, user.getName(), user.getEmail());
}

2️⃣ Update (UPDATE)

public int updateUserEmail(Long id, String newEmail) {
    String sql = "UPDATE users SET email = ? WHERE id = ?";
    return jdbcTemplate.update(sql, newEmail, id);
}

3️⃣ Delete (DELETE)

public int deleteUser(Long id) {
    String sql = "DELETE FROM users WHERE id = ?";
    return jdbcTemplate.update(sql, id);
}

🔍 Querying for Single Values

// Count users
public int countUsers() {
    String sql = "SELECT COUNT(*) FROM users";
    return jdbcTemplate.queryForObject(sql, Integer.class);
}

// Get name by email
public String findNameByEmail(String email) {
    String sql = "SELECT name FROM users WHERE email = ?";
    return jdbcTemplate.queryForObject(sql, String.class, email);
}

🧱 NamedParameterJdbcTemplate

NamedParameterJdbcTemplate allows you to use named parameters instead of ? placeholders, making complex queries easier to read.

@Repository
public class ProductRepository {

    private final NamedParameterJdbcTemplate namedJdbc;

    public ProductRepository(NamedParameterJdbcTemplate namedJdbc) {
        this.namedJdbc = namedJdbc;
    }

    public List<Product> findByPriceRange(double minPrice, double maxPrice) {
        String sql = "SELECT id, name, price FROM products " +
                     "WHERE price BETWEEN :minPrice AND :maxPrice";

        Map<String, Object> params = new HashMap<>();
        params.put("minPrice", minPrice);
        params.put("maxPrice", maxPrice);

        return namedJdbc.query(sql, params, new ProductRowMapper());
    }
}

🔐 Transactions with JdbcTemplate

You can combine JdbcTemplate with Spring's @Transactional annotation to ensure that multiple database operations either all succeed or all fail.

@Service
public class BankService {

    private final JdbcTemplate jdbcTemplate;

    public BankService(JdbcTemplate jdbcTemplate) {
        this.jdbcTemplate = jdbcTemplate;
    }

    @Transactional
    public void transferMoney(Long fromAccountId, Long toAccountId, double amount) {
        // Debit sender
        String debitSql = "UPDATE accounts SET balance = balance - ? WHERE id = ?";
        jdbcTemplate.update(debitSql, amount, fromAccountId);

        // Credit receiver
        String creditSql = "UPDATE accounts SET balance = balance + ? WHERE id = ?";
        jdbcTemplate.update(creditSql, amount, toAccountId);

        // If any exception occurs here, both updates are rolled back
    }
}

📊 When to Use JdbcTemplate vs JPA

  • ✅ Use JdbcTemplate when:
    • You need full control over SQL
    • You're working with complex, hand-optimized queries
    • You're doing reporting or batch operations
  • ✅ Use JPA / Spring Data JPA when:
    • You're working with object graphs (entities and relationships)
    • You want to minimize SQL and focus on domain models
    • You need database-agnostic code

🎯 Key Takeaway

JdbcTemplate is ideal when you want the power of SQL with less boilerplate than raw JDBC. It's simple, lightweight, and fits well in applications that don't need full ORM features. In the next sections, you'll see how JPA and Spring Data JPA can further simplify data access by working at the object level.
