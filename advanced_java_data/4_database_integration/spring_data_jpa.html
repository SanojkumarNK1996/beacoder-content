📦 Spring Data JPA

🎯 Goal

Understand how Spring Data JPA builds on top of JPA to provide repository-based, highly productive data access with minimal boilerplate.

❓ Why Spring Data JPA?

With plain JPA you still need to write repository classes and use EntityManager manually. Spring Data JPA removes much of this boilerplate.

  • ✅ No manual repository implementation for basic CRUD
  • ✅ Query methods generated from method names
  • ✅ Integration with Spring Boot, transactions, and validation

🧱 Lego Set Analogy

JPA gives you individual Lego bricks. Spring Data JPA gives you pre-built Lego modules for common patterns like CRUD, pagination, and sorting.

⚙️ Setup

1️⃣ Dependencies

<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-data-jpa</artifactId>
</dependency>

<dependency>
  <groupId>org.postgresql</groupId>
  <artifactId>postgresql</artifactId>
  <scope>runtime</scope>
</dependency>

2️⃣ Configuration

# application.yml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/bookdb
    username: bookuser
    password: secret

  jpa:
    hibernate:
      ddl-auto: update     # create, update, validate, create-drop
    show-sql: true
    properties:
      hibernate:
        format_sql: true

📂 Creating a Repository Interface

With Spring Data JPA, you define interfaces that extend repository base interfaces — no implementation class required for basic CRUD.

import org.springframework.data.jpa.repository.JpaRepository;

public interface BookRepository extends JpaRepository<Book, Long> {

    // Query methods derived from method names
    List<Book> findByAuthor(String author);

    List<Book> findByTitleContainingIgnoreCase(String keyword);

    Optional<Book> findByIsbn(String isbn);
}

Spring Data JPA will automatically provide implementations for these methods at runtime.

🔁 Using the Repository in a Service

@Service
public class BookService {

    private final BookRepository bookRepository;

    public BookService(BookRepository bookRepository) {
        this.bookRepository = bookRepository;
    }

    public List<Book> getAllBooks() {
        return bookRepository.findAll();
    }

    public Book createBook(Book book) {
        return bookRepository.save(book);
    }

    public Optional<Book> getBookById(Long id) {
        return bookRepository.findById(id);
    }

    public void deleteBook(Long id) {
        bookRepository.deleteById(id);
    }
}

🧩 Derived Query Methods

Spring Data JPA can generate queries from method names:

  • findByTitle
  • findByAuthorAndPriceLessThan
  • findByTitleContainingIgnoreCase
public interface BookRepository extends JpaRepository<Book, Long> {

    List<Book> findByAuthor(String author);

    List<Book> findByAuthorAndPriceLessThan(String author, Double price);

    List<Book> findByTitleContainingIgnoreCaseOrderByPriceDesc(String keyword);
}

🔤 Method Naming Rules

  • findBy + property name → equality
  • Keywords: And, Or, LessThan, GreaterThan, Containing, Between, etc.
  • OrderBy + property + Asc/Desc → sorting

🧾 Custom JPQL / Native Queries

1️⃣ JPQL Queries with @Query

public interface BookRepository extends JpaRepository<Book, Long> {

    @Query("SELECT b FROM Book b WHERE b.price > :minPrice")
    List<Book> findExpensiveBooks(@Param("minPrice") double minPrice);
}

2️⃣ Native SQL Queries

public interface BookRepository extends JpaRepository<Book, Long> {

    @Query(value = "SELECT * FROM books WHERE isbn LIKE %:prefix%", 
           nativeQuery = true)
    List<Book> searchByIsbnPrefix(@Param("prefix") String prefix);
}

📄 Pagination & Sorting

public interface BookRepository extends JpaRepository<Book, Long> {

    Page<Book> findByAuthor(String author, Pageable pageable);
}

// Usage in service
public Page<Book> getBooksByAuthor(String author, int page, int size) {
    Pageable pageable = PageRequest.of(page, size, Sort.by("title").ascending());
    return bookRepository.findByAuthor(author, pageable);
}

🔐 Transactions with Spring Data JPA

Spring Data JPA integrates tightly with Spring's transaction management. By default, CrudRepository and JpaRepository methods are transactional where appropriate.

@Service
public class OrderService {

    private final OrderRepository orderRepository;
    private final PaymentService paymentService;

    public OrderService(OrderRepository orderRepository, PaymentService paymentService) {
        this.orderRepository = orderRepository;
        this.paymentService = paymentService;
    }

    @Transactional
    public Order placeOrder(Order order) {
        // Save order
        Order savedOrder = orderRepository.save(order);

        // Process payment
        paymentService.charge(order);

        // If paymentService throws an exception, transaction is rolled back
        return savedOrder;
    }
}

🧪 Testing Spring Data JPA Repositories

@DataJpaTest
class BookRepositoryTest {

    @Autowired
    private TestEntityManager entityManager;

    @Autowired
    private BookRepository bookRepository;

    @Test
    void testFindByAuthor() {
        Book book = new Book();
        book.setTitle("Spring in Action");
        book.setAuthor("Craig Walls");
        entityManager.persist(book);
        entityManager.flush();

        List<Book> result = bookRepository.findByAuthor("Craig Walls");

        assertEquals(1, result.size());
    }
}

🎯 Key Takeaway

Spring Data JPA lets you build powerful data access layers using simple interfaces and method names. It removes most of the boilerplate of JPA while still giving you the flexibility to use custom queries, pagination, sorting, and transactions. For most enterprise applications, it's the default choice for relational data access in Spring.
