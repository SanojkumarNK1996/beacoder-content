🚀 Introduction to Spring Boot

🎯 What is Spring Boot?

Spring Boot is a powerful extension of the Spring framework that eliminates boilerplate configuration and enables rapid application development with opinionated defaults and auto-configuration.

🚀 Spring Boot

Convention over Configuration approach to Spring development that provides:

  • ⚡ Auto-Configuration: Automatically configures Spring based on dependencies
  • 📦 Embedded Servers: Run apps without external server setup
  • 🎯 Starter Dependencies: Pre-configured dependency bundles
  • 🔧 Production-Ready: Built-in monitoring, health checks

🤔 Why Was Spring Boot Created?

😣 Traditional Spring Problems:

  • 📝 Configuration Hell: Hundreds of XML lines
  • 🏗️ Complex Setup: Application server configuration
  • 📚 Dependency Management: Version conflicts nightmare
  • 🐌 Slow Development: Long setup time for new projects
  • ⚙️ Boilerplate Code: Repetitive configuration patterns

✅ Spring Boot Solutions:

  • 🎯 Auto-Configuration: No XML configuration needed
  • 📦 Embedded Servers: Tomcat/Jetty built-in
  • 🎁 Starter POMs: Curated dependencies
  • ⚡ Rapid Development: Create apps in minutes
  • 🎨 Opinionated Defaults: Best practices out-of-the-box

🌟 Core Spring Boot Features

1️⃣ Auto-Configuration

Spring Boot automatically configures your application based on the dependencies you add.

🗄️ Database Auto-Configuration

// Add dependency in pom.xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-jpa</artifactId>
</dependency>
<dependency>
    <groupId>mysql</groupId>
    <artifactId>mysql-connector-java</artifactId>
</dependency>

// Spring Boot automatically configures:
// ✅ DataSource bean
// ✅ EntityManagerFactory
// ✅ JpaTransactionManager
// ✅ JPA repositories

🌐 Web Auto-Configuration

// Add web starter dependency
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
</dependency>

// Spring Boot automatically configures:
// ✅ Embedded Tomcat server
// ✅ Spring MVC
// ✅ JSON message converters
// ✅ Static resource handling
// ✅ Error page mapping

2️⃣ Embedded Servers

No need for external application servers - Spring Boot includes embedded Tomcat, Jetty, or Undertow.

@SpringBootApplication
public class MyApplication {
    public static void main(String[] args) {
        SpringApplication.run(MyApplication.class, args);
        // 🚀 Tomcat server starts automatically on port 8080
    }
}

// Run as simple Java application:
java -jar myapp.jar

// Server starts, no external Tomcat needed!

3️⃣ Starter Dependencies

Pre-configured dependency bundles that bring in all related libraries with compatible versions.

🌐 Web Applications

spring-boot-starter-web

Includes: Spring MVC, Tomcat, Jackson JSON

💾 Database Access

spring-boot-starter-data-jpa

Includes: Spring Data JPA, Hibernate, JDBC

🔒 Security

spring-boot-starter-security

Includes: Spring Security, authentication, authorization

🧪 Testing

spring-boot-starter-test

Includes: JUnit, Mockito, Spring Test, AssertJ

1. web.xml (100+ lines)

<web-app version="3.1" xmlns="http://java.sun.com/xml/ns/javaee">
    <servlet>
        <servlet-name>dispatcher</servlet-name>
        <servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
        <init-param>
            <param-name>contextConfigLocation</param-name>
            <param-value>/WEB-INF/spring/appServlet/servlet-context.xml</param-value>
        </init-param>
        <load-on-startup>1</load-on-startup>
    </servlet>
    <!-- ... more XML configuration ... -->

2. applicationContext.xml (200+ lines)

<?xml version="1.0" encoding="UTF-8"?>

    <!-- Database configuration -->

        <!-- ... complex JPA setup ... -->
    </bean>
    <!-- ... hundreds more lines ... -->

🚀 Spring Boot Setup Example

Complete Application (10 lines!)

@SpringBootApplication
public class MyApplication {
    public static void main(String[] args) {
        SpringApplication.run(MyApplication.class, args);
    }
}

@RestController
public class UserController {
    @GetMapping("/users")
    public List<User> getUsers() {
        return Arrays.asList(new User("John"), new User("Jane"));
    }
}

Configuration (application.properties)

# Database configuration (3 lines vs 50+ XML lines)
spring.datasource.url=jdbc:mysql://localhost:3306/mydb
spring.datasource.username=user
spring.datasource.password=password

# JPA configuration (2 lines vs 30+ XML lines)  
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true

🎯 Key Spring Boot Annotations

@SpringBootApplication

The main annotation - combines three annotations:

  • @Configuration - Marks as configuration class
  • @EnableAutoConfiguration - Enables auto-configuration
  • @ComponentScan - Scans for Spring components
@SpringBootApplication
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}

@RestController

Combines @Controller and @ResponseBody

@RestController
public class ApiController {
    @GetMapping("/api/hello")
    public String hello() {
        return "Hello, Spring Boot!";
    }
}

@ConfigurationProperties

Type-safe configuration properties

@ConfigurationProperties(prefix = "app")
public class AppProperties {
    private String name;
    private String version;
    // getters and setters
}

🎯 Key Takeaway

Spring Boot revolutionized Java development by eliminating configuration complexity while maintaining Spring's power and flexibility. It's the default choice for modern Java enterprise applications.

💡 Philosophy: "Just run" - Spring Boot handles the plumbing so you focus on business value creation.
