📨 Request and Response Handling

🎯 Goal

Understand how Spring handles HTTP requests and responses, including JSON mapping, validation, and response entities.

📥 Mapping Request Data

1️⃣ Path Variables

Used to capture values from the URL path.

// GET /api/orders/100
@GetMapping("/orders/{id}")
public Order getOrder(@PathVariable Long id) {
    return orderService.getOrderById(id);
}

2️⃣ Query Parameters

// GET /api/orders?status=PAID&page=0&size=10
@GetMapping("/orders")
public List<Order> getOrders(@RequestParam String status,
                              @RequestParam(defaultValue = "0") int page,
                              @RequestParam(defaultValue = "10") int size) {
    return orderService.getOrdersByStatus(status, page, size);
}

3️⃣ Request Body (JSON → Java)

@RequestBody maps the JSON body to a Java object using Jackson.

// JSON request body
{
  "title": "Clean Code",
  "author": "Robert C. Martin",
  "price": 45.99
}

// Controller method
@PostMapping("/books")
public ResponseEntity<Book> createBook(@RequestBody Book book) {
    Book saved = bookService.save(book);
    return ResponseEntity.status(HttpStatus.CREATED).body(saved);
}

📤 Building Responses

1️⃣ Simple Return Types

Returning an object from a @RestController method automatically produces JSON with status 200 OK.

@GetMapping("/profile")
public UserProfile getProfile() {
    return profileService.getCurrentUserProfile();
}

2️⃣ Using ResponseEntity

ResponseEntity<T> gives full control over:

  • HTTP status code
  • Headers
  • Body
@GetMapping("/books/{id}")
public ResponseEntity<Book> getBook(@PathVariable Long id) {
    return bookService.findById(id)
        .map(ResponseEntity::ok)
        .orElse(ResponseEntity.notFound().build());
}

@PostMapping("/books")
public ResponseEntity<Book> createBook(@RequestBody Book book) {
    Book created = bookService.save(book);
    URI location = URI.create("/api/books/" + created.getId());
    return ResponseEntity.created(location).body(created);
}

🧪 JSON Serialization and Deserialization

Spring Boot uses Jackson by default to convert between Java objects and JSON.

1️⃣ Controlling JSON with Annotations

public class User {

    private Long id;

    @JsonProperty("full_name")
    private String name;

    @JsonIgnore
    private String password;

    @JsonFormat(pattern = "yyyy-MM-dd")
    private LocalDate birthDate;

    // getters and setters
}
  • @JsonProperty – rename field in JSON
  • @JsonIgnore – exclude from JSON
  • @JsonFormat – format dates

✅ Validation of Requests

1️⃣ Add Validation Annotations

Use Jakarta Bean Validation annotations to enforce constraints.

public class CreateUserRequest {

    @NotBlank
    private String name;

    @Email
    @NotBlank
    private String email;

    @Size(min = 8, message = "Password must be at least 8 characters")
    private String password;

    // getters and setters
}

2️⃣ Use @Valid in Controller

@PostMapping("/users")
public ResponseEntity<User> createUser(@Valid @RequestBody CreateUserRequest request) {
    User user = userService.createUser(request);
    return ResponseEntity.status(HttpStatus.CREATED).body(user);
}

On validation failure, Spring automatically returns a 400 Bad Request response. You can customize the error format using a global exception handler (covered in the next subtopic).

🧾 Custom Response Models (DTOs)

It's a good practice to separate:

  • 📥 Request DTOs (input)
  • 📤 Response DTOs (output)
  • 💾 Entity models (database)
public class BookResponse {
    private Long id;

📄 Setting Headers

@GetMapping("/reports/{id}")
public ResponseEntity<byte[]> downloadReport(@PathVariable Long id) {
    byte[] pdf = reportService.generatePdfReport(id);

    return ResponseEntity.ok()
            .header(HttpHeaders.CONTENT_DISPOSITION, "attachment; filename=report-" + id + ".pdf")
            .contentType(MediaType.APPLICATION_PDF)
            .body(pdf);
}

🎯 Summary

  • Spring maps request data using @PathVariable, @RequestParam, and @RequestBody
  • Responses can be customized using ResponseEntity with status, headers, and body
  • Jackson handles JSON mapping; Bean Validation ensures input correctness
