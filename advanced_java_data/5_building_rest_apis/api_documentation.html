📘 API Documentation (Swagger / OpenAPI)

🎯 Why Document APIs?

Good API documentation helps:

  • 👨‍💻 Frontend and mobile developers understand your endpoints
  • 🤖 Automated tools generate clients and tests
  • 📚 New team members quickly learn the system

For REST APIs, OpenAPI (formerly Swagger) is the de facto standard.

📄 What is OpenAPI?

OpenAPI is a specification for describing REST APIs in a machine-readable format (JSON or YAML). It describes:

  • Endpoints and paths
  • HTTP methods
  • Request and response models
  • Parameters, headers, auth, etc.

🧰 Springdoc OpenAPI (for Spring Boot)

springdoc-openapi is a popular library that automatically generates OpenAPI docs and a Swagger UI for Spring Boot apps.

1️⃣ Add Dependency

<dependency>
  <groupId>org.springdoc</groupId>
  <artifactId>springdoc-openapi-starter-webmvc-ui</artifactId>
  <version>2.5.0</version>
</dependency>

2️⃣ Run the Application

By default, Springdoc will expose:

  • OpenAPI JSON: /v3/api-docs
  • Swagger UI: /swagger-ui.html or /swagger-ui/index.html

🧾 Example: Simple Book API with OpenAPI Annotations

@RestController
@RequestMapping("/api/books")
@Tag(name = "Books", description = "Operations related to books")
public class BookController {

    private final BookService bookService;

    public BookController(BookService bookService) {
        this.bookService = bookService;
    }

    @Operation(summary = "Get all books")
    @ApiResponses({
        @ApiResponse(responseCode = "200", description = "List of books returned")
    })
    @GetMapping
    public List<BookResponse> getAllBooks() {
        return bookService.getAllBooks();
    }

    @Operation(summary = "Get book by ID")
    @ApiResponses({
        @ApiResponse(responseCode = "200", description = "Book found"),
        @ApiResponse(responseCode = "404", description = "Book not found")
    })
    @GetMapping("/{id}")
    public ResponseEntity<BookResponse> getBookById(
            @Parameter(description = "ID of the book to retrieve")
            @PathVariable Long id) {

        return bookService.getBookById(id)
                .map(ResponseEntity::ok)
                .orElse(ResponseEntity.notFound().build());
    }

    @Operation(summary = "Create a new book")
    @ApiResponses({
        @ApiResponse(responseCode = "201", description = "Book created"),
        @ApiResponse(responseCode = "400", description = "Invalid input")
    })
    @PostMapping
    public ResponseEntity<BookResponse> createBook(
            @Valid @RequestBody CreateBookRequest request) {

        BookResponse created = bookService.createBook(request);
        return ResponseEntity.status(HttpStatus.CREATED).body(created);
    }
}

📊 Swagger UI Demo

With Springdoc configured, navigating to /swagger-ui.html gives you an interactive UI where you can:

  • See all endpoints grouped by controller
  • Inspect request/response models
  • Try endpoints directly in the browser

🧱 Customizing OpenAPI Metadata

@Configuration
public class OpenApiConfig {

    @Bean
    public OpenAPI customOpenAPI() {
        return new OpenAPI()
                .info(new Info()
                        .title("Book Management API")
                        .version("1.0.0")
                        .description("API documentation for the Book Management system")
                        .contact(new Contact()
                                .name("Beacoder Team")
                                .email("support@beacoder.com"))
                );
    }
}

🧩 Benefits of OpenAPI / Swagger

  • 📖 Human-readable, interactive documentation (Swagger UI)
  • ⚙️ Machine-readable spec for generating clients (Java, JS, etc.)
  • ✅ Easier testing and onboarding

🎯 Summary

  • Use springdoc-openapi to generate OpenAPI specs for Spring Boot
  • Annotate controllers and models to enrich documentation
  • Leverage Swagger UI for interactive API exploration

With proper API documentation in place, your REST APIs become much easier to use, test, and maintain—both for your team and external consumers.
