🌱 Spring Boot Testing

🎯 Goal

Learn how to test Spring Boot applications using Spring's testing support: @SpringBootTest, slice tests, and MockMvc.

🧰 spring-boot-starter-test

Spring Boot provides a convenient starter that bundles common testing libraries:

  • JUnit 5
  • Mockito
  • Spring Test & Spring Boot Test
  • AssertJ, Hamcrest, JSONassert, etc.
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-test</artifactId>
  <scope>test</scope>
</dependency>

🏗️ Testing Levels in Spring Boot

  • Unit tests – Test a single class (e.g., service) with mocks
  • Slice tests – Test a layer (web, data) with minimal context
  • Full integration tests – Start entire application context

🧪 @SpringBootTest – Full Integration Test

@SpringBootTest
class BookManagementApplicationTests {

    @Test
    void contextLoads() {
        // Verifies the Spring context starts successfully
    }
}

@SpringBootTest loads the full application context – ideal for integration tests where you want to test components together.

Example: Service Integration Test

@SpringBootTest
class BookServiceIntegrationTest {

    @Autowired
    private BookService bookService;

    @Test
    void createAndFindBook_shouldWorkThroughRealBeans() {
        CreateBookRequest request = new CreateBookRequest();
        request.setTitle("Spring Boot in Action");
        request.setAuthor("Craig Walls");

        Book created = bookService.createBook(request);

        Book found = bookService.getBookById(created.getId());

        assertEquals("Spring Boot in Action", found.getTitle());
    }
}

🌐 Testing REST Controllers with @WebMvcTest

@WebMvcTest loads only the web layer (controllers, filters, etc.) and uses MockMvc to simulate HTTP calls.

@WebMvcTest(BookController.class)
class BookControllerTest {

    @Autowired
    private MockMvc mockMvc;

    @MockBean
    private BookService bookService;  // mocked dependency

    @Test
    void getAllBooks_shouldReturnJsonList() throws Exception {
        BookResponse book = new BookResponse();
        book.setId(1L);
        book.setTitle("Spring in Action");

        when(bookService.getAllBooks()).thenReturn(List.of(book));

        mockMvc.perform(get("/api/books"))
                .andExpect(status().isOk())
                .andExpect(content().contentType(MediaType.APPLICATION_JSON))
                .andExpect(jsonPath("$[0].title").value("Spring in Action"));
    }
}

💾 Testing Repositories with @DataJpaTest

@DataJpaTest configures an in-memory database (like H2) and scans for JPA repositories.

@DataJpaTest
class BookRepositoryTest {

    @Autowired
    private TestEntityManager entityManager;

    @Autowired
    private BookRepository bookRepository;

    @Test
    void findByTitle_shouldReturnMatchingBooks() {
        Book book = new Book();
        book.setTitle("Spring Boot 3");
        book.setAuthor("John Doe");

        entityManager.persist(book);
        entityManager.flush();

        List<Book> result = bookRepository.findByTitle("Spring Boot 3");

        assertEquals(1, result.size());
        assertEquals("John Doe", result.get(0).getAuthor());
    }
}

🔌 Using Test Slices

Spring Boot provides specialized test annotations (slices):

  • @WebMvcTest – Web layer
  • @DataJpaTest – JPA repositories
  • @JsonTest – JSON serialization/deserialization
  • @WebFluxTest – WebFlux controllers

🧪 Example: JSON Test

@JsonTest
class BookJsonTest {

    @Autowired
    private JacksonTester<Book> json;

    @Test
    void serializeBook() throws Exception {
        Book book = new Book();
        book.setTitle("Spring Boot");
        book.setAuthor("Craig Walls");

        assertThat(json.write(book)).isEqualToJson("{" +
                "  'title': 'Spring Boot'," +
                "  'author': 'Craig Walls'" +
                "}");
    }
}

🚀 Test Configuration Tips

  • Use application-test.yml with an in-memory database for tests
  • Keep integration tests isolated and reset DB state between tests
  • Use profiles (e.g., test) to separate test configuration from production
# application-test.yml
spring:
  datasource:
    url: jdbc:h2:mem:testdb
    driverClassName: org.h2.Driver
    username: sa
    password: 
  jpa:
    hibernate:
      ddl-auto: create-drop
    show-sql: true

🎯 Summary

  • @SpringBootTest – for full integration tests
  • @WebMvcTest + MockMvc – for controller (web layer) tests
  • @DataJpaTest – for repository (data layer) tests
  • Use profiles and test slices to keep tests fast and focused

With JUnit, Mockito, and Spring Boot's testing support, you can build a comprehensive test suite that covers your Java applications end-to-end.
