🧪 Testing Fundamentals

🎯 Why Testing Matters

Testing ensures your code is correct, reliable, and maintainable. In enterprise Java applications, tests help you:

  • 🛡️ Catch bugs early (before production)
  • 🔁 Refactor code with confidence
  • 📦 Prevent regressions when adding new features
  • 🤝 Improve collaboration (others can trust your code)

🏗️ Building Safety Analogy

Testing is like inspecting a building:

  • Structural checks = Unit tests for core logic
  • Fire drills = Integration tests for end-to-end flows
  • Daily maintenance = Regression tests

Without tests, you're "building skyscrapers without safety checks".

🏛️ Types of Tests

1️⃣ Unit Tests

  • Test small pieces of code (methods/classes) in isolation
  • Use mocks for external dependencies (database, network, etc.)
  • Fast, reliable, run frequently (every build / commit)

2️⃣ Integration Tests

  • Test how multiple components work together
  • May involve real database, messaging, or external systems (or test doubles)
  • Slower than unit tests but more realistic

3️⃣ End-to-End (E2E) / System Tests

  • Simulate real user flows across the full system
  • Often involve UI, backend, and database
  • Slowest, but closest to real-world behavior

4️⃣ Other Types

  • ⚡ Performance tests
  • 🔒 Security tests
  • 📈 Load and stress tests

📐 The Test Pyramid

          UI / E2E Tests (few)
       Integration Tests (some)
    Unit Tests (many, fast, cheap)

Most tests should be unit tests, with fewer integration and E2E tests.

✅ Characteristics of Good Tests

  • Fast – Run quickly so you execute them often
  • Independent – Tests don't depend on each other
  • Repeatable – Same result every time (no randomness / time dependency)
  • Readable – Clearly express the intention
  • Maintainable – Easy to update when code changes

🧪 Basic Structure of a Test (AAA)

Most tests follow the Arrange–Act–Assert pattern:

  • Arrange – Set up data and dependencies
  • Act – Call the method under test
  • Assert – Verify the result
// Pseudo-code
@Test
void calculateTotalPrice_shouldIncludeTax() {
    // Arrange
    ShoppingCart cart = new ShoppingCart();
    cart.addItem("Book", 100.0);

    // Act
    double total = cart.calculateTotalPrice();

    // Assert
    assertEquals(118.0, total); // 18% tax
}

🧰 Testing Tools in Java

  • 🧪 JUnit 5 – Standard testing framework for Java
  • 🕵️ Mockito – Mocking framework for replacing dependencies
  • 🌱 Spring Test / Spring Boot Test – Testing Spring applications

⚙️ Adding JUnit to a Maven Project

<dependency>
  <groupId>org.junit.jupiter</groupId>
  <artifactId>junit-jupiter</artifactId>
  <version>5.10.0</version>
  <scope>test</scope>
</dependency>

<build>
  <plugins>
    <plugin>
      <groupId>org.apache.maven.plugins</groupId>
      <artifactId>maven-surefire-plugin</artifactId>
      <version>3.1.2</version>
      <configuration>
        <useModulePath>false</useModulePath>
      </configuration>
    </plugin>
  </plugins>
</build>

With Spring Boot, JUnit 5 is already included via spring-boot-starter-test.

🧪 Example: Simple Unit Test with JUnit 5

// Production code
public class Calculator {
    public int add(int a, int b) {
        return a + b;
    }
}

// Test code
import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

class CalculatorTest {

    @Test
    void add_shouldReturnSumOfTwoNumbers() {
        // Arrange
        Calculator calculator = new Calculator();

        // Act
        int result = calculator.add(2, 3);

        // Assert
        assertEquals(5, result);
    }
}

📦 Test Naming Conventions

  • Class-level: CalculatorTest tests Calculator
  • Method-level: methodName_condition_expectedResult

Examples:

  • add_twoPositiveNumbers_returnsCorrectSum
  • withdraw_insufficientBalance_throwsException

🔁 Regression Testing

When you find a bug:

  1. Write a test that fails because of the bug
  2. Fix the bug
  3. Ensure the test now passes

This prevents the same bug from silently returning later.

🧪 Test Pyramid in Practice for Spring Apps

  • Many JUnit + Mockito tests for services and utilities
  • Some Spring Boot integration tests for REST APIs and repositories
  • Few end-to-end tests using tools like Selenium, Cypress, etc.

🎯 Key Takeaway

Testing is a core skill for any Java developer. Understanding test types, the test pyramid, and the basics of JUnit will prepare you for more advanced tools like Mockito and Spring Boot Testing in the next lessons.
