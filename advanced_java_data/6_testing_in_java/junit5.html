🧪 JUnit 5

🎯 What is JUnit 5?

JUnit 5 is the modern version of the JUnit testing framework for Java. It consists of:

  • JUnit Platform – Launches tests
  • JUnit Jupiter – New programming model and annotations
  • JUnit Vintage – Backward compatibility with JUnit 3/4

⚙️ Dependency (Maven)

<dependency>
  <groupId>org.junit.jupiter</groupId>
  <artifactId>junit-jupiter</artifactId>
  <version>5.10.0</version>
  <scope>test</scope>
</dependency>

In Spring Boot projects, spring-boot-starter-test already includes JUnit 5 by default.

🔑 Core Annotations

  • @Test – Marks a test method
  • @BeforeEach – Runs before each test
  • @AfterEach – Runs after each test
  • @BeforeAll – Runs once before all tests (static method)
  • @AfterAll – Runs once after all tests (static method)
  • @Disabled – Temporarily disable a test
  • @DisplayName – Custom name for test in reports

🧪 Example: Lifecycle Methods

import org.junit.jupiter.api.*;

class UserServiceTest {

    @BeforeAll
    static void initAll() {
        System.out.println("Runs once before all tests");
    }

    @BeforeEach
    void init() {
        System.out.println("Runs before each test");
    }

    @Test
    @DisplayName("Create user should succeed with valid data")
    void createUser_validData_success() {
        System.out.println("Running test 1");
        // your test code here
    }

    @Test
    void anotherTest() {
        System.out.println("Running test 2");
    }

    @AfterEach
    void tearDown() {
        System.out.println("Runs after each test");
    }

    @AfterAll
    static void tearDownAll() {
        System.out.println("Runs once after all tests");
    }
}

📏 Assertions

Assertions verify that the actual result matches the expected result.

import static org.junit.jupiter.api.Assertions.*;

@Test
void assertionsExamples() {
    int result = 2 + 3;

    assertEquals(5, result);              // equality
    assertNotEquals(4, result);           // inequality

    assertTrue(result > 0);               // boolean condition
    assertFalse(result < 0);

    String name = "JUnit";
    assertNotNull(name);                  // not null

    int[] numbers = {1, 2, 3};
    assertArrayEquals(new int[]{1, 2, 3}, numbers);
}

🧨 Asserting Exceptions

@Test
void shouldThrowExceptionWhenInvalid() {
    IllegalArgumentException exception = assertThrows(
            IllegalArgumentException.class,
            () -> {
                // Code under test
                throw new IllegalArgumentException("Invalid input");
            }
    );

    assertEquals("Invalid input", exception.getMessage());
}

🧪 Parameterized Tests

Run the same test logic with different inputs.

import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.ValueSource;

class StringUtilsTest {

    @ParameterizedTest
    @ValueSource(strings = {"racecar", "madam", "level"})
    void isPalindrome_shouldReturnTrueForPalindromes(String candidate) {
        assertTrue(StringUtils.isPalindrome(candidate));
    }
}

🏷️ Tagging and Filtering Tests

Use @Tag to group tests (e.g., slow, integration).

@Tag("slow")
@Test
void longRunningTest() {
    // ...
}

You can configure Maven/Gradle to run or skip certain tags (e.g., skip slow tests in CI).

🧪 Example: Testing a Service Class

// Service under test
public class DiscountService {

    public double applyDiscount(double amount, int percentage) {
        if (percentage < 0 || percentage > 100) {
            throw new IllegalArgumentException("Invalid discount percentage");
        }
        return amount - (amount * percentage / 100.0);
    }
}

// Unit tests
class DiscountServiceTest {

    private DiscountService discountService = new DiscountService();

    @Test
    void applyDiscount_validPercentage_returnsDiscountedAmount() {
        double result = discountService.applyDiscount(1000, 10);
        assertEquals(900.0, result);
    }

    @Test
    void applyDiscount_invalidPercentage_throwsException() {
        assertThrows(IllegalArgumentException.class,
                () -> discountService.applyDiscount(1000, -5));
    }
}

🎯 Summary

  • JUnit 5 is the standard testing framework in modern Java
  • Provides rich annotations, assertions, and parameterized tests
  • Forms the foundation for more advanced tools like Mockito and Spring Boot Testing
