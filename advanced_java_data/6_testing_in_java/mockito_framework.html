🕵️ Mockito Framework

🎯 Why Use Mocks?

In unit tests, you want to test one class in isolation, without involving:

  • Real databases
  • External services (HTTP, messaging)
  • Slow or flaky dependencies

Mockito lets you create mock objects that simulate the behavior of dependencies.

🎭 Theater Analogy

Real dependencies are like full actors; mocks are like stand-ins that behave just enough for rehearsal. They help you test your scene (class) without the full cast.

🧪 Basic Mockito Example

1️⃣ Service to Test

public class OrderService {

    private final PaymentGateway paymentGateway;

    public OrderService(PaymentGateway paymentGateway) {
        this.paymentGateway = paymentGateway;
    }

    public boolean placeOrder(double amount) {
        PaymentResponse response = paymentGateway.charge(amount);
        return response.getStatus() == PaymentStatus.SUCCESS;
    }
}

public interface PaymentGateway {
    PaymentResponse charge(double amount);
}

public class PaymentResponse {
    private PaymentStatus status;
    // constructor, getters, setters
}

public enum PaymentStatus { SUCCESS, FAILURE }

2️⃣ Unit Test with Mockito

import org.junit.jupiter.api.Test;
import org.mockito.Mockito;

import static org.junit.jupiter.api.Assertions.*;
import static org.mockito.Mockito.*;

class OrderServiceTest {

    @Test
    void placeOrder_successfulPayment_returnsTrue() {
        // Arrange: create mock
        PaymentGateway paymentGateway = mock(PaymentGateway.class);
        OrderService orderService = new OrderService(paymentGateway);

        // Define mock behavior
        when(paymentGateway.charge(1000.0))
                .thenReturn(new PaymentResponse(PaymentStatus.SUCCESS));

        // Act
        boolean result = orderService.placeOrder(1000.0);

        // Assert
        assertTrue(result);
        verify(paymentGateway, times(1)).charge(1000.0);
    }
}

🔑 Key Mockito Concepts

  • mock(Class) – Create a mock object
  • when(...).thenReturn(...) – Stub method behavior
  • verify(mock) – Verify interactions (method calls)
  • ArgumentMatchers – Match any / specific arguments

🧪 Using Mockito with JUnit 5 Annotations

import org.junit.jupiter.api.extension.ExtendWith;
import org.mockito.InjectMocks;
import org.mockito.Mock;
import org.mockito.junit.jupiter.MockitoExtension;

@ExtendWith(MockitoExtension.class)
class NotificationServiceTest {

    @Mock
    private EmailClient emailClient;   // mock dependency

    @InjectMocks
    private NotificationService notificationService; // class under test

    @Test
    void sendWelcomeEmail_shouldUseEmailClient() {
        // Arrange
        String email = "user@example.com";

        // Act
        notificationService.sendWelcomeEmail(email);

        // Assert / Verify
        verify(emailClient).sendEmail(eq(email), anyString(), anyString());
    }
}

🎯 Verifying Interactions

verify(emailClient).sendEmail(anyString(), anyString(), anyString());
verify(emailClient, times(1)).sendEmail(...);
verify(emailClient, never()).sendEmail(...);
verifyNoMoreInteractions(emailClient);

🎭 Stubbing with Different Answers

// Return different values
when(repository.count()).thenReturn(0L, 1L, 2L);

// Throw exception
when(paymentGateway.charge(anyDouble()))
        .thenThrow(new RuntimeException("Payment system down"));

// Custom Answer
when(clock.now()).thenAnswer(invocation -> LocalDateTime.of(2026, 3, 19, 10, 0));

🧩 Argument Matchers

when(userRepository.findByEmail(eq("test@example.com")))
        .thenReturn(user);

when(orderRepository.findByStatusAndDate(eq("PAID"), any(LocalDate.class)))
        .thenReturn(List.of(order));

Important: When using matchers like any(), use them for all parameters of that method call.

🧪 Spies vs Mocks

  • Mock – No real behavior unless stubbed
  • Spy – Wraps a real object, but you can stub or verify certain methods
List<String> list = new ArrayList<>();
List<String> spyList = spy(list);

spyList.add("one");
spyList.add("two");

verify(spyList).add("one");
verify(spyList).add("two");

assertEquals(2, spyList.size());

🎯 Summary

  • Mockito helps you isolate the class under test by mocking dependencies
  • Use when/thenReturn for stubbing and verify() for interaction testing
  • Integrates seamlessly with JUnit 5 and Spring Boot tests
