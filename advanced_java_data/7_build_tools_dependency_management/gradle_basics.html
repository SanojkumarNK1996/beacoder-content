⚡ Gradle Basics

🎯 What is Gradle?

Gradle is a modern, flexible build tool for JVM and other ecosystems. It combines the best ideas from Maven and Ant, with:

  • 🧠 A powerful DSL (Groovy or Kotlin)
  • ⚡ Incremental builds and caching for speed
  • 📦 Great support for multi-module projects

📄 build.gradle (Groovy DSL)

plugins {
    id 'java'
}

group = 'com.example'
version = '1.0.0'

repositories {
    mavenCentral()
}

dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-web:3.2.0'
    testImplementation 'org.springframework.boot:spring-boot-starter-test:3.2.0'
}

java {
    sourceCompatibility = JavaVersion.VERSION_17
    targetCompatibility = JavaVersion.VERSION_17
}

🔧 Gradle Wrapper

The Gradle Wrapper (gradlew) allows you to run a specific Gradle version without installing it globally.

./gradlew build
./gradlew test
./gradlew clean

In Windows, use gradlew.bat.

🏗️ Common Tasks

  • clean – Delete build directory
  • compileJava – Compile main source
  • test – Run tests
  • bootRun – Run Spring Boot app (with Spring plugin)
  • build – Full build (compilation + tests + packaging)
./gradlew clean build
./gradlew test
./gradlew bootRun

📦 Dependencies

Gradle uses configurations similar to Maven scopes:

  • implementation – Main dependencies
  • api – Exposed to consumers (for libraries)
  • testImplementation – Test dependencies
  • runtimeOnly – Only at runtime
dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-web'
    implementation 'org.projectlombok:lombok:1.18.30'
    testImplementation 'org.springframework.boot:spring-boot-starter-test'
}

🌱 Spring Boot with Gradle

plugins {
    id 'java'
    id 'org.springframework.boot' version '3.2.0'
    id 'io.spring.dependency-management' version '1.1.4'
}

group = 'com.example'
version = '0.0.1-SNAPSHOT'

repositories {
    mavenCentral()
}

dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-web'
    testImplementation 'org.springframework.boot:spring-boot-starter-test'
}

java {
    sourceCompatibility = JavaVersion.VERSION_17
}

With the Spring Boot plugin:

  • bootRun – Runs the app
  • bootJar – Builds an executable JAR

📁 Multi-Module Projects

Gradle makes multi-project builds straightforward:

my-multi-project/
├── settings.gradle
├── build.gradle (root)
├── app/
│   └── build.gradle
└── core/
    └── build.gradle
  
// settings.gradle
rootProject.name = 'my-multi-project'
include 'app', 'core'

⚖️ Maven vs Gradle (High-Level)

Aspect Maven Gradle
Config format XML (pom.xml) Groovy/Kotlin DSL (build.gradle)
Philosophy Convention over configuration Highly flexible, scriptable
Speed Good Very good (incremental build, caching)
Readability Very explicit XML Concise DSL

🧪 Useful Gradle Commands

# Show tasks
./gradlew tasks

# Show dependencies
./gradlew dependencies --configuration implementation

# Run a single test
./gradlew test --tests com.example.CalculatorTest

🎯 Summary

  • Gradle is a modern, fast, and flexible build tool
  • Uses a Groovy/Kotlin DSL instead of XML
  • Works very well with Spring Boot via dedicated plugins
