🧩 Managing Dependencies

🎯 Why Dependency Management Matters

Modern Java apps rely on dozens or hundreds of third-party libraries. Dependency management tools (Maven, Gradle) help you:

  • 📦 Declare libraries you need
  • 🔁 Resolve transitive dependencies (dependencies of dependencies)
  • ⚠️ Detect and resolve version conflicts
  • 📈 Keep versions consistent across modules

🧬 Transitive Dependencies

If your app depends on library A, and A depends on B, then B is a transitive dependency.

Your app
  └── Spring Boot Starter Web
        ├── Spring Web
        ├── Jackson
        ├── Validation API
        └── ... many others

Build tools automatically download these transitives for you.

📦 Declaring Dependencies (Maven)

<dependencies>
  <dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
  </dependency>

  <dependency>
    <groupId>com.fasterxml.jackson.core</groupId>
    <artifactId>jackson-databind</artifactId>
    <version>2.16.0</version>
  </dependency>
</dependencies>

📦 Declaring Dependencies (Gradle)

dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-web'
    implementation 'com.fasterxml.jackson.core:jackson-databind:2.16.0'
}

⚠️ Version Conflicts

Sometimes different dependencies bring different versions of the same library.

  • Library A needs log4j:2.17.0
  • Library B needs log4j:2.15.0

The build tool must choose one version. This is where dependency resolution comes in.

Inspecting the Dependency Tree

# Maven
a) mvn dependency:tree

# Gradle
b) ./gradlew dependencies --configuration implementation

🧱 Dependency Management in Maven

Use the <dependencyManagement> block in a parent POM to define versions in one place.

<dependencyManagement>
  <dependencies>
    <dependency>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-dependencies</artifactId>
      <version>3.2.0</version>
      <type>pom</type>
      <scope>import</scope>
    </dependency>
  </dependencies>
</dependencyManagement>

Then in child modules, you can omit the version:

<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-web</artifactId>
</dependency>

🧱 Dependency Management in Gradle (with Spring)

Gradle uses the io.spring.dependency-management plugin or platform/BOM support.

plugins {
    id 'java'
    id 'org.springframework.boot' version '3.2.0'
    id 'io.spring.dependency-management' version '1.1.4'
}

dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-web'
    testImplementation 'org.springframework.boot:spring-boot-starter-test'
}

🛡️ Excluding Problematic Transitive Dependencies

Maven

<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-web</artifactId>
  <exclusions>
    <exclusion>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-starter-logging</artifactId>
    </exclusion>
  </exclusions>
</dependency>

Gradle

dependencies {
    implementation('org.springframework.boot:spring-boot-starter-web') {
        exclude group: 'org.springframework.boot', module: 'spring-boot-starter-logging'
    }
}

🧪 Keeping Dependencies Up to Date

  • Use plugins/tools to check for updates (e.g., Versions Maven Plugin, Gradle Versions Plugin)
  • Regularly review and update dependency versions
  • Watch for security advisories (log4j-style vulnerabilities)
# Maven: show available updates
mvn versions:display-dependency-updates

# Gradle: with versions plugin
./gradlew dependencyUpdates

🔐 Security Best Practices

  • Remove unused dependencies
  • Prefer official repositories (Maven Central)
  • Keep dependencies patched and updated
  • Use tools like OWASP Dependency-Check to find vulnerable libraries

🎯 Summary

  • Dependency management is critical for keeping your project stable, secure, and maintainable
  • Maven and Gradle both offer powerful tools to manage versions, resolve conflicts, and exclude unwanted libraries
  • Use dependency trees, BOMs, and plugins to control and monitor your dependency graph

With solid understanding of build tools and dependency management, you're ready to work on large-scale, multi-module enterprise Java projects with confidence.
