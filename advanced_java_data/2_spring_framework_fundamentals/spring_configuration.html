⚙️ Spring Configuration

🎯 What is Spring Configuration?

Spring Configuration is about telling Spring how to create, wire, and manage your beans. It's the blueprint that guides the Spring IoC container in building your application.

🏗️ Construction Blueprint Analogy

Just like a building needs architectural drawings to guide construction workers:

  • 📋 Blueprint: Spring Configuration
  • 👷 Construction Workers: Spring IoC Container
  • 🏢 Final Building: Your Running Application

📄 1. XML Configuration (Legacy)

🔧 Basic XML Configuration

applicationContext.xml:

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="http://www.springframework.org/schema/beans 
       http://www.springframework.org/schema/beans/spring-beans.xsd">

    <!-- Bean definitions -->
    <bean id="userRepository" class="com.example.UserRepositoryImpl"/>
    
    <bean id="emailService" class="com.example.EmailServiceImpl"/>
    
    <!-- Dependency injection via constructor -->
    <bean id="userService" class="com.example.UserService">
        <constructor-arg ref="userRepository"/>
        <constructor-arg ref="emailService"/>
    </bean>
    
    <!-- Property injection -->
    <bean id="databaseConfig" class="com.example.DatabaseConfig">
        <property name="url" value="jdbc:mysql://localhost:3306/mydb"/>
        <property name="username" value="user"/>
        <property name="password" value="password"/>
    </bean>

</beans>

Loading XML Configuration:

// Load XML configuration
ApplicationContext context = new ClassPathXmlApplicationContext("applicationContext.xml");

// Get beans
UserService userService = context.getBean("userService", UserService.class);
DatabaseConfig dbConfig = context.getBean("databaseConfig", DatabaseConfig.class);

✅ XML Configuration Pros:

  • 🔍 Explicit: Everything clearly defined
  • 🔧 External: Configuration separate from code
  • 📊 Centralized: All beans in one place
  • 🔄 No Recompilation: Change config without rebuilding

❌ XML Configuration Cons:

  • 📝 Verbose: Lots of boilerplate code
  • ⚠️ Error-Prone: No compile-time checking
  • 🔄 Maintenance: Hard to refactor
  • 📚 Learning Curve: XML schemas to learn

🎯 2. Annotation-Based Configuration (Modern)

🔧 Core Annotations

@Component

Generic Spring-managed component

@Component
public class UserRepository {
    // Spring will create and manage this bean
}

@Service

Business logic layer component

@Service
public class UserService {
    // Semantic annotation for service layer
}

@Repository

Data access layer component

@Repository
public class UserRepository {
    // Includes exception translation
}

@Controller

Web layer component

@Controller
public class UserController {
    // Web MVC controller
}

🎯 Complete Annotation Example

// Repository layer
@Repository
public class UserRepository {
    public User save(User user) {
        // Database operations
        return user;
    }
    
    public User findById(Long id) {
        // Database query
        return new User();
    }
}

// Service layer
@Service
public class UserService {
    private final UserRepository userRepository;
    private final EmailService emailService;
    
    // Constructor injection (no @Autowired needed in Spring 4.3+)
    public UserService(UserRepository userRepository, EmailService emailService) {
        this.userRepository = userRepository;
        this.emailService = emailService;
    }
    
    public User createUser(String email) {
        User user = new User(email);
        User savedUser = userRepository.save(user);
        emailService.sendWelcomeEmail(email);
        return savedUser;
    }
}

// Configuration to enable component scanning
@Configuration
@ComponentScan(basePackages = "com.example")
public class AppConfig {
    // Spring will scan and register @Component, @Service, @Repository, etc.
}

// Alternative: Enable scanning in main class
@SpringBootApplication  // Includes @ComponentScan
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}

☕ 3. Java Configuration (Type-Safe)

🔧 Java Config Class

@Configuration
public class AppConfig {
    
    // Bean definition method
    @Bean
    public UserRepository userRepository() {
        return new UserRepositoryImpl();
    }
    
    @Bean
    public EmailService emailService() {
        return new EmailServiceImpl();
    }
    
    // Method injection - Spring calls methods to get dependencies
    @Bean
    public UserService userService(UserRepository userRepository, EmailService emailService) {
        return new UserService(userRepository, emailService);
    }
    
    // Configuration properties
    @Bean
    public DatabaseConfig databaseConfig() {
        DatabaseConfig config = new DatabaseConfig();
        config.setUrl("jdbc:mysql://localhost:3306/mydb");
        config.setUsername("user");
        config.setPassword("password");
        return config;
    }
    
    // Conditional bean creation
    @Bean
    @Profile("production")  // Only create in production profile
    public PaymentService paymentService() {
        return new RealPaymentService();
    }
    
    @Bean
    @Profile("test")  // Only create in test profile  
    public PaymentService mockPaymentService() {
        return new MockPaymentService();
    }
}

🎯 Loading Java Configuration

// Load Java configuration
ApplicationContext context = new AnnotationConfigApplicationContext(AppConfig.class);

// Get beans
UserService userService = context.getBean(UserService.class);
DatabaseConfig dbConfig = context.getBean(DatabaseConfig.class);

// Or with Spring Boot (auto-configuration)
@SpringBootApplication
public class Application {
    public static void main(String[] args) {
        ApplicationContext context = SpringApplication.run(Application.class, args);
        UserService userService = context.getBean(UserService.class);
    }
}

🔧 Advanced Configuration Features

1️⃣ Property Injection

// application.properties
app.name=MyApplication
app.version=1.0.0
database.url=jdbc:mysql://localhost:3306/mydb
email.enabled=true

// Configuration class
@Configuration
@PropertySource("classpath:application.properties")
public class AppConfig {
    
    @Value("${app.name}")
    private String appName;
    
    @Value("${app.version}")
    private String appVersion;
    
    @Bean
    public AppInfo appInfo() {
        return new AppInfo(appName, appVersion);
    }
}

// In any Spring-managed bean
@Service
public class UserService {
    @Value("${database.url}")
    private String databaseUrl;
    
    @Value("${email.enabled:false}")  // Default value if property not found
    private boolean emailEnabled;
}

2️⃣ Profiles for Different Environments

// Development configuration
@Configuration
@Profile("development")
public class DevConfig {
    @Bean
    public DataSource dataSource() {
        return new H2DataSource(); // In-memory database for dev
    }
}

// Production configuration
@Configuration
@Profile("production")
public class ProdConfig {
    @Bean
    public DataSource dataSource() {
        return new MySQLDataSource(); // Real database for prod
    }
}

// Activate profiles
// 1. In application.properties:
spring.profiles.active=development

// 2. As JVM argument:
java -Dspring.profiles.active=production -jar myapp.jar

// 3. Environment variable:
export SPRING_PROFILES_ACTIVE=production

3️⃣ Conditional Bean Creation

@Configuration
public class AppConfig {
    
    // Create only if class is on classpath
    @Bean
    @ConditionalOnClass(RedisTemplate.class)
    public CacheManager redisCacheManager() {
        return new RedisCacheManager();
    }
    
    // Create only if property has specific value
    @Bean
    @ConditionalOnProperty(name = "cache.provider", havingValue = "ehcache")
    public CacheManager ehcacheCacheManager() {
        return new EhcacheCacheManager();
    }
    
    // Create only if another bean is missing
    @Bean
    @ConditionalOnMissingBean(CacheManager.class)
    public CacheManager defaultCacheManager() {
        return new NoOpCacheManager(); // Default fallback
    }
}

🎯 Modern Best Practices

1️⃣ Use Spring Boot Auto-Configuration

@SpringBootApplication  // Combines @Configuration + @ComponentScan + @EnableAutoConfiguration
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}

// Spring Boot automatically configures:
// - DataSource (if database dependency found)
// - JPA (if JPA dependency found)
// - Web MVC (if web dependency found)
// - Security (if security dependency found)

2️⃣ Externalize Configuration

# application.yml (preferred over .properties)
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/mydb
    username: ${DB_USER:user}
    password: ${DB_PASSWORD:password}
  
  jpa:
    hibernate:
      ddl-auto: validate
    show-sql: false

app:
  name: My Application
  version: @project.version@  # Maven property injection
  features:
    - email-notifications
    - user-registration
    - payment-processing

3️⃣ Use Configuration Properties Classes

// Type-safe configuration properties
@ConfigurationProperties(prefix = "app")
@Data  // Lombok for getters/setters
public class AppProperties {
    private String name;
    private String version;
    private List<String> features;
    private Database database = new Database();
    
    @Data
    public static class Database {
        private String url;
        private String username;
        private String password;
    }
}

// Enable configuration properties
@Configuration
@EnableConfigurationProperties(AppProperties.class)
public class AppConfig {
    // Configuration properties available as beans
}

// Use in any component
@Service
public class UserService {
    private final AppProperties appProperties;
    
    public UserService(AppProperties appProperties) {
        this.appProperties = appProperties;
    }
}

⚠️ Common Configuration Mistakes

❌ Mixing Configuration Styles

// ❌ Don't mix - confusing and hard to maintain
@Configuration
@ImportResource("classpath:old-beans.xml")  // XML import
public class MixedConfig {
    @Bean
    public UserService userService() {  // Java config
        return new UserService();
    }
}

// ✅ Choose one approach and stick to it
@Configuration
public class JavaConfig {
    // All beans defined in Java
}

❌ Hardcoding Values

// ❌ Bad - hardcoded values
@Bean
public DataSource dataSource() {
    HikariConfig config = new HikariConfig();
    config.setJdbcUrl("jdbc:mysql://localhost:3306/mydb");  // Hardcoded!
    config.setUsername("user");                              // Hardcoded!
    config.setPassword("password");                          // Hardcoded!
    return new HikariDataSource(config);
}

// ✅ Good - externalized configuration
@Bean
public DataSource dataSource(@Value("${database.url}") String url,
                           @Value("${database.username}") String username,
                           @Value("${database.password}") String password) {
    HikariConfig config = new HikariConfig();
    config.setJdbcUrl(url);
    config.setUsername(username);
    config.setPassword(password);
    return new HikariDataSource(config);
}

🎯 Key Takeaway

Modern Spring development favors annotation-based configuration with Spring Boot auto-configuration. Use Java configuration for complex scenarios and externalize properties for environment-specific settings.

💡 Modern Stack: Spring Boot + Annotations + YAML properties + Configuration Properties classes = Clean, maintainable, type-safe configuration
