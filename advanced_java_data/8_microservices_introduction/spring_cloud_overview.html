☁️ Spring Cloud Overview

🎯 What is Spring Cloud?

Spring Cloud is a set of tools built on top of Spring Boot to help you build distributed systems and microservices.

It provides ready-made solutions for common challenges:

  • 🔍 Service discovery
  • ⚙️ Centralized configuration
  • 📡 Intelligent routing
  • 🔁 Load balancing
  • 🛡️ Resilience (circuit breakers, retries)

🧩 Key Spring Cloud Components

  • Spring Cloud Config – Centralized configuration server
  • Spring Cloud Netflix / Discovery – Service discovery (Eureka)
  • Spring Cloud Gateway – API Gateway / edge service
  • Spring Cloud OpenFeign – Declarative REST clients
  • Spring Cloud Circuit Breaker – Integrations with Resilience4j, etc.

📂 Centralized Configuration with Spring Cloud Config

Instead of hardcoding configuration in each service, use a Config Server.

  1. Create a Config Server (Spring Boot app)
  2. Point it to a Git repository with properties/YAML files
  3. Each microservice fetches its configuration from the server
# application.yml for config server
spring:
  cloud:
    config:
      server:
        git:
          uri: https://github.com/your-org/config-repo

🔍 Service Discovery with Eureka

Eureka Server acts as a registry where services register themselves.

  1. Run Eureka Server (Spring Boot + Eureka Server dependency)
  2. Each microservice adds Eureka Client dependency
  3. Services register with Eureka and discover each other by name
# application.yml snippet for a service
spring:
  application:
    name: order-service

# This service registers as "order-service" in Eureka

🚪 API Gateway with Spring Cloud Gateway

Spring Cloud Gateway is a modern, reactive API Gateway that handles:

  • Routing (path-based, host-based, etc.)
  • Filters (authentication, logging, rate limiting)
  • Load balancing across services
# application.yml for a simple route
spring:
  cloud:
    gateway:
      routes:
        - id: order-service
          uri: lb://order-service
          predicates:
            - Path=/orders/**

📡 Declarative HTTP Clients with OpenFeign

OpenFeign lets you define HTTP clients as Java interfaces.

@FeignClient(name = "inventory-service")
public interface InventoryClient {

    @GetMapping("/inventory/{productId}")
    InventoryResponse getInventory(@PathVariable String productId);
}

Spring Cloud integrates Feign with service discovery and load balancing automatically.

🛡️ Resilience with Spring Cloud Circuit Breaker

Use circuit breakers and retries to protect your system from cascading failures.

  • Integrates with Resilience4j
  • Configure fallbacks for failing remote calls

🧪 Typical Microservices Stack with Spring

  • Spring Boot for each microservice
  • Spring Cloud Config for configuration
  • Eureka (or another registry) for discovery
  • Spring Cloud Gateway as API Gateway
  • OpenFeign for inter-service calls
  • Resilience4j for resilience patterns

🎯 Summary

  • Spring Cloud provides building blocks for microservices: config, discovery, gateway, clients, and resilience
  • It builds on top of Spring Boot, so you can reuse what you already know
  • Combining these tools lets you build production-ready distributed systems in Java
