📦 Application Packaging

🎯 Goal

Before deploying a Java application, you must package it into a distributable format. In the Java world, that usually means JAR or WAR files.

🧱 JAR vs WAR

  • JAR (Java ARchive)
    • Used for libraries or standalone applications
    • Can be made executable with a Main-Class entry
    • Spring Boot apps commonly use executable JARs
  • WAR (Web Application Archive)
    • Used for web applications deployed to external app servers (Tomcat, JBoss, etc.)
    • Contains WEB-INF, web.xml (legacy), servlets, JSPs, etc.
    • More common in older Java EE deployments

🏃 Executable JARs (Spring Boot)

Spring Boot promotes executable JARs with an embedded server (Tomcat/Jetty/Undertow).

  • Simplifies deployment: java -jar app.jar
  • No need to install and manage a separate app server

⚙️ Maven Packaging

In Maven, packaging type is defined in pom.xml:

<project>
  <modelVersion>4.0.0</modelVersion>
  <groupId>com.example</groupId>
  <artifactId>my-app</artifactId>
  <version>1.0.0</version>
  <packaging>jar</packaging>
</project>

To build:

mvn clean package

Maven creates a JAR (or WAR) under the target/ directory.

🌱 Spring Boot Packaging with Maven

Spring Boot config typically includes the Spring Boot Maven plugin:

<build>
  <plugins>
    <plugin>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-maven-plugin</artifactId>
    </plugin>
  </plugins>
</build>

Then build and run:

mvn clean package
java -jar target/my-app-1.0.0.jar

⚙️ Gradle Packaging

With Gradle and Spring Boot plugin:

plugins {
    id 'org.springframework.boot' version '3.2.0'
    id 'io.spring.dependency-management' version '1.1.4'
    id 'java'
}

// Build executable JAR
./gradlew clean bootJar

📁 Application Layout

Typical packaged Spring Boot JAR layout:

  • Your compiled classes
  • Dependency JARs (nested)
  • Bootstrap code for starting embedded server

🎯 Summary

  • JAR and WAR are standard Java packaging formats
  • Modern Spring apps usually use executable JARs with embedded servers
  • Maven/Gradle plus Spring Boot plugins handle packaging details for you
