🐳 Docker Fundamentals

🎯 What is Docker?

Docker is a platform for building, shipping, and running applications in containers.

  • Containers package your app plus its dependencies
  • They run consistently across environments (dev, test, prod)
  • Lighter than full virtual machines

🏗️ Image vs Container

  • Image – Read-only template describing the filesystem and configuration
  • Container – A running instance of an image

📄 Dockerfile

A Dockerfile is a text file with instructions to build an image.

# Simple Java runtime image
FROM eclipse-temurin:17-jre

WORKDIR /app
COPY target/app.jar app.jar

ENTRYPOINT ["java", "-jar", "app.jar"]

🧪 Basic Docker Commands

# Build an image with tag my-app:1.0
docker build -t my-app:1.0 .

# List images
docker images

# Run a container from an image
docker run --name my-app-container -p 8080:8080 my-app:1.0

# List running containers
docker ps

# Stop and remove container
docker stop my-app-container
docker rm my-app-container

📦 Layers and Caching

  • Docker images are built in layers, each corresponding to a Dockerfile instruction
  • Layers are cached – small changes can reuse previous layers, speeding up builds
  • Order of instructions affects cache efficiency

🌍 Ports and Networking

  • -p hostPort:containerPort maps ports between host and container
  • Multiple containers can be connected with Docker networks

📂 Volumes

Volumes allow data to persist outside the container filesystem.

# Mount host directory ./logs into container
docker run -v $(pwd)/logs:/app/logs my-app:1.0

🎯 Summary

  • Docker packages apps into portable containers
  • Images are built from Dockerfiles, containers are running instances
  • Understanding images, containers, ports, and volumes is the foundation for containerizing Java apps
