☁️ Cloud Deployment

🎯 Goal

Understand common options for deploying Java/Spring applications to the cloud (AWS, Azure, Google Cloud).

🧱 Deployment Models

  • IaaS (Infrastructure as a Service)
    • You manage VMs, OS, runtime, app
    • Example: EC2 (AWS), Compute Engine (GCP), Virtual Machines (Azure)
  • PaaS (Platform as a Service)
    • Platform manages runtime and scaling
    • Example: Elastic Beanstalk, App Engine, Azure App Service
  • CaaS / Kubernetes
    • You deploy and manage containers
    • Example: EKS (AWS), AKS (Azure), GKE (GCP)

🚀 Deploying a Spring Boot JAR to a VM

  1. Create a VM instance (EC2, Compute Engine, Azure VM)
  2. Install Java runtime (e.g., Temurin 17)
  3. Copy the JAR (SCP, SFTP, CI/CD pipeline)
  4. Run: java -jar app.jar, usually managed by a process manager (systemd, supervisor)

🐳 Deploying Containers

Using Docker images, you can deploy to container platforms:

  • AWS – ECS, EKS, Fargate
  • Azure – AKS, Container Instances
  • GCP – GKE, Cloud Run

Basic steps:

  1. Build Docker image
  2. Push to container registry (ECR, GCR, ACR, Docker Hub)
  3. Create service/deployment definitions (ECS task, Kubernetes Deployment)
  4. Expose via load balancer or ingress

📦 PaaS-style Deployment

  • Upload JAR or connect to Git repo
  • Platform builds and runs app
  • Auto-scaling, logging, monitoring often built-in

📊 Observability in the Cloud

  • Use platform logging and metrics (CloudWatch, Stackdriver, Azure Monitor)
  • Expose health and metrics endpoints (Spring Boot Actuator)
  • Set up alerts for errors, high latency, or resource exhaustion

💰 Cost and Scaling

  • Auto-scaling policies based on CPU, memory, or custom metrics
  • Right-size instances/containers for cost efficiency
  • Use multiple environments (dev, test, prod) with separate resources

🎯 Summary

  • You can deploy Spring apps as JARs on VMs, as containers, or via PaaS platforms
  • Containers + orchestrators (like Kubernetes) are now the common standard for scalable deployments
  • Cloud deployment is not just about running the app—it includes scaling, observability, security, and cost management
