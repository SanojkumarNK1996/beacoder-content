📦 Containerizing Spring Boot Apps

🎯 Goal

Turn a Spring Boot application into a Docker container that can run the same way in any environment.

🧱 Prerequisites

  • Spring Boot app packaged as an executable JAR (e.g., app.jar)
  • Docker installed locally

📄 Basic Dockerfile for Spring Boot

FROM eclipse-temurin:17-jre

WORKDIR /app

COPY target/app.jar app.jar

EXPOSE 8080

ENTRYPOINT ["java", "-jar", "app.jar"]

🏗️ Building and Running

# Build JAR
mvn clean package

# Build Docker image
docker build -t my-spring-app:1.0 .

# Run container
docker run -p 8080:8080 my-spring-app:1.0

🚀 Multi-stage Docker Builds

Multi-stage builds let you build and run your app in one Dockerfile while keeping the final image small.

# Stage 1: Build
FROM eclipse-temurin:17-jdk AS builder
WORKDIR /build
COPY . .
RUN ./mvnw clean package -DskipTests

# Stage 2: Run
FROM eclipse-temurin:17-jre
WORKDIR /app
COPY --from=builder /build/target/app.jar app.jar

EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]

⚙️ Externalizing Configuration

Avoid baking environment-specific settings into the image:

  • Use environment variables for DB URLs, credentials, etc.
  • Mount external config files if needed
# Set Spring profile and DB URL
docker run -e "SPRING_PROFILES_ACTIVE=prod" \
           -e "SPRING_DATASOURCE_URL=jdbc:postgresql://db:5432/app" \
           -p 8080:8080 my-spring-app:1.0

📊 Health Checks

Expose health endpoints (e.g., Spring Boot Actuator /actuator/health) so orchestrators (Kubernetes, ECS) can monitor container health.

🎯 Summary

  • Package Spring Boot as an executable JAR, then create a Docker image around it
  • Use multi-stage builds for smaller, cleaner images
  • Externalize configuration so the same image can be used in multiple environments
