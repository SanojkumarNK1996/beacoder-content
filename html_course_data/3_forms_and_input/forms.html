HTML Forms

🎯 Why This Exists (Indian Real-Life Example)

  • IRCTC booking, bank login, job application portals, and college admission forms all rely on HTML forms to collect user data.
  • Without forms, users cannot search, register, log in, or submit any information.

📖 What Is This

  • HTML forms provide a structured way to collect input from users.
  • They use controls like text fields, checkboxes, radios, dropdowns, and buttons.

🧾 Syntax / Basic Structure ⭐

<form action="/submit" method="post">
  <label for="name">Name:</label>
  <input type="text" id="name" name="name" />

  <label for="age">Age:</label>
  <input type="number" id="age" name="age" />

  <button type="submit">Submit</button>
</form>

🔍 How It Works (Step-by-Step Execution) ⭐

  1. <form> wraps all input fields and defines where (action) and how (method) data is sent.
  2. Each <input>, <select>, or <textarea> collects a piece of data using its name attribute.
  3. When the user clicks the submit button, the browser sends the data to the server.

When to Use

  • Registration/login pages, search bars, feedback forms, checkouts.

When NOT to Use

  • Don’t use a full form when a simple link or button is enough (e.g., navigation).

Minimum You MUST Remember

  • <form action="..." method="get|post"> ... </form>.
  • Each control must have a name for its value to be submitted.

Common Mistakes

  • Missing name attribute on inputs (data will not be sent).
  • Using GET for sensitive data like passwords.
