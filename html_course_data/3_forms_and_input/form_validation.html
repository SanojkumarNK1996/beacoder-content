Form Validation

🎯 Why This Exists (Indian Real-Life Example)

  • Bank and UPI apps do not allow submitting a form with invalid phone numbers or empty required fields.
  • Validation prevents bad or incomplete data from being sent to the server.

📖 What Is This

  • Form validation ensures the data entered by the user meets specific rules.
  • HTML provides built-in validation using attributes and input types.

🧾 Syntax / Basic Usage ⭐

<form>
  <input type="email" name="email" required />
  <input type="password" name="password" minlength="6" />
  <button type="submit">Login</button>
</form>

🔍 How It Works (Step-by-Step Execution) ⭐

  1. Attributes like required, minlength, maxlength, and pattern define rules.
  2. When the user clicks submit, the browser checks all rules.
  3. If any rule fails, the form is not submitted and the browser shows an error.

When to Use

  • Always for critical forms like login, signup, and payments.

When NOT to Use

  • Never rely only on client-side validation; always validate on the server too.

Minimum You MUST Remember

  • Use required for mandatory fields.
  • Use appropriate input types (email, number, url, etc.).
  • Use pattern for custom formats (like Indian phone numbers).

Common Mistakes

  • Trusting only client-side validation.
  • Not giving clear error messages to the user.
