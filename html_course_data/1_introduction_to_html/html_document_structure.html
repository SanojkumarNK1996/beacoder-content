HTML Document Structure

🎯 Why This Exists (Indian Real-Life Example)

  • IRCTC booking pages, bank login pages, and UPI success screens all rely on a proper HTML document structure so the browser can render them correctly.
  • If the structure is broken, the page may not load or behave as expected.

📖 What Is This

  • The HTML document structure is the basic skeleton that every web page must follow.
  • It tells the browser where the metadata is and where the visible content starts.

🧾 Syntax / Basic Structure ⭐

<!DOCTYPE html>
<html>
  <head>
    <meta charset="UTF-8" />
    <title>My Page</title>
  </head>
  <body>
    <h1>Hello World</h1>
    <p>This is my first HTML page.</p>
  </body>
</html>

🔍 How It Works (Step-by-Step Execution) ⭐

  1. <!DOCTYPE html> tells the browser to use HTML5.
  2. <html> wraps the entire document.
  3. <head> contains metadata (title, charset, links, scripts).
  4. <body> contains everything that appears on the screen.

When to Use

  • Always, for every HTML page you create.
  • Whenever you build any web UI—static or dynamic.

When NOT to Use

  • This structure is specific to HTML pages; not used for plain text files, PDFs, etc.

Minimum You MUST Remember

  • Always include <!DOCTYPE html> at the top.
  • Use <html>, <head>, and <body> in every page.
  • Visible content must be inside <body>.

Common Mistakes

  • Putting content outside <body>.
  • Forgetting the doctype (can lead to quirks mode in browsers).
  • Missing closing tags for <html> or <body>.

Interviewer’s Perspective

  • What are the mandatory tags in a valid HTML5 document?
  • Where do you place CSS and JavaScript includes?
  • What happens if you skip the doctype?
