Accessibility Basics

🎯 Why This Exists (Indian Real-Life Example)

  • Government sites (like UIDAI for Aadhaar) must be usable by people with visual or motor impairments.
  • Accessible websites support screen readers, keyboard navigation, and clear content for all users.

📖 What Is This

  • Web accessibility means designing sites usable by everyone, including people with disabilities.
  • It involves proper semantics, alt text, ARIA roles, focus management, and more.

🧾 Key Concepts ⭐

  • Alt text for images (<img alt="..." />).
  • Keyboard-focusable elements (links, buttons, form controls).
  • ARIA roles and labels when semantics alone are not enough.

🔍 How It Works (Step-by-Step Execution) ⭐

  1. Screen readers read out content based on semantics and ARIA attributes.
  2. Keyboard users navigate using Tab/Shift+Tab, so focus order matters.

When to Use

  • Always—accessibility should be part of every web project.

When NOT to Use

  • There is no "don’t use"—but avoid misusing ARIA when HTML semantics are enough.

Minimum You MUST Remember

  • Always provide alt text for informative images.
  • Ensure all interactive elements are reachable via keyboard.
  • Use semantic HTML first, ARIA only when required.

Common Mistakes

  • Using click handlers on non-focusable elements like plain <div> without handling keyboard events.
  • Missing labels for form fields.

Interviewer’s Perspective

  • What is web accessibility and why is it important?
  • How do you make images and forms accessible?

Practice Questions

  1. Take an existing page and add alt text, labels, and proper focus order.
