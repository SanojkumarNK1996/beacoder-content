Semantic HTML Tags

🎯 Why This Exists (Indian Real-Life Example)

  • News sites, blogs, and government portals in India use semantic sections like header, nav, article, and footer.
  • Semantic HTML helps search engines and assistive technologies understand your page.

📖 What Is This

  • Semantic tags describe the meaning of content (e.g., <header>, <nav>, <main>, <section>, <article>, <aside>, <footer>).
  • They replace generic <div> elements with meaningful structure.

🧾 Syntax / Basic Example ⭐

<header>Site Header</header>
<nav>Main Navigation</nav>
<main>
  <article>
    <h1>News Title</h1>
    <p>News content...</p>
  </article>
</main>
<footer>Copyright ©</footer>

🔍 How It Works (Step-by-Step Execution) ⭐

  1. The browser treats semantic tags like <div> visually but gives them meaning.
  2. Search engines and screen readers use this meaning to understand page structure.

When to Use

  • Always prefer semantic tags when they match the content purpose.

When NOT to Use

  • Don’t overuse semantic tags where a simple <div> is enough.

Minimum You MUST Remember

  • Use <header>, <nav>, <main>, <section>, <article>, <aside>, <footer> appropriately.

Common Mistakes

  • Using multiple <main> elements.
  • Using semantic tags only for styling without caring about structure.

  • Give examples of semantic tags and their use cases.
