HTML Tables

🎯 Why This Exists (Indian Real-Life Example)

  • IRCTC, bank statements, and college mark sheets all show data in rows and columns.
  • Such structured data is best represented using tables on the web.

📖 What Is This

  • HTML tables arrange data into rows and columns using <table>, <tr>, <th>, and <td> tags.
  • They are ideal for tabular data like reports, schedules, and price lists.

🧾 Syntax / Basic Structure ⭐

<table>
  <thead>
    <tr>
      <th>Item</th>
      <th>Price</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Idli</td>
      <td>₹30</td>
    </tr>
    <tr>
      <td>Dosa</td>
      <td>₹50</td>
    </tr>
  </tbody>
</table>

🔍 How It Works (Step-by-Step Execution) ⭐

  1. <table> wraps the whole table.
  2. <tr> defines a row; <th> defines header cells; <td> defines data cells.
  3. <thead>, <tbody>, and <tfoot> group rows semantically.

When to Use

  • For tabular data: price charts, statements, schedules, comparison tables.

When NOT to Use

  • Don’t use tables for page layout; use CSS (flex/grid) instead.
  • Don’t put large paragraphs in multiple cells just to align them.

Minimum You MUST Remember

  • Use <th> for header cells, <td> for normal cells.
  • Use CSS to style borders, spacing, and colors.

Common Mistakes

  • Missing closing tags for <tr> or <td> causing layout issues.
  • Overusing tables for layout, which makes pages hard to maintain and less accessible.

Interviewer’s Perspective

  • How do you create a table with headers and two rows?
  • Why is using tables for layout considered a bad practice?
