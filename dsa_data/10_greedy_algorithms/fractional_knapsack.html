💰 Fractional Knapsack & Coin Change

🎒 Fractional Knapsack

Unlike 0/1 knapsack, you can take fractions of items.

Greedy choice: Sort by value/weight ratio, take as much as possible of the highest-ratio item first.

double fractionalKnapsack(int[] wt, int[] val, int W) {
    Integer[] idx = IntStream.range(0, wt.length).boxed().toArray(Integer[]::new);
    Arrays.sort(idx, (a, b) -> Double.compare((double)val[b]/wt[b], (double)val[a]/wt[a]));
    double total = 0; int remaining = W;
    for (int i : idx) {
        if (remaining == 0) break;
        int take = Math.min(remaining, wt[i]);
        total += (double) take * val[i] / wt[i];
        remaining -= take;
    }
    return total;
}

🪙 Coin Change (Greedy works for canonical systems)

For standard coin systems (e.g., {1, 5, 10, 25}), greedy works: always take the largest coin possible. For arbitrary denominations, use DP.

🎯 Summary

Fractional Knapsack is a clear greedy win. Coin change is a pitfall — greedy works only for canonical denominations; use DP for the general case.
