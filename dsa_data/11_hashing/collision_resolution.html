💥 Collision Resolution

🎯 What is a Collision?

A collision occurs when two different keys hash to the same bucket index.

📋 Resolution Strategies

⛓️ Separate Chaining

Each bucket holds a linked list (or another structure). On collision, append to the list.

  • Simple to implement; never runs out of space
  • Performance degrades with long chains (bad hash function)

🔭 Open Addressing

On collision, probe for the next available bucket:

  • Linear probing: Check bucket+1, bucket+2, ... (simple but causes clustering)
  • Quadratic probing: Check bucket+1², bucket+2², ... (reduces clustering)
  • Double hashing: Use a second hash function to determine step size (best distribution)

📌 Java's Strategy

Java's HashMap uses separate chaining. Since Java 8, chains that exceed 8 entries convert to a balanced tree (O(log n) worst case instead of O(n)).

🎯 Summary

Separate chaining is simpler; open addressing avoids pointer overhead. Java HashMap uses chaining with tree fallback.
