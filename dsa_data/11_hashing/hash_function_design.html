🔧 Hash Function Design

🎯 Properties of a Good Hash Function

  • Deterministic: Same input always produces same output
  • Uniform distribution: Spreads keys evenly across buckets
  • Fast to compute: O(1) or O(k) where k = key length
  • Avalanche effect: Small change in input → large change in output

📐 Simple Integer Hash

hash(key) = key % tableSize — use a prime table size to reduce collisions.

♻️ Java's hashCode()

Every Java object has a hashCode() method. For Strings, Java computes: s[0]*31^(n-1) + s[1]*31^(n-2) + ... + s[n-1]

⚠️ Load Factor

When the ratio (number of entries / table size) exceeds ~0.75, the table is rehashed (size doubled, all entries re-inserted). Java's HashMap default load factor = 0.75.

🎯 Summary

Good hash functions distribute keys uniformly. Java's hashCode() contract requires that equal objects have equal hash codes — always override both equals() and hashCode() together.
