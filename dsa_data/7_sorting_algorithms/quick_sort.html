⚡ Quick Sort

🎯 Concept

Quick Sort picks a pivot, partitions the array so all elements less than pivot go left and greater go right, then recursively sorts both halves.

📐 Complexity

  • Average: O(n log n)
  • Worst case: O(n²) — pivot is always the min/max (always-sorted input)
  • Space: O(log n) stack (in-place)
  • Not stable
int partition(int[] arr, int low, int high) {
    int pivot = arr[high], i = low - 1;
    for (int j = low; j < high; j++) {
        if (arr[j] <= pivot) { i++; int t = arr[i]; arr[i] = arr[j]; arr[j] = t; }
    }
    int t = arr[i+1]; arr[i+1] = arr[high]; arr[high] = t;
    return i + 1;
}

📌 Optimization

Use random pivot selection or median-of-three to avoid worst case.

🎯 Summary

Quick Sort is the fastest in practice due to cache locality. Java's Arrays.sort() uses a Dual-Pivot Quicksort for primitives.
