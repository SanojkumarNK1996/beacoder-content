🧩 Common Array Problems

🔁 Two Pointers Technique

Use two indices that move toward each other to solve problems in O(n) instead of O(n²).

// Check if array has a pair summing to target
public boolean hasPairWithSum(int[] sorted, int target) {
    int left = 0, right = sorted.length - 1;
    while (left < right) {
        int sum = sorted[left] + sorted[right];
        if (sum == target) return true;
        else if (sum < target) left++;
        else right--;
    }
    return false;
}

🪟 Sliding Window Technique

Maintain a window of fixed or variable size as you scan the array.

// Maximum sum subarray of size k
public int maxSumSubarray(int[] arr, int k) {
    int sum = 0;
    for (int i = 0; i < k; i++) sum += arr[i];
    int max = sum;
    for (int i = k; i < arr.length; i++) {
        sum += arr[i] - arr[i - k];
        max = Math.max(max, sum);
    }
    return max;
}

➕ Prefix Sums

Precompute cumulative sums to answer range-sum queries in O(1).

🎯 Summary

Two pointers, sliding window, and prefix sums are essential array problem patterns that appear frequently in coding interviews.
