📊 Algorithm Analysis

🎯 What is Algorithm Analysis?

Algorithm analysis measures how the time and memory an algorithm uses grows as input size increases.

📐 Big O Notation

Big O describes the upper bound of an algorithm's growth rate — the worst case.

  • O(1) – Constant: doesn't depend on input size (e.g., array index access)
  • O(log n) – Logarithmic: halves the problem each step (e.g., binary search)
  • O(n) – Linear: scales with input (e.g., linear search)
  • O(n log n) – Log-linear: efficient sorting (e.g., merge sort)
  • O(n²) – Quadratic: nested loops (e.g., bubble sort)
  • O(2ⁿ) – Exponential: recursive brute-force problems

🧮 Space Complexity

Besides time, consider space complexity – how much extra memory an algorithm uses.

🎯 Summary

  • Use Big O to compare algorithm efficiency
  • Always consider both time and space trade-offs
