📈 Best, Average, and Worst Case Analysis

🎯 Three Cases

  • Best case – Fewest operations possible (e.g., target is the first element in linear search)
  • Average case – Expected performance for typical inputs
  • Worst case – Most operations possible (e.g., target not found in linear search)

🔍 Example: Linear Search

  • Best: O(1) — element at index 0
  • Average: O(n/2) → O(n)
  • Worst: O(n) — element not found

🔍 Example: Binary Search

  • Best: O(1) — target is the middle element
  • Average: O(log n)
  • Worst: O(log n) — element found at last comparison

🎯 Summary

In practice, worst-case analysis is most commonly used because it gives guarantees. Average-case is useful when inputs are random and typical.
