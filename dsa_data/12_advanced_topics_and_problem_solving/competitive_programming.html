🏆 Competitive Programming Techniques

⚡ Core Techniques

  • Sliding Window: O(n) solutions for subarray/substring problems with a fixed or variable size window
  • Two Pointers: Solve sorted-array and string problems in O(n) instead of O(n²)
  • Prefix Sums: Precompute cumulative sums for O(1) range sum queries
  • Monotonic Stack/Queue: Next Greater Element, Largest Rectangle in Histogram
  • Union-Find (DSU): Efficiently track connected components; used in Kruskal's MST
  • Segment Tree / Fenwick Tree: O(log n) range queries and point updates

📌 String Algorithms

  • KMP: Pattern matching in O(n+m)
  • Rabin-Karp: Hash-based pattern matching
  • Z-algorithm: Compute longest prefix suffix efficiently

🎯 Summary

These patterns are the "vocabulary" of competitive programming. Recognizing which pattern applies is the first step to solving hard interview problems.
