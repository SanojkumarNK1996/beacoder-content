🎯 Problem Solving Strategies

📋 General Approach

  1. Understand: Read carefully; clarify constraints (input size, edge cases)
  2. Example: Work through examples by hand
  3. Brute force first: Think of the simplest O(n²) or O(n³) solution
  4. Optimize: Apply patterns to reduce complexity
  5. Code: Write clean, working code
  6. Test: Edge cases (empty, single element, all same, already sorted)

🔑 Pattern Recognition Guide

  • Find/check in O(1) → HashMap / HashSet
  • Subarray with condition → Sliding window or prefix sum
  • Sorted array + pair → Two pointers
  • Best/max/min with overlapping subproblems → DP
  • All combinations/permutations → Backtracking
  • Shortest path in unweighted graph → BFS
  • Detect cycle / path existence → DFS
  • Minimize maximum / maximize minimum → Binary search on answer

🎯 Summary

Problem solving is a skill built by recognizing patterns and applying the right tool. Practice with intentional pattern-matching, not just random problem-solving.
