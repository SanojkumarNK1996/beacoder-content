🧾 SQL Syntax Basics (Beginner Friendly)

Now that you know what SQL is, let’s learn how to write SQL properly so that the database understands you.

1. 🎯 Why This Exists (Indian Real-Life Example)

  • ATM withdrawal:
    • When you enter your PIN and amount, the bank’s system runs a SQL query to check your balance.
    • If balance is enough, another query updates the remaining amount.
  • Swiggy / Zomato:
    • When you tap on “Veg only” or “Below ₹300”, SQL queries with conditions (filters) run in the backend.
    • Correct syntax ensures the app shows only the right restaurants.
  • College ERP / Marks portal:
    • When a faculty uploads marks, an INSERT or UPDATE query runs.
    • If syntax is wrong, marks will not be saved properly.

So, knowing the correct SQL syntax is like knowing the proper grammar of a language — without it, the database simply won’t understand you.

2. 📖 What Is This

  • SQL syntax = the rules and structure for writing SQL statements.
  • It tells you the order of words (SELECT, FROM, WHERE, etc.).
  • It also covers things like semicolons, comments, naming style, etc.
  • For beginners: think of it as “how to write a correct SQL sentence”.

3. 🧾 Syntax / Basic Structure ⭐

Most simple SQL queries follow this pattern:

SELECT column1, column2
FROM table_name
WHERE condition;

Example: Get names and age of students older than 18.

SELECT Name, Age
FROM Students
WHERE Age > 18;

Important syntax points:

  • Keywords like SELECT, FROM, WHERE are usually written in UPPERCASE.
  • Table and column names are usually written in lowercase or mixed_case, but be consistent.
  • Most databases allow one statement per semicolon (;).

a) Keywords and Case

These two queries are the same, but the second one is easier to read:

select * from Students;
SELECT * FROM Students;

b) Basic Clauses You’ll Use Daily

Clause Purpose Example
SELECT Which columns to show SELECT Name, Age
FROM From which table FROM Students
WHERE Filter rows based on condition WHERE Age > 18
ORDER BY Sort result ORDER BY Name ASC
GROUP BY Group rows by a column GROUP BY Department
HAVING Filter groups after GROUP BY HAVING COUNT(StudentID) > 5

4. 🔍 How It Works (Step-by-Step Execution) ⭐

Let’s take a full query and break it line by line:

SELECT Department, COUNT(StudentID) AS TotalStudents
FROM Students
WHERE Age > 18
GROUP BY Department
HAVING COUNT(StudentID) > 5
ORDER BY Department ASC;
  1. FROM Students
    • Database picks the Students table as the starting point.
  2. WHERE Age > 18
    • From all rows, it keeps only those where Age is greater than 18.
  3. GROUP BY Department
    • Rows are grouped department-wise (CS, ECE, MECH, etc.).
  4. COUNT(StudentID) AS TotalStudents
    • For each department group, it counts how many students are there.
  5. HAVING COUNT(StudentID) > 5
    • Now it removes departments that have 5 or fewer students.
  6. SELECT Department, COUNT(StudentID) AS TotalStudents
    • It decides which columns to show in the final output.
  7. ORDER BY Department ASC
    • Finally, it sorts the remaining rows alphabetically by department.

Even though we write the query top to bottom, most databases internally process it in steps like: FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY.

5. ✅ When to Use

  • Whenever you need to read or filter data from one or more tables.
  • When doing reports – total sales per city, students per department, etc.
  • When building dashboards for management, college admin, finance, etc.
  • Any time you are writing SELECT queries – you are using SQL syntax rules.

6. ⛔ When NOT to Use

  • You don’t “avoid” syntax – but you don’t overcomplicate simple queries.
  • Don’t write a single very long query if it can be broken into smaller, clearer ones.
  • Don’t use advanced clauses like GROUP BY/HAVING if a simple WHERE does the job.

7. 🧠 Minimum You MUST Remember

  • Basic SELECT pattern: SELECT ... FROM ... WHERE ...;
  • ORDER BY sorts, GROUP BY groups, HAVING filters groups.
  • Always try to write keywords in UPPERCASE for readability.
  • End your statements with ; (especially in tools that support multiple commands).
  • Use meaningful table and column names (not t1, c1 everywhere).

8. ⚠️ Common Mistakes

  • Forgetting the FROM clause or writing it in the wrong place.
  • Writing WHERE after GROUP BY instead of HAVING.
  • Using HAVING without GROUP BY (in most practical use cases).
  • Misplacing commas in the SELECT list or after the last column.
  • Using SELECT * everywhere instead of selecting only required columns.
  • Forgetting the semicolon ; and wondering why the tool is still “waiting”.

9. 🎤 Interviewer’s Perspective

Common fresher-level questions around syntax:

  • “Write a basic SELECT query to fetch data from a table.”
  • “Explain the difference between WHERE and HAVING.”
  • “In what order does the database execute SELECT, FROM, WHERE, GROUP BY, HAVING, ORDER BY?”
  • “Why do we use GROUP BY? Can you give a real-life example?”

What they expect:

  • Clear, simple query examples based on a Students or Employees table.
  • Understanding that WHERE filters rows and HAVING filters groups.
  • Ability to write and read a small multi-clause query without fear.

10. 📝 Practice Questions ⭐

  1. Very Easy – Identify Parts:

    Underline/mark the SELECT, FROM, and WHERE parts in this query:

    SELECT Name, City
FROM Customers
WHERE City = 'Mumbai';
  2. Easy – Write a Simple Query:

    There is a table Employees(EmpID, Name, Department, Salary).
    Write a query to show Name and Salary of employees in the 'IT' department.

  3. Easy – Use ORDER BY:

    Using the same Employees table, write a query to list all employees sorted by Salary in descending order.

  4. Medium – GROUP BY + HAVING:

    Consider a table Orders(OrderID, CustomerName, City, Amount).
    Write a query to show each City and the total Amount spent there, but only for cities where the total is more than ₹10,000.

  5. Medium – Explain in Words:

    In your own words, explain the difference between WHERE and HAVING using a simple college or shop example.
