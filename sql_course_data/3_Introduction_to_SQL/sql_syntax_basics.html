SQL Syntax Basics

Now that you know what SQL is and the types of SQL commands, let’s understand how to write SQL statements correctly.

Just like English sentences follow grammar rules, SQL statements follow a particular syntax (structure). This ensures that the database understands your request and performs the right action.

1. SQL Statement Structure

Most SQL statements follow a simple pattern:

COMMAND columns
FROM table_name
WHERE condition;

Example:

SELECT Name, Age
FROM Students
WHERE Age > 18;

This statement means: “Select the name and age of all students whose age is greater than 18.”

Note: Each SQL statement ends with a semicolon (;).

2. SQL Keywords

SQL uses a set of predefined words called keywords. These keywords are not case-sensitive, but it’s a good practice to write them in UPPERCASE for clarity.

Example: The following two queries are identical:

select * from Students;
SELECT * FROM Students;

Writing keywords in uppercase helps make your queries more readable.

3. Common SQL Clauses

SQL statements are built using smaller parts called clauses. Each clause performs a specific function in the query.

Clause Purpose Example
SELECT Specifies which columns to display SELECT Name, Age
FROM Specifies which table to get data from FROM Students
WHERE Filters data based on a condition WHERE Age > 18
ORDER BY Sorts the result (ascending or descending) ORDER BY Name ASC
GROUP BY Groups rows based on a column’s value GROUP BY Department
HAVING Applies a condition on grouped data HAVING COUNT(StudentID) > 5

4. Writing a Complete Query

Let’s combine these clauses to form a complete SQL query:

SELECT Department, COUNT(StudentID) AS TotalStudents
FROM Students
WHERE Age > 18
GROUP BY Department
HAVING COUNT(StudentID) > 5
ORDER BY Department ASC;

Explanation:

  • SELECT Department, COUNT(StudentID): We want to see each department and how many students are in it.
  • FROM Students: The data comes from the Students table.
  • WHERE Age > 18: Only include students older than 18.
  • GROUP BY Department: Group all students by their department.
  • HAVING COUNT(StudentID) > 5: Only show departments with more than 5 students.
  • ORDER BY Department ASC: Sort the results alphabetically by department name.

This is a typical example of how multiple SQL clauses work together to produce meaningful data.

5. Naming Conventions

  • Table and column names are not case-sensitive, but consistency is recommended.
  • Use snake_case or camelCase for naming (e.g., student_id or studentId).
  • Use clear, meaningful names for tables and columns.

6. SQL Comments

Comments are used to explain parts of your SQL code. They help you or others understand what your query is doing later.

Single-line comment:

-- This retrieves all students older than 18
SELECT * FROM Students WHERE Age > 18;

Multi-line comment:

/*
This query shows the number of students in each department
who are older than 18.
*/
SELECT Department, COUNT(StudentID)
FROM Students
WHERE Age > 18
GROUP BY Department;

7. Best Practices for Writing SQL

  • Use uppercase for SQL keywords (e.g., SELECT, WHERE, FROM).
  • End every statement with a semicolon ;.
  • Indent your SQL for readability when using multiple clauses.
  • Use comments generously to describe your logic.
  • Always test your queries on a small dataset before applying them to production data.

8. Real-World Example

Imagine a small online store with a table called Orders:

+----------+-------------+------------+----------+
| Order_ID | Customer    | Product    | Amount   |
+----------+-------------+------------+----------+
| 1        | Riya        | Laptop     | 50000    |
| 2        | Arjun       | Keyboard   | 1500     |
| 3        | Meera       | Mouse      | 800      |
| 4        | Riya        | Monitor    | 7000     |
+----------+-------------+------------+----------+

If we want to know how much each customer spent in total, we could write:

SELECT Customer, SUM(Amount) AS Total_Spent
FROM Orders
GROUP BY Customer
ORDER BY Total_Spent DESC;

Output:

+----------+--------------+
| Customer | Total_Spent  |
+----------+--------------+
| Riya     | 57000        |
| Arjun    | 1500         |
| Meera    | 800          |
+----------+--------------+

SQL helps you find exactly what you need from large amounts of data — quickly and efficiently.

Summary

  • SQL syntax defines how SQL commands are written and structured.
  • Clauses like SELECT, FROM, WHERE, and ORDER BY form the building blocks of SQL queries.
  • Use comments and naming conventions for clean, understandable code.
  • Always follow proper formatting for clarity and maintenance.

By mastering SQL syntax, you can easily create, query, and manage any database — whether for a small app or a large enterprise system.
