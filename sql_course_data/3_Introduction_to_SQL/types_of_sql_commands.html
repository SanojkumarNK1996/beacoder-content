Types of SQL Commands: 10-Part Beginner-Friendly Guide

1. Why Does This Topic Exist?

When you use a database—like for a college admission system or a railway booking app—you need to create tables, add data, change it, and control who can access it. SQL commands help you do all these things in a structured way.

2. What Is This?

SQL commands are instructions you give to a database. They are grouped into four main types: DDL, DML, DCL, and TCL. Each type has a special job, just like different tools in a toolbox.

3. Syntax/Structure

  • DDL (Data Definition Language): CREATE TABLE Students (ID INT, Name VARCHAR(50));
  • DML (Data Manipulation Language): INSERT INTO Students VALUES (1, 'Riya');
  • DCL (Data Control Language): GRANT SELECT ON Students TO user1;
  • TCL (Transaction Control Language): COMMIT; or ROLLBACK;

4. Step-by-Step: How It Works

  1. DDL: Design your database structure (like making a blueprint for a house).
  2. DML: Add, update, or remove data (like putting furniture in the house).
  3. DCL: Decide who can enter or change things (like giving keys to family members).
  4. TCL: Save or undo changes (like saving your work or pressing undo in MS Word).

5. When to Use

  • DDL: When you need to create or change tables (e.g., new semester, new student fields).
  • DML: When you want to add or update student marks, delete old records, or view data.
  • DCL: When you want to give teachers access to marks but not to fee details.
  • TCL: When you want to make sure all changes are saved together, or undo if there’s a mistake.

6. When Not to Use

  • Don’t use DDL for changing data (it’s for structure, not content).
  • Don’t use DML to change who can access the database (use DCL for that).
  • Don’t forget TCL when making multiple changes that must succeed or fail together (like transferring money between accounts).

7. Minimum to Remember

  • DDL = Structure, DML = Data, DCL = Control, TCL = Transactions.
  • Each type has its own set of commands and purpose.

8. Common Mistakes

  • Using DML commands before creating tables with DDL.
  • Forgetting to use COMMIT after making changes (data may not be saved!).
  • Giving too many permissions with DCL (security risk).

9. Interviewer’s Perspective

  • “Explain the difference between DDL and DML with examples.”
  • “Why is TCL important in banking applications?”
  • “How would you restrict access to a table for a specific user?”

10. Practice Questions

  1. Write a DDL command to create a table for storing Indian city names and their populations.
  2. Give an example of a DML command to update a student’s marks.
  3. How would you use DCL to allow only the principal to view fee details?
  4. What is the use of ROLLBACK in TCL? Give a real-life scenario.
