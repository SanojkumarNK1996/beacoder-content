Types of SQL Commands

SQL (Structured Query Language) provides different types of commands to perform various tasks in a database — like creating tables, inserting data, updating records, or managing permissions.

All SQL commands are grouped into four main categories:

  • DDL – Data Definition Language
  • DML – Data Manipulation Language
  • DCL – Data Control Language
  • TCL – Transaction Control Language

Let’s explore each one in detail with examples.

1. DDL – Data Definition Language

DDL commands are used to define or modify the structure of the database — such as creating, altering, or deleting tables and other database objects.

Common DDL Commands:

  • CREATE – Creates a new table or database
  • ALTER – Modifies an existing table (add/remove columns)
  • DROP – Deletes a table or database permanently
  • TRUNCATE – Removes all data from a table but keeps the structure

Example:

CREATE TABLE Students (
    StudentID INT,
    Name VARCHAR(50),
    Age INT
);

This creates a new table named Students with three columns: StudentID, Name, and Age.

Analogy: DDL is like designing or remodeling a building — you create or change the structure before people (data) move in.

2. DML – Data Manipulation Language

DML commands are used to work with the actual data inside the database. Once the structure exists (thanks to DDL), DML helps you add, change, or remove information.

Common DML Commands:

  • INSERT – Add new data
  • UPDATE – Modify existing data
  • DELETE – Remove data
  • SELECT – Retrieve data from one or more tables

Examples:

-- Add a new student
INSERT INTO Students (StudentID, Name, Age)
VALUES (1, 'Riya', 20);

-- Update an existing record
UPDATE Students
SET Age = 21
WHERE StudentID = 1;

-- Retrieve all students
SELECT * FROM Students;

Analogy: If the database is a house, DML is how you manage what’s inside — adding furniture (data), rearranging it, or removing it.

3. DCL – Data Control Language

DCL commands manage access and permissions to the database. They control who can view, modify, or delete data.

Common DCL Commands:

  • GRANT – Give access or privileges to a user
  • REVOKE – Remove previously given privileges

Example:

-- Give permission to a user to view the Students table
GRANT SELECT ON Students TO user1;

-- Remove permission
REVOKE SELECT ON Students FROM user1;

Analogy: DCL is like giving or taking away keys to a room — you decide who can enter, who can make changes, and who can’t.

4. TCL – Transaction Control Language

TCL commands deal with the transactions in a database — ensuring data integrity and allowing you to control how and when changes are permanently saved.

A transaction is a group of one or more SQL operations that are treated as a single unit of work. If any part fails, the entire transaction can be rolled back to keep the data consistent.

Common TCL Commands:

  • COMMIT – Save all changes permanently
  • ROLLBACK – Undo changes made by the current transaction
  • SAVEPOINT – Set a temporary save point within a transaction

Example:

BEGIN TRANSACTION;

UPDATE Students SET Age = 22 WHERE StudentID = 1;

-- Oops! If something goes wrong:
ROLLBACK;

-- If all good:
COMMIT;

Analogy: TCL is like “Save” and “Undo” in a document editor. You can commit your changes to make them permanent, or rollback to revert them.

Summary

Category Full Form Purpose Examples
DDL Data Definition Language Defines database structure CREATE, ALTER, DROP, TRUNCATE
DML Data Manipulation Language Works with actual data SELECT, INSERT, UPDATE, DELETE
DCL Data Control Language Manages access and permissions GRANT, REVOKE
TCL Transaction Control Language Manages transactions and saves/undoes changes COMMIT, ROLLBACK, SAVEPOINT

Together, these four command types cover everything you need to work with a database — from creating its structure to managing data, access, and safe transactions.
