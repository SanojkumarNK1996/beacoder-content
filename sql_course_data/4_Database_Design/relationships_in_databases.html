Relationships in Databases: 10-Part Beginner-Friendly Guide

1. Why Does This Topic Exist?

In real life, everything is connected: students join courses, doctors treat patients, customers place orders. Databases need to capture these connections to make data useful and meaningful.

2. What Is This?

A relationship in a database is a connection between two or more tables, usually using keys. It helps you link related data together, like which student is in which class.

3. Syntax/Structure

-- Example: One-to-Many (1:N)
CREATE TABLE Teachers (
  TeacherID INT PRIMARY KEY,
  Name VARCHAR(50)
);

CREATE TABLE Courses (
  CourseID INT PRIMARY KEY,
  CourseName VARCHAR(50),
  TeacherID INT,
  FOREIGN KEY (TeacherID) REFERENCES Teachers(TeacherID)
);

4. Step-by-Step: How It Works

  1. Decide which tables need to be connected (e.g., Students and Courses).
  2. Add a key (usually a Primary Key) to one table.
  3. Add a Foreign Key in the other table to create the link.
  4. For many-to-many, create a third “junction” table with keys from both tables.

5. When to Use

  • Whenever you need to connect related data (students to courses, orders to customers, etc.).
  • To avoid repeating the same information in multiple places.
  • To make searching and reporting easier.

6. When Not to Use

  • For very simple, single-table data (like a personal contact list).
  • When you don’t need to connect data across tables.

7. Minimum to Remember

  • Relationships are built using Primary Keys and Foreign Keys.
  • Three main types: One-to-One, One-to-Many, Many-to-Many.
  • Many-to-Many needs a junction table.

8. Common Mistakes

  • Forgetting to use Foreign Keys, leading to “orphan” records.
  • Trying to store many-to-many relationships in just two tables (always use a junction table).
  • Not enforcing referential integrity (allowing invalid links).

9. Interviewer’s Perspective

  • “Explain the difference between One-to-Many and Many-to-Many relationships.”
  • “How do you implement a many-to-many relationship in SQL?”
  • “What is a foreign key? Why is it important?”

10. Practice Questions

  1. Write SQL to create a junction table for students and courses.
  2. Give a real-life Indian example of a one-to-many relationship.
  3. What happens if you don’t use foreign keys in your database?
