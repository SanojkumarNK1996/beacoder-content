Indexing Basics

As your database grows, it may store thousands or even millions of records. Finding a specific record in such a large table can take time if the database has to check every single row.

This is where Indexing comes in. An Index makes data search faster — just like an index in the back of a book helps you quickly find a topic without reading every page.

What is an Index?

An Index is a special data structure that stores a small portion of your data in an organized way (usually sorted), so the database can locate information faster.

Example (Book Analogy):

Imagine a 500-page book. You want to find where “Database Normalization” is mentioned.

  • Without an index — you’d flip through every page (slow).
  • With an index — you look at the back of the book, find “Database Normalization → page 120” (fast).

The same idea applies in databases. Instead of checking every record, the database uses an index to jump directly to the location where the data exists.

Why Indexing is Important

Indexes improve the speed of data retrieval (SELECT queries) dramatically, especially in large databases.

However, indexes also take up extra storage space and can slightly slow down data modifications (INSERT, UPDATE, DELETE), because the index needs to be updated whenever data changes.

In short:

  • ✅ Speeds up searches
  • ❌ Slightly slows down data updates
  • ✅ Great for frequently searched columns

How Indexing Works (Simple Example)

Let’s say we have a table of students:

Students Table
+-----------+-----------+--------+
| StudentID | Name      | Age    |
+-----------+-----------+--------+
| 1         | Riya      | 21     |
| 2         | Arjun     | 22     |
| 3         | Meera     | 20     |
| 4         | Ravi      | 23     |
+-----------+-----------+--------+

Without an index, if you run this query:

SELECT * FROM Students WHERE Name = 'Meera';

The database has to check every row until it finds “Meera.” This process is called a full table scan.

If we create an index on the Name column:

CREATE INDEX idx_name ON Students(Name);

Indexing in SQL: 10-Part Beginner-Friendly Guide

1. Why Does This Topic Exist?

Imagine searching for your name in a list of 10 crore Aadhaar records! Indexing helps databases find information quickly, just like the index in the back of a textbook or a signboard in a supermarket.

2. What Is This?

An index is a special data structure that makes searching for data in a table much faster. It’s like a shortcut for the database to jump directly to the right place, instead of checking every row one by one.

3. Syntax/Structure

-- Create an index on the Name column
CREATE INDEX idx_name ON Students(Name);

-- Create a unique index (no duplicate emails)
CREATE UNIQUE INDEX idx_email ON Users(Email);

-- Composite index (for searching by Name and Age together)
CREATE INDEX idx_name_age ON Students(Name, Age);

4. Step-by-Step: How It Works

  1. Database creates a sorted list (index) for the chosen column(s).
  2. When you search, the database checks the index (fast), not every row (slow).
  3. If you update or insert data, the index is also updated.
  4. Indexes use structures like B-Trees or Hashes internally (handled by the database).

5. When to Use

  • When a column is often used in WHERE, JOIN, ORDER BY, or GROUP BY.
  • For large tables (thousands or millions of rows).
  • For columns that must be unique (like email or Aadhaar number).

6. When Not to Use

  • Don’t create indexes on every column (wastes space, slows down updates).
  • Don’t use indexes for very small tables (no real benefit).
  • Don’t use indexes on columns that change very frequently.

7. Minimum to Remember

  • Index = shortcut for searching.
  • Primary key columns are indexed automatically.
  • Too many indexes = slower updates.

8. Common Mistakes

  • Indexing every column “just in case.”
  • Forgetting to index columns used in JOINs or WHERE for big tables.
  • Not removing unused indexes (wastes space).

9. Interviewer’s Perspective

  • “What is an index? Why do we need it?”

  • “What is a composite index? Give an example.”
