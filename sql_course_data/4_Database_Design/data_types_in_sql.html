Data Types in SQL: 10-Part Beginner-Friendly Guide

1. Why Does This Topic Exist?

When you fill a form online (like Aadhaar, college admission, or Swiggy order), each field expects a certain type of data: numbers, names, dates, or even images. Databases use data types to make sure each column stores the right kind of information, saves space, and avoids mistakes.

2. What Is This?

A data type in SQL defines what kind of value a column can hold—like numbers, text, dates, or files. Choosing the right data type is crucial for accuracy and efficiency.

3. Syntax/Structure

CREATE TABLE Students (
  StudentID INT,
  Name VARCHAR(50),
  Age INT,
  JoinDate DATE
);
  • INT: Whole numbers (roll numbers, IDs)
  • VARCHAR(50): Text up to 50 characters (names, emails)
  • DATE: Dates (birthdays, joining date)

4. Step-by-Step: How It Works

  1. Decide what kind of data each column will store (number, text, date, etc.).
  2. Pick the most suitable data type for each column.
  3. Write the CREATE TABLE statement with chosen data types.
  4. When you insert data, the database checks if it matches the data type.
  5. If you try to store text in a number column, the database will show an error.

5. When to Use

  • Every time you create a new table or add a new column.
  • When you want to ensure only valid data is stored (e.g., age must be a number).
  • To save space and improve performance (e.g., use INT for IDs, not VARCHAR).

6. When Not to Use

  • Don’t use a large data type (like TEXT) for small values (like age or pin code).
  • Don’t use VARCHAR for dates or numbers—use DATE or INT instead.
  • Don’t use BLOB or JSON unless you really need to store files or complex data.

7. Minimum to Remember

  • INT for numbers, VARCHAR for text, DATE for dates.
  • Pick the smallest data type that fits your needs.
  • Wrong data type = errors or wasted space.

8. Common Mistakes

  • Storing phone numbers as INT (should be VARCHAR, as they may start with 0 or have +91).
  • Using VARCHAR for everything (makes searching and sorting slow).
  • Forgetting to set the right size for VARCHAR (too small = errors, too big = wasted space).

9. Interviewer’s Perspective

  • “What data type would you use for a student’s date of birth?”
  • “Why not store salary as VARCHAR?”
  • “How would you store a profile photo in a database?”
