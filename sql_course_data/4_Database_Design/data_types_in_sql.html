Data Types in SQL

In a database, every column in a table must store a specific type of data — such as numbers, text, or dates. This type of data is called a Data Type.

Choosing the correct data type ensures that your data is stored efficiently and correctly — for example, you can’t store a date as text if you want to compare it properly.

What Are Data Types?

A Data Type defines what kind of values a column can hold.

Example:

CREATE TABLE Students (
  StudentID INT,
  Name VARCHAR(50),
  Age INT,
  JoinDate DATE
);

Here:

  • StudentID → stores a number (INT)
  • Name → stores text (VARCHAR)
  • Age → stores a number (INT)
  • JoinDate → stores a date (DATE)

Main Categories of SQL Data Types

SQL data types can be grouped into several main categories:

  1. Numeric Types
  2. Character (String) Types
  3. Date and Time Types
  4. Boolean Type
  5. Binary and Miscellaneous Types

1️⃣ Numeric Data Types

Used for storing numbers — both whole numbers and decimals.

Data Type Example Value Description
INT / INTEGER 42 Stores whole numbers (no decimals).
BIGINT 9876543210 Stores very large whole numbers.
DECIMAL(p, s) 12.75 Fixed number with decimals — p = total digits, s = digits after decimal.
FLOAT / DOUBLE 3.14159 Stores floating-point (approximate) decimal numbers.

Example:

Price DECIMAL(6,2);  -- Allows values like 1234.56

2️⃣ Character (String) Data Types

Used to store letters, numbers, and symbols as text.

Data Type Example Value Description
CHAR(n) 'ABC' Fixed-length text (always stores exactly n characters).
VARCHAR(n) 'Hello World' Variable-length text — stores up to n characters.
TEXT 'A long article or paragraph.' Used for large amounts of text.

Example:

Name VARCHAR(50); -- Stores up to 50 characters

3️⃣ Date and Time Data Types

Used to store date, time, or both.

Data Type Example Value Description
DATE '2025-10-31' Stores only the date (year, month, day).
TIME '14:30:00' Stores only the time (hours, minutes, seconds).
DATETIME '2025-10-31 14:30:00' Stores both date and time together.
TIMESTAMP '2025-10-31 14:30:00' Similar to DATETIME, often used for tracking record creation or update times.

Example:

CreatedAt TIMESTAMP DEFAULT CURRENT_TIMESTAMP;

4️⃣ Boolean Data Type

Used to store True or False values.

Data Type Example Value Description
BOOLEAN TRUE / FALSE Stores logical values (often represented as 1 for TRUE, 0 for FALSE).

Example:

IsActive BOOLEAN DEFAULT TRUE;

5️⃣ Binary and Miscellaneous Data Types

Used for storing non-text data such as images, files, or raw data.

Data Type Example Value Description
BLOB (Binary Large Object) (binary data) Used for storing images, audio, or other binary files.
JSON { "name": "Riya", "age": 21 } Stores structured data in JSON format.

Choosing the Right Data Type

  • Use INT for numbers without decimals (IDs, counts).
  • Use DECIMAL or FLOAT for prices or measurements.
  • Use VARCHAR for text that varies in length (names, emails).
  • Use DATE or DATETIME for time-based information.
  • Use BOOLEAN for true/false flags.
  • Use BLOB or JSON only when necessary (as they take more space).

Real-Life Example

Let’s design a small Employees table using different data types:

CREATE TABLE Employees (
  EmployeeID INT PRIMARY KEY,
  Name VARCHAR(100),
  Age INT,
  Salary DECIMAL(10,2),
  JoiningDate DATE,
  IsActive BOOLEAN DEFAULT TRUE
);

This table uses:

  • INT for IDs and Age
  • VARCHAR for names
  • DECIMAL for salary
  • DATE for joining date
  • BOOLEAN to track if the employee is active

Summary

Category Examples Used For
Numeric INT, DECIMAL, FLOAT Numbers, prices, quantities
Character CHAR, VARCHAR, TEXT Names, addresses, descriptions
Date/Time DATE, TIME, DATETIME Tracking when things happen
Boolean BOOLEAN Yes/No or True/False data
Binary / Misc BLOB, JSON Images, files, structured data

In short: Choosing the right data type helps your database run faster, save space, and prevent errors. Think carefully about what kind of data you want to store before designing your tables.
