Savepoints in SQL

1️⃣ Why Does This Topic Exist?

Savepoints help you undo only part of a transaction, not the whole thing. In Indian banking or railway booking, this can help fix just the last step if something goes wrong, without losing all progress.

2️⃣ What Is a Savepoint?

A savepoint is a checkpoint inside a transaction. You can roll back to a savepoint if you make a mistake, instead of rolling back the entire transaction.

3️⃣ Syntax/Structure

SAVEPOINT savepoint_name;
-- SQL statements
ROLLBACK TO SAVEPOINT savepoint_name;
RELEASE SAVEPOINT savepoint_name;

4️⃣ Step-by-Step Execution

  1. Start a transaction with BEGIN.
  2. Do some work, then set a savepoint.
  3. Do more work. If there’s a mistake, use ROLLBACK TO SAVEPOINT to undo only the recent changes.
  4. Commit the transaction when done.

5️⃣ When to Use

  • When you want to undo only part of a transaction (e.g., batch updates, multi-step processes).
  • When debugging or testing complex changes.

6️⃣ When Not to Use

  • For very simple transactions with only one or two steps.
  • If your database does not support savepoints.

7️⃣ Minimum to Remember

  • Savepoints are checkpoints inside a transaction.
  • Use ROLLBACK TO SAVEPOINT to undo only recent changes.

8️⃣ Common Mistakes

  • Forgetting to release savepoints (in some databases, this can use extra resources).
  • Assuming all databases support savepoints.

9️⃣ Interviewer’s Perspective

  • What is a savepoint? How is it different from a transaction?
  • Give a real-life Indian example where savepoints are useful (e.g., batch salary processing).
  • How do you roll back to a savepoint?

🔟 Practice Questions

  1. Write SQL to use a savepoint and roll back to it after an error.
  2. What is the difference between ROLLBACK and ROLLBACK TO SAVEPOINT?
  3. When would you use multiple savepoints in a transaction?
  4. How do you release a savepoint?

7️⃣ Important Notes

  • Savepoints only work inside an active transaction (after BEGIN).
  • Using ROLLBACK without specifying a savepoint undoes the entire transaction.
  • Some databases automatically release savepoints after a ROLLBACK TO or COMMIT.

8️⃣ Summary

  • Savepoints let you create checkpoints in a transaction.
  • ROLLBACK TO SAVEPOINT undoes only part of the transaction.
  • They are useful for handling partial errors gracefully.
