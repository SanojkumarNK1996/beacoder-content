Transactions in SQL

In a database, a transaction is a group of one or more operations that are treated as a single logical unit of work. This means that either all the operations in the transaction are completed successfully, or none of them are applied to the database.

Transactions are used to maintain data consistency and integrity, especially when multiple users or systems are interacting with the database at the same time.

1️⃣ What Is a Transaction?

Think of a transaction as a process that must be completed fully or not at all.

Example: Imagine transferring ₹100 from Account A to Account B.

  • Step 1: Deduct ₹100 from Account A
  • Step 2: Add ₹100 to Account B

If something goes wrong after Step 1 (like a power failure or network error), the transaction should not be completed — otherwise, ₹100 will be lost. SQL transactions ensure that both steps happen together, or neither happens at all.

2️⃣ Why Are Transactions Important?

Transactions help maintain the ACID properties of a database:

  • Atomicity — All or nothing.
  • Consistency — Data remains valid before and after the transaction.
  • Isolation — One transaction doesn’t interfere with another.
  • Durability — Once committed, the data is permanently saved.

These properties ensure reliability even when multiple users are working with the same data at the same time.

3️⃣ Basic Transaction Commands in SQL

SQL provides three main commands to manage transactions:

  • BEGIN — Starts a new transaction.
  • COMMIT — Saves all the changes made during the transaction permanently.
  • ROLLBACK — Cancels the transaction and restores the database to its previous state.

4️⃣ Example of a Transaction

Let’s see how these commands are used in a real example:

BEGIN;

UPDATE accounts SET balance = balance - 100 WHERE account_id = 1;
UPDATE accounts SET balance = balance + 100 WHERE account_id = 2;

COMMIT;

Here’s what happens:

  • The BEGIN command starts the transaction.
  • The two UPDATE statements transfer ₹100 between accounts.
  • If everything works correctly, COMMIT saves the changes permanently.

If an error occurs during the process, you can cancel everything using:

ROLLBACK;

This ensures that no partial updates remain — both accounts stay as they were before.

5️⃣ Real-Life Analogy

Think of a bank transfer as a transaction. If the debit happens but the credit fails, it would cause an error in your account balances. That’s why the system ensures that both steps are completed successfully — otherwise, it reverts everything back.

Transactions in SQL help databases work exactly like that — ensuring accuracy, reliability, and safety of data.

6️⃣ When to Use Transactions

Transactions are especially useful when:

  • You are transferring money between accounts.
  • Making an order and updating inventory at the same time.
  • Processing multiple changes that must all succeed together.
  • Preventing partial updates during errors or power failures.

7️⃣ Summary

  • A transaction is a set of operations that act as one unit.
  • It ensures consistency and prevents data corruption.
  • Use BEGIN, COMMIT, and ROLLBACK to manage transactions.
