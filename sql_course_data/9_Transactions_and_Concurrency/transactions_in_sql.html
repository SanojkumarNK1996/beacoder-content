Transactions in SQL

1️⃣ Why Does This Topic Exist?

Transactions are used to keep data safe and consistent, especially when multiple people or systems are using the database at the same time. In Indian banks, railway bookings, and e-commerce, transactions prevent money loss or double bookings.

2️⃣ What Is a Transaction?

A transaction is a group of one or more operations (like insert, update, delete) that are treated as a single unit. Either all changes happen, or none do. This prevents partial updates and keeps data correct.

3️⃣ Syntax/Structure

BEGIN;
-- SQL statements
COMMIT;   -- or ROLLBACK;

4️⃣ Step-by-Step Execution

  1. Start a transaction with BEGIN.
  2. Make your changes (INSERT, UPDATE, DELETE).
  3. If everything is correct, use COMMIT to save changes.
  4. If there is an error, use ROLLBACK to undo all changes.

5️⃣ When to Use

  • Transferring money between accounts.
  • Booking tickets and updating seat availability.
  • Making orders and updating inventory together.
  • Any situation where all steps must succeed together.

6️⃣ When Not to Use

  • For single, simple queries that do not affect multiple rows or tables.
  • When performance is more important than strict consistency (rare in real-world Indian business).

7️⃣ Minimum to Remember

  • Transactions = all or nothing.
  • Use BEGIN, COMMIT, ROLLBACK.
  • Prevents data corruption and partial updates.

8️⃣ Common Mistakes

  • Forgetting to use COMMIT or ROLLBACK at the end.
  • Leaving transactions open, which can lock tables and slow down the system.
  • Not handling errors, so partial changes remain.
