Isolation Levels in SQL

Isolation levels define how transaction integrity is maintained when multiple users access the same data concurrently. They control the visibility of changes made by one transaction to others.

1. READ UNCOMMITTED

The lowest isolation level. Transactions can read uncommitted changes made by others — known as dirty reads. 

SET TRANSACTION ISOLATION LEVEL READ UNCOMMITTED;
SELECT * FROM orders;

2. READ COMMITTED

Prevents dirty reads by only allowing access to committed data. However, non-repeatable reads and phantom reads may still occur. 

SET TRANSACTION ISOLATION LEVEL READ COMMITTED;
SELECT * FROM customers WHERE city = 'Delhi';

3. REPEATABLE READ

Ensures that if a transaction reads a row twice, the data will not change in between. Prevents dirty and non-repeatable reads, but phantom rows can still appear. 

SET TRANSACTION ISOLATION LEVEL REPEATABLE READ;
SELECT * FROM accounts WHERE type = 'savings';

4. SERIALIZABLE

The highest isolation level. Transactions are executed as if they were serialized one after another. Prevents dirty, non-repeatable, and phantom reads — but reduces concurrency. 

SET TRANSACTION ISOLATION LEVEL SERIALIZABLE;
SELECT * FROM products WHERE category = 'Electronics';
