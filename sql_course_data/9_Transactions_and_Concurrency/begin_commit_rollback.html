BEGIN, COMMIT, and ROLLBACK in SQL

1️⃣ Why Does This Topic Exist?

These commands help you control transactions and keep your data safe. In Indian banking, ticket booking, and e-commerce, they prevent mistakes like double debits or incomplete orders.

2️⃣ What Are BEGIN, COMMIT, and ROLLBACK?

  • BEGIN: Starts a transaction (a group of changes).
  • COMMIT: Saves all changes made in the transaction.
  • ROLLBACK: Cancels all changes since BEGIN.

3️⃣ Syntax/Structure

BEGIN;
-- SQL statements
COMMIT;   -- or ROLLBACK;

4️⃣ Step-by-Step Execution

  1. Start with BEGIN.
  2. Make your changes (INSERT, UPDATE, DELETE).
  3. If all is well, use COMMIT to save.
  4. If there’s a problem, use ROLLBACK to undo.

5️⃣ When to Use

  • When you need to make several changes that must all succeed together (e.g., transferring money, booking tickets).
  • When you want to prevent partial updates in case of errors.

6️⃣ When Not to Use

  • For single, simple queries that don’t need to be grouped.
  • When you don’t need to undo changes if something fails.

7️⃣ Minimum to Remember

  • BEGIN starts, COMMIT saves, ROLLBACK undoes.
  • Always end a transaction with COMMIT or ROLLBACK.

8️⃣ Common Mistakes

  • Forgetting to commit or rollback, leaving the transaction open.
  • Not handling errors, so partial changes remain.

9️⃣ Interviewer’s Perspective

  • What is the difference between COMMIT and ROLLBACK?
  • Give a real-life Indian example where rollback is important (e.g., failed UPI payment).
  • What happens if you forget to commit a transaction?

🔟 Practice Questions

  1. Write SQL to start a transaction, update two accounts, and commit.
  2. How would you undo changes if an error occurs during a transaction?
  3. What happens if you don’t use COMMIT or ROLLBACK?
  4. Give an example of a real-world scenario where rollback is used.
COMMIT; -- If an error occurs: ROLLBACK;

Explanation:

  • BEGIN starts the transaction.
  • Two updates are made.
  • If both updates succeed, COMMIT saves the changes.
  • If one fails, ROLLBACK cancels everything.

5️⃣ Important Points to Remember

  • Use transactions to group multiple related changes together.
  • Always COMMIT when you’re sure the transaction is correct.
  • Use ROLLBACK when something goes wrong.
  • Without BEGIN, some databases automatically commit each statement (this is called autocommit mode).

6️⃣ Real-World Analogy

Imagine you are shopping online:

  • Adding items to your cart → BEGIN
  • Clicking “Place Order” and payment succeeds → COMMIT
  • Payment fails or order is cancelled → ROLLBACK

This ensures your data (money, order, inventory) always stays accurate.

7️⃣ Summary

  • BEGIN — Starts a transaction.
  • COMMIT — Saves all changes permanently.
  • ROLLBACK — Undoes all uncommitted changes.

Together, these commands ensure that your database transactions are safe, consistent, and reliable.
