BEGIN, COMMIT, and ROLLBACK in SQL

When working with databases, it’s common to make changes like adding, updating, or deleting data. But sometimes, things don’t go as planned — maybe the system crashes or a query runs incorrectly. That’s where transaction control commands — BEGIN, COMMIT, and ROLLBACK — help keep your data safe and consistent.

1️⃣ BEGIN (Start a Transaction)

The BEGIN command starts a new transaction. It tells the database that a set of changes will be made as one single unit of work.

Syntax:

BEGIN;
-- SQL statements go here

Example:

BEGIN;
UPDATE accounts SET balance = balance - 100 WHERE account_id = 1;
UPDATE accounts SET balance = balance + 100 WHERE account_id = 2;

Here, both updates happen within the same transaction. No change will be made permanent until you use the COMMIT command.

2️⃣ COMMIT (Save Changes)

The COMMIT command is used to save all the changes made in the current transaction permanently to the database.

Syntax:

COMMIT;

Example:

BEGIN;
UPDATE accounts SET balance = balance - 100 WHERE account_id = 1;
UPDATE accounts SET balance = balance + 100 WHERE account_id = 2;
COMMIT;

Once you execute COMMIT:

  • The database confirms that all changes are successful.
  • Data is permanently stored and cannot be undone using ROLLBACK.
  • Other users can now see the changes.

Real-world example: When you complete an online money transfer and get a “Transaction Successful” message — that’s like the COMMIT step. It means your transaction has been safely saved in the system.

3️⃣ ROLLBACK (Undo Changes)

The ROLLBACK command is used to cancel a transaction and undo all the changes made since the last BEGIN.

Syntax:

ROLLBACK;

Example:

BEGIN;
UPDATE accounts SET balance = balance - 100 WHERE account_id = 1;
UPDATE accounts SET balance = balance + 100 WHERE account_id = 2;

-- Suppose there’s an error here
ROLLBACK;

After executing ROLLBACK, all changes are undone. It’s as if the transaction never happened.

Real-world example: When your online payment fails midway, the deducted money is automatically reversed — that’s similar to a ROLLBACK operation.

4️⃣ Transaction Flow Example

Here’s a full example showing how these commands work together:

BEGIN;

UPDATE accounts SET balance = balance - 100 WHERE account_id = 1;
UPDATE accounts SET balance = balance + 100 WHERE account_id = 2;

-- If both succeed:
COMMIT;

-- If an error occurs:
ROLLBACK;

Explanation:

  • BEGIN starts the transaction.
  • Two updates are made.
  • If both updates succeed, COMMIT saves the changes.
  • If one fails, ROLLBACK cancels everything.

5️⃣ Important Points to Remember

  • Use transactions to group multiple related changes together.
  • Always COMMIT when you’re sure the transaction is correct.
  • Use ROLLBACK when something goes wrong.
  • Without BEGIN, some databases automatically commit each statement (this is called autocommit mode).

6️⃣ Real-World Analogy

Imagine you are shopping online:

  • Adding items to your cart → BEGIN
  • Clicking “Place Order” and payment succeeds → COMMIT
  • Payment fails or order is cancelled → ROLLBACK

This ensures your data (money, order, inventory) always stays accurate.

7️⃣ Summary

  • BEGIN — Starts a transaction.
  • COMMIT — Saves all changes permanently.
  • ROLLBACK — Undoes all uncommitted changes.

Together, these commands ensure that your database transactions are safe, consistent, and reliable.
