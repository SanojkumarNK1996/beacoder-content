Grouping Data (GROUP BY, HAVING): 10-Part Beginner-Friendly Guide

1. Why Does This Topic Exist?

In Indian schools, businesses, or government reports, you often need summaries by category—like average marks per subject, sales per city, or number of users per state. GROUP BY and HAVING help you organize and analyze data by groups.

2. What Is This?

GROUP BY is an SQL clause that groups rows sharing the same value in specified columns, so you can run aggregate functions (like COUNT, AVG) on each group. HAVING filters these groups after aggregation.

3. Syntax/Structure

-- Group by one column
SELECT column, AGG_FUNC(column2)
FROM table_name
GROUP BY column;

-- Group by multiple columns
SELECT col1, col2, AGG_FUNC(col3)
FROM table_name
GROUP BY col1, col2;

-- Filter groups
SELECT column, AGG_FUNC(column2)
FROM table_name
GROUP BY column
HAVING AGG_FUNC(column2) > value;

4. Step-by-Step: How It Works

  1. Decide which column(s) you want to group by (e.g., Subject, City).
  2. Write a SELECT with an aggregate function (COUNT, AVG, etc.).
  3. Add GROUP BY with the column(s).
  4. Optionally use HAVING to filter groups (e.g., only show groups with more than 10 members).

5. When to Use

  • To find average marks per subject in a class.
  • To count orders per city in an e-commerce app.
  • To get total sales per month.

6. When Not to Use

  • When you want to see individual records (use plain SELECT instead).
  • When you don’t need summaries or totals.

7. Minimum to Remember

  • GROUP BY groups rows for aggregation.
  • HAVING filters groups after aggregation (like WHERE for groups).
  • Use aggregate functions (COUNT, SUM, AVG, etc.) with GROUP BY.

8. Common Mistakes

  • Forgetting to include all non-aggregated columns in GROUP BY.
  • Using WHERE instead of HAVING to filter groups.
  • Misspelling column names or using wrong aggregate functions.

9. Interviewer’s Perspective

  • “How do you get the average salary per department?”

10. Practice Questions

  1. Write SQL to get the average marks per subject from the Students table.
  2. How would you count the number of employees in each department?
  3. Write a query to show only those cities with more than 100 orders.
SELECT Subject, COUNT(*) AS TotalStudents FROM Students GROUP BY Subject;

This shows the number of students in each subject.

8️⃣ Combining GROUP BY with SUM()

Example with a Sales table:

SELECT Region, SUM(SalesAmount) AS TotalSales
FROM Sales
GROUP BY Region;

This gives total sales per region.

Now, using HAVING to show only regions where total sales are above 10000:

SELECT Region, SUM(SalesAmount) AS TotalSales
FROM Sales
GROUP BY Region
HAVING SUM(SalesAmount) > 10000;

9️⃣ Ordering Grouped Results

You can sort grouped data using ORDER BY.

SELECT Subject, AVG(Marks) AS AvgMarks
FROM Students
GROUP BY Subject
ORDER BY AvgMarks DESC;

This lists subjects by average marks in descending order (highest first).

12️⃣ Summary

  • GROUP BY groups rows that have the same values.
  • HAVING filters results after grouping.
  • Use aggregate functions like SUM(), AVG(), COUNT() with grouping.
  • ORDER BY can be used to sort grouped data.
  • WHERE filters before grouping, HAVING filters after grouping.
