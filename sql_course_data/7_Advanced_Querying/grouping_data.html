Grouping Data (GROUP BY, HAVING)

The GROUP BY clause is used to arrange identical data into groups, allowing you to perform aggregate calculations like totals, averages, or counts for each group. The HAVING clause is used to filter those grouped results — just like WHERE filters individual rows.

1️⃣ Why Use GROUP BY?

When you want to calculate summaries for specific categories (like total sales per region or average marks per subject), GROUP BY helps you organize the data by those categories.

2️⃣ Basic Syntax

SELECT column_name, AGGREGATE_FUNCTION(column_name)
FROM table_name
GROUP BY column_name;

The GROUP BY clause comes after the WHERE (if any) and before ORDER BY.

3️⃣ Example: Grouping Student Marks by Subject

SELECT Subject, AVG(Marks) AS AverageMarks
FROM Students
GROUP BY Subject;

This groups students by their subject and calculates the average marks for each one.

Example result:

SubjectAverageMarks
Math88.5
Science78.0

4️⃣ Grouping with Multiple Columns

You can group by more than one column to get more detailed summaries.

SELECT Subject, Class, AVG(Marks) AS AvgMarks
FROM Students
GROUP BY Subject, Class;

This calculates the average marks for each combination of subject and class.

5️⃣ Using GROUP BY with WHERE

Use WHERE to filter rows before grouping.

SELECT Subject, COUNT(*) AS StudentCount
FROM Students
WHERE Marks > 50
GROUP BY Subject;

This counts only students who scored more than 50 marks, grouped by subject.

6️⃣ Using HAVING to Filter Groups

The HAVING clause is used to filter the grouped results after the aggregation is done.

SELECT Subject, AVG(Marks) AS AverageMarks
FROM Students
GROUP BY Subject
HAVING AVG(Marks) > 80;

This displays only subjects with an average mark above 80.

Difference between WHERE and HAVING:

  • WHERE filters rows before grouping.
  • HAVING filters groups after aggregation.

7️⃣ Using GROUP BY with COUNT()

You can use COUNT() to find how many rows are in each group.

SELECT Subject, COUNT(*) AS TotalStudents
FROM Students
GROUP BY Subject;

This shows the number of students in each subject.

8️⃣ Combining GROUP BY with SUM()

Example with a Sales table:

SELECT Region, SUM(SalesAmount) AS TotalSales
FROM Sales
GROUP BY Region;

This gives total sales per region.

Now, using HAVING to show only regions where total sales are above 10000:

SELECT Region, SUM(SalesAmount) AS TotalSales
FROM Sales
GROUP BY Region
HAVING SUM(SalesAmount) > 10000;

9️⃣ Ordering Grouped Results

You can sort grouped data using ORDER BY.

SELECT Subject, AVG(Marks) AS AvgMarks
FROM Students
GROUP BY Subject
ORDER BY AvgMarks DESC;

This lists subjects by average marks in descending order (highest first).

11️⃣ Common Use Cases

  • Total or average salary per department
  • Total sales per product or region
  • Number of customers per city
  • Average test score per subject

12️⃣ Summary

  • GROUP BY groups rows that have the same values.
  • HAVING filters results after grouping.
  • Use aggregate functions like SUM(), AVG(), COUNT() with grouping.
  • ORDER BY can be used to sort grouped data.
  • WHERE filters before grouping, HAVING filters after grouping.
