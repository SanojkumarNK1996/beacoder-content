Set Operations in SQL: 10-Part Beginner-Friendly Guide

1. Why Does This Topic Exist?

In real-world scenarios—like merging student lists from two years, or finding customers who bought from both Amazon and Flipkart—you need to combine or compare results from multiple queries. SQL set operations make this easy.

2. What Is This?

Set operations in SQL let you combine results from two or more SELECT queries. The main ones are UNION, UNION ALL, INTERSECT, and EXCEPT (or MINUS).

3. Syntax/Structure

-- Combine results and remove duplicates
SELECT ... FROM ...
UNION
SELECT ... FROM ...;

-- Combine results and keep duplicates
SELECT ... FROM ...
UNION ALL
SELECT ... FROM ...;

-- Find common rows
SELECT ... FROM ...
INTERSECT
SELECT ... FROM ...;

-- Find rows in first query but not in second
SELECT ... FROM ...
EXCEPT
SELECT ... FROM ...;

4. Step-by-Step: How It Works

  1. Write two or more SELECT queries with the same number and type of columns.
  2. Combine them using UNION, UNION ALL, INTERSECT, or EXCEPT.
  3. Run the query to get the combined or compared results.

5. When to Use

  • To merge lists from different years or sources (e.g., students from 2024 and 2025).
  • To find customers who appear in both or only one of two lists.
  • To compare results from different queries.

6. When Not to Use

  • When queries return different numbers or types of columns.
  • When you only need data from one table (use SELECT or JOIN instead).

7. Minimum to Remember

  • UNION: Combines and removes duplicates.
  • UNION ALL: Combines and keeps duplicates.
  • INTERSECT: Only common rows.
  • EXCEPT: Rows in first query, not in second.

8. Common Mistakes

  • Different column counts or types in queries (causes errors).
  • Assuming UNION keeps duplicates (it removes them).
  • Using set operations with queries in different column order.

9. Interviewer’s Perspective

  • “What is the difference between UNION and UNION ALL?”
  • “How do you find students who are in both 2024 and 2025 batches?”
  • “How do you get all unique products sold in two different years?”

10. Practice Questions

  1. Write SQL to combine two lists of employees, removing duplicates.
  2. How would you find customers who bought in both 2024 and 2025?
  3. Write a query to get all orders from 2025 that were not present in 2024.

The INTERSECT operator returns only the rows that exist in both queries.

SELECT Name, Course FROM Students2024
INTERSECT
SELECT Name, Course FROM Students2025;

Result:

NameCourse
MaryScience
AlexEnglish

Note: Only rows common to both tables are returned.

6️⃣ EXCEPT (or MINUS)

The EXCEPT (or MINUS in Oracle) operator returns rows from the first query that do not appear in the second query.

SELECT Name, Course FROM Students2024
EXCEPT
SELECT Name, Course FROM Students2025;

Result:

NameCourse
JohnMath

Only John’s record appears because it exists only in Students2024.

7️⃣ Rules for Set Operations

  • All SELECT queries must have the same number of columns.
  • Data types in corresponding columns must be compatible.
  • Column names in the final result come from the first query.
  • You can use ORDER BY at the very end of the last query.

Example:

SELECT Name, Course FROM Students2024
UNION
SELECT Name, Course FROM Students2025
ORDER BY Name;

8️⃣ Real-World Examples

  • UNION — combining current and archived data (e.g., sales_2024 and sales_2025)
  • INTERSECT — finding users present in both “subscribers” and “customers” lists
  • EXCEPT — finding users who unsubscribed or were removed
  • UNION ALL — merging logs or transactions from multiple systems without filtering duplicates

9️⃣ Combining Multiple Operations

You can combine more than two queries in sequence:

SELECT Name, Course FROM Students2024
UNION
SELECT Name, Course FROM Students2025
EXCEPT
SELECT Name, Course FROM ExpelledStudents;

This combines students from both years but removes those who are in the expelled list.

🔟 Performance Tips

  • UNION ALL is faster than UNION (no duplicate removal).
  • Use INTERSECT and EXCEPT carefully on large datasets — they may require sorting or hashing.
  • Always use ORDER BY only at the end of all set operations.

12️⃣ Summary

  • UNION → Combines results, removes duplicates.
  • UNION ALL → Combines results, keeps duplicates.
  • INTERSECT → Returns only common rows.
  • EXCEPT → Returns rows in the first query but not in the second.
  • All queries must have the same number of columns and compatible types.

Set operations are powerful for comparing or merging datasets across multiple sources. They’re often used in analytics, reporting, and data migration tasks.
