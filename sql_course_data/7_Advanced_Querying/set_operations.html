Set Operations in SQL (UNION, UNION ALL, INTERSECT, EXCEPT)

SQL set operations combine the results of two or more SELECT queries into a single result set. They work just like mathematical sets — allowing you to find the union, intersection, or difference between query results.

Each query in a set operation must:

  • Return the same number of columns
  • Have compatible data types
  • Return columns in the same order

1️⃣ What Are SQL Set Operations?

SQL supports these primary set operations:

  • UNION — combines results and removes duplicates
  • UNION ALL — combines results including duplicates
  • INTERSECT — returns only common rows
  • EXCEPT (or MINUS in some databases) — returns rows from the first query that don’t appear in the second

2️⃣ Example Tables

Students2024 table:

NameCourse
JohnMath
MaryScience
AlexEnglish

Students2025 table:

NameCourse
MaryScience
AlexEnglish
LindaHistory

3️⃣ UNION

The UNION operator combines the results of two queries and removes duplicates.

SELECT Name, Course FROM Students2024
UNION
SELECT Name, Course FROM Students2025;

Result:

NameCourse
JohnMath
MaryScience
AlexEnglish
LindaHistory

Note: Duplicates (like Mary and Alex) appear only once.

4️⃣ UNION ALL

The UNION ALL operator combines results but keeps duplicates.

SELECT Name, Course FROM Students2024
UNION ALL
SELECT Name, Course FROM Students2025;

Result:

NameCourse
JohnMath
MaryScience
AlexEnglish
MaryScience
AlexEnglish
LindaHistory

Difference: UNION ALL is faster than UNION because it skips duplicate checks.

5️⃣ INTERSECT

The INTERSECT operator returns only the rows that exist in both queries.

SELECT Name, Course FROM Students2024
INTERSECT
SELECT Name, Course FROM Students2025;

Result:

NameCourse
MaryScience
AlexEnglish

Note: Only rows common to both tables are returned.

6️⃣ EXCEPT (or MINUS)

The EXCEPT (or MINUS in Oracle) operator returns rows from the first query that do not appear in the second query.

SELECT Name, Course FROM Students2024
EXCEPT
SELECT Name, Course FROM Students2025;

Result:

NameCourse
JohnMath

Only John’s record appears because it exists only in Students2024.

7️⃣ Rules for Set Operations

  • All SELECT queries must have the same number of columns.
  • Data types in corresponding columns must be compatible.
  • Column names in the final result come from the first query.
  • You can use ORDER BY at the very end of the last query.

Example:

SELECT Name, Course FROM Students2024
UNION
SELECT Name, Course FROM Students2025
ORDER BY Name;

8️⃣ Real-World Examples

  • UNION — combining current and archived data (e.g., sales_2024 and sales_2025)
  • INTERSECT — finding users present in both “subscribers” and “customers” lists
  • EXCEPT — finding users who unsubscribed or were removed
  • UNION ALL — merging logs or transactions from multiple systems without filtering duplicates

9️⃣ Combining Multiple Operations

You can combine more than two queries in sequence:

SELECT Name, Course FROM Students2024
UNION
SELECT Name, Course FROM Students2025
EXCEPT
SELECT Name, Course FROM ExpelledStudents;

This combines students from both years but removes those who are in the expelled list.

🔟 Performance Tips

  • UNION ALL is faster than UNION (no duplicate removal).
  • Use INTERSECT and EXCEPT carefully on large datasets — they may require sorting or hashing.
  • Always use ORDER BY only at the end of all set operations.

12️⃣ Summary

  • UNION → Combines results, removes duplicates.
  • UNION ALL → Combines results, keeps duplicates.
  • INTERSECT → Returns only common rows.
  • EXCEPT → Returns rows in the first query but not in the second.
  • All queries must have the same number of columns and compatible types.

Set operations are powerful for comparing or merging datasets across multiple sources. They’re often used in analytics, reporting, and data migration tasks.
