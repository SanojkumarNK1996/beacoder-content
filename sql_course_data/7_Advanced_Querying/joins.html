Joins in SQL: 10-Part Beginner-Friendly Guide

1. Why Does This Topic Exist?

In Indian businesses, schools, or apps, data is split across tables (students, courses, orders, products). Joins let you combine related data for a complete picture—like showing which student is in which course.

2. What Is This?

A JOIN is an SQL operation that combines rows from two or more tables based on a related column (usually a key).

3. Syntax/Structure

-- Basic join
SELECT columns
FROM table1
JOIN table2 ON table1.col = table2.col;

-- Types: INNER, LEFT, RIGHT, FULL OUTER, CROSS

4. Step-by-Step: How It Works

  1. Identify the tables you want to combine (e.g., Students and Courses).
  2. Decide which columns link them (e.g., CourseID).
  3. Choose the type of join (INNER, LEFT, etc.).
  4. Write the SELECT with JOIN and ON condition.
  5. Run the query to see combined data.

5. When to Use

  • To show all students and their courses.
  • To list orders with customer details.
  • To combine any related data from different tables.

6. When Not to Use

  • When all data is in a single table.
  • When you don’t need combined information.

7. Minimum to Remember

  • INNER JOIN: Only matching rows in both tables.
  • LEFT JOIN: All rows from left, matched from right (NULL if no match).
  • RIGHT JOIN: All rows from right, matched from left (NULL if no match).
  • FULL OUTER JOIN: All rows from both tables (NULL where no match).
  • CROSS JOIN: All possible combinations (rarely used).

8. Common Mistakes

  • Forgetting the ON condition (causes huge, meaningless results).
  • Confusing LEFT and RIGHT joins.
  • Not handling NULLs in results.

9. Interviewer’s Perspective

  • “What’s the difference between INNER JOIN and LEFT JOIN?”
  • “How do you find all customers who haven’t placed any orders?”
  • “Write SQL to list all products and their suppliers, even if some products have no supplier.”

10. Practice Questions

  1. Write SQL to list all students and their course names (even if some students are not enrolled in any course).
  2. How would you find all courses that have no students enrolled?
  3. What is the difference between INNER JOIN and LEFT JOIN?
