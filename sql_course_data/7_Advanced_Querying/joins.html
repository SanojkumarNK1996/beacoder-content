Joins in SQL (INNER JOIN, LEFT JOIN, RIGHT JOIN, FULL OUTER JOIN, CROSS JOIN)

In a database, data is often spread across multiple tables to avoid duplication. A JOIN combines data from two or more tables based on a related column between them — usually a key.

1️⃣ Why Do We Need Joins?

Instead of storing everything in a single table, we keep related data in different tables (for example, Students and Courses). When you need a complete view — like which student is enrolled in which course — you use a JOIN to combine them.

2️⃣ Basic JOIN Syntax

SELECT columns
FROM table1
JOIN table2
ON table1.column_name = table2.column_name;

The ON condition tells SQL how the tables are related.

4️⃣ INNER JOIN

The INNER JOIN returns only the rows where there is a match in both tables.

SELECT Students.Name, Courses.CourseName
FROM Students
INNER JOIN Courses
ON Students.CourseID = Courses.CourseID;

Result:

NameCourseName
JohnMath
MaryScience

Explanation: Alex’s CourseID (103) doesn’t match any in the Courses table, so his record is not included.

5️⃣ LEFT JOIN

The LEFT JOIN returns all rows from the left table (Students), and the matched rows from the right table (Courses). If there’s no match, you’ll see NULL values for columns from the right table.

SELECT Students.Name, Courses.CourseName
FROM Students
LEFT JOIN Courses
ON Students.CourseID = Courses.CourseID;

Result:

NameCourseName
JohnMath
MaryScience
AlexNULL

Alex’s course is missing, but his data is still shown.

6️⃣ RIGHT JOIN

The RIGHT JOIN returns all rows from the right table (Courses), and matched rows from the left table (Students).

SELECT Students.Name, Courses.CourseName
FROM Students
RIGHT JOIN Courses
ON Students.CourseID = Courses.CourseID;

Result:

NameCourseName
JohnMath
MaryScience
NULLEnglish

Explanation: The “English” course has no student enrolled, so Name is NULL.

7️⃣ FULL OUTER JOIN

The FULL OUTER JOIN combines the results of both LEFT and RIGHT joins. It returns all records from both tables, with NULL where there is no match.

SELECT Students.Name, Courses.CourseName
FROM Students
FULL OUTER JOIN Courses
ON Students.CourseID = Courses.CourseID;

Result:

NameCourseName
JohnMath
MaryScience
AlexNULL
NULLEnglish

Note: Not all databases support FULL OUTER JOIN directly (for example, MySQL requires a workaround using UNION).

8️⃣ CROSS JOIN

The CROSS JOIN returns all possible combinations of rows from both tables. This is also known as the Cartesian Product.

SELECT Students.Name, Courses.CourseName
FROM Students
CROSS JOIN Courses;

Result: Every student appears with every course.

NameCourseName
JohnMath
JohnScience
JohnEnglish
MaryMath
MaryScience
MaryEnglish
AlexMath
AlexScience
AlexEnglish

Use CROSS JOIN carefully — it can generate a very large number of rows.

9️⃣ Visual Summary of Joins

  • INNER JOIN → Only matching rows from both tables.
  • LEFT JOIN → All rows from left, matched rows from right.
  • RIGHT JOIN → All rows from right, matched rows from left.
  • FULL OUTER JOIN → All rows from both tables.
  • CROSS JOIN → All possible combinations (Cartesian Product).

🔟 Example Use Cases

  • Finding students with their enrolled courses (INNER JOIN)
  • Finding all students, including those not enrolled yet (LEFT JOIN)
  • Finding all courses, including those with no students (RIGHT JOIN)
  • Combining both studentless courses and unenrolled students (FULL JOIN)

11️⃣ Best Practices

  • Always specify the join condition (ON clause) clearly.
  • Use table aliases (e.g., S for Students, C for Courses) to improve readability.
  • Limit results with WHERE or LIMIT when working with large datasets.
  • Understand the relationship between tables (primary and foreign keys) before joining.

12️⃣ Summary

  • Joins combine related data from multiple tables.
  • Different join types control what data is included.
  • INNER JOIN returns only matches, LEFT/RIGHT JOINs include unmatched rows, FULL JOIN returns all, and CROSS JOIN makes combinations.
  • Understanding joins is key to advanced querying and report generation.
