Subqueries in SQL (Single-row and Multi-row)

A subquery is a query nested inside another SQL query. It allows you to use the result of one query as input for another. Subqueries help in filtering, comparing, or calculating values dynamically based on other data in the database.

1️⃣ What is a Subquery?

A subquery (also called an inner query) is enclosed in parentheses and used inside another SQL query (called the outer query).

Example:

SELECT name
FROM Students
WHERE marks > (
    SELECT AVG(marks)
    FROM Students
);

This finds all students who scored higher than the class average. Here, the inner query calculates the average marks, and the outer query uses that value.

2️⃣ Types of Subqueries

  • Single-row Subquery — returns only one value (one row, one column).
  • Multi-row Subquery — returns multiple values (one column, many rows).
  • Multi-column Subquery — returns multiple columns (used rarely).

3️⃣ Single-row Subquery

A single-row subquery returns exactly one value and is often used with comparison operators like =, >, <, >=, or <=.

Example:

SELECT name, marks
FROM Students
WHERE marks > (
    SELECT AVG(marks)
    FROM Students
);

This query compares each student’s marks to a single average value returned by the subquery.

4️⃣ Example: Comparing Salaries

SELECT name, salary
FROM Employees
WHERE salary > (
    SELECT salary
    FROM Employees
    WHERE name = 'John'
);

This shows all employees who earn more than John. Here, the subquery returns one row — John’s salary.

5️⃣ Multi-row Subquery

A multi-row subquery returns multiple rows, and is used with operators like:

  • IN — matches any of the values
  • ANY or SOME — compares to any value
  • ALL — compares to all returned values

6️⃣ Example with IN

SELECT name
FROM Students
WHERE course_id IN (
    SELECT course_id
    FROM Courses
    WHERE instructor = 'Dr. Smith'
);

This finds all students who are enrolled in courses taught by Dr. Smith.

7️⃣ Example with ANY

SELECT name, salary
FROM Employees
WHERE salary > ANY (
    SELECT salary
    FROM Employees
    WHERE department = 'Sales'
);

This returns employees whose salary is greater than at least one employee in the Sales department.

8️⃣ Example with ALL

SELECT name, salary
FROM Employees
WHERE salary > ALL (
    SELECT salary
    FROM Employees
    WHERE department = 'Sales'
);

This shows employees whose salary is higher than all employees in the Sales department (i.e., higher than the maximum salary in Sales).

9️⃣ Subquery in FROM Clause

You can also use a subquery as a temporary table inside the FROM clause. This is called a derived table.

Example:

SELECT department, AVG(avg_salary) AS overall_average
FROM (
    SELECT department, AVG(salary) AS avg_salary
    FROM Employees
    GROUP BY department
) AS dept_summary
GROUP BY department;

Here, the inner query calculates the average salary per department, and the outer query performs another aggregation on that result.

🔟 Subquery in SELECT Clause

Subqueries can also appear inside the SELECT list — for example, to show calculated values for each row.

SELECT name,
       salary,
       (SELECT AVG(salary) FROM Employees) AS avg_salary
FROM Employees;

This shows each employee’s salary along with the overall average salary.

11️⃣ Subquery in WHERE Clause (Most Common)

The most common use of a subquery is inside the WHERE clause to filter rows based on another query’s result.

SELECT name
FROM Students
WHERE course_id IN (
    SELECT id
    FROM Courses
    WHERE level = 'Advanced'
);

12️⃣ Correlated Subqueries

A correlated subquery depends on the outer query for its values — it is executed once for each row of the outer query.

Example:

SELECT name, marks
FROM Students s
WHERE marks > (
    SELECT AVG(marks)
    FROM Students
    WHERE course_id = s.course_id
);

This finds students who scored above the average in their specific course.

13️⃣ Performance Tips

  • Use EXISTS instead of IN for better performance with large datasets.
  • Where possible, convert subqueries to JOIN for faster execution.
  • Always test subqueries separately to ensure they return expected results.

14️⃣ Example Using EXISTS

SELECT name
FROM Students s
WHERE EXISTS (
    SELECT 1
    FROM Courses c
    WHERE c.course_id = s.course_id
    AND c.level = 'Beginner'
);

This returns students enrolled in beginner-level courses.

15️⃣ Summary

  • Subqueries allow you to nest queries for dynamic filtering and calculations.
  • Single-row subqueries return one value (use =, >, <, etc.).
  • Multi-row subqueries return multiple values (use IN, ANY, ALL).
  • Correlated subqueries depend on the outer query.
  • EXISTS and IN are commonly used for existence checks.

Subqueries make SQL more powerful by enabling conditional logic and data comparisons that would be difficult to express otherwise. Next, we’ll explore Set Operations — how to combine results from multiple queries.
