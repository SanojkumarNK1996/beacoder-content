Subqueries in SQL: 10-Part Beginner-Friendly Guide

1. Why Does This Topic Exist?

Sometimes you need to answer questions like “Who scored above average?” or “Which products cost more than the cheapest item in another category?” Subqueries let you use the result of one query inside another, making SQL much more powerful.

2. What Is This?

A subquery is a query inside another SQL query. It’s used to filter, compare, or calculate values based on other data in the database.

3. Syntax/Structure

-- Basic subquery
SELECT ...
FROM ...
WHERE column OPERATOR (
    SELECT ... FROM ... WHERE ...
);

4. Step-by-Step: How It Works

  1. Write the inner query (subquery) that gets the value(s) you need.
  2. Use the subquery inside the main (outer) query, usually in WHERE or HAVING.
  3. Run the combined query to get your answer.

5. When to Use

  • To compare each row to a calculated value (like average marks).
  • To filter results based on another table’s data.
  • To answer “top N” or “exists in another set” questions.

6. When Not to Use

  • When a simple JOIN or WHERE is enough.
  • When performance is critical and the subquery is slow (consider JOINs or CTEs).

7. Minimum to Remember

  • Subqueries go inside parentheses.
  • Single-row subqueries return one value (use =, >, <, etc.).
  • Multi-row subqueries return many values (use IN, ANY, ALL).

8. Common Mistakes

  • Using = with a multi-row subquery (causes errors—use IN instead).
  • Forgetting parentheses around the subquery.
  • Not checking if the subquery returns the expected number of rows.

9. Interviewer’s Perspective

  • “How do you find students who scored above the class average?”
  • “What’s the difference between a single-row and multi-row subquery?”
  • “How do you use a subquery to filter results from another table?”

10. Practice Questions

  1. Write SQL to find all employees who earn more than the average salary.
  2. How would you list products that cost more than the cheapest item in another category?
  3. Write a query to find students enrolled in courses taught by “Dr. Sharma” using a subquery.
SELECT name, salary
FROM Employees
WHERE salary > ANY (
    SELECT salary
    FROM Employees
    WHERE department = 'Sales'
);

This returns employees whose salary is greater than at least one employee in the Sales department.

8️⃣ Example with ALL

SELECT name, salary
FROM Employees
WHERE salary > ALL (
    SELECT salary
    FROM Employees
    WHERE department = 'Sales'
);

This shows employees whose salary is higher than all employees in the Sales department (i.e., higher than the maximum salary in Sales).

9️⃣ Subquery in FROM Clause

You can also use a subquery as a temporary table inside the FROM clause. This is called a derived table.

Example:

SELECT department, AVG(avg_salary) AS overall_average
FROM (
    SELECT department, AVG(salary) AS avg_salary
    FROM Employees
    GROUP BY department
) AS dept_summary
GROUP BY department;

Here, the inner query calculates the average salary per department, and the outer query performs another aggregation on that result.

🔟 Subquery in SELECT Clause

Subqueries can also appear inside the SELECT list — for example, to show calculated values for each row.

SELECT name,
       salary,
       (SELECT AVG(salary) FROM Employees) AS avg_salary
FROM Employees;

This shows each employee’s salary along with the overall average salary.

11️⃣ Subquery in WHERE Clause (Most Common)

The most common use of a subquery is inside the WHERE clause to filter rows based on another query’s result.

SELECT name
FROM Students
WHERE course_id IN (
    SELECT id
    FROM Courses
    WHERE level = 'Advanced'
);

12️⃣ Correlated Subqueries

A correlated subquery depends on the outer query for its values — it is executed once for each row of the outer query.

Example:

SELECT name, marks
FROM Students s
WHERE marks > (
    SELECT AVG(marks)
    FROM Students
    WHERE course_id = s.course_id
);

This finds students who scored above the average in their specific course.

13️⃣ Performance Tips

  • Use EXISTS instead of IN for better performance with large datasets.
  • Where possible, convert subqueries to JOIN for faster execution.
  • Always test subqueries separately to ensure they return expected results.

14️⃣ Example Using EXISTS

SELECT name
FROM Students s
WHERE EXISTS (
    SELECT 1
    FROM Courses c
    WHERE c.course_id = s.course_id
    AND c.level = 'Beginner'
);

This returns students enrolled in beginner-level courses.

15️⃣ Summary

  • Subqueries allow you to nest queries for dynamic filtering and calculations.
  • Single-row subqueries return one value (use =, >, <, etc.).
  • Multi-row subqueries return multiple values (use IN, ANY, ALL).
  • Correlated subqueries depend on the outer query.
  • EXISTS and IN are commonly used for existence checks.
