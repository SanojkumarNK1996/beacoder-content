Deleting Data (DELETE)

The DELETE statement is used to remove rows from a table in a database. It’s a powerful command, so it must be used carefully — once data is deleted, it usually can’t be recovered unless you have a backup.

1️⃣ Basic DELETE Syntax

DELETE FROM table_name
WHERE condition;

This command deletes one or more rows from the table that match the given condition.

2️⃣ Example: Deleting a Specific Record

DELETE FROM Students
WHERE StudentID = 101;

This removes the record where the StudentID is 101.

3️⃣ Important: Always Use WHERE!

If you forget the WHERE clause, all rows in the table will be deleted.

DELETE FROM Students;

⚠️ This command removes every record in the Students table. Always double-check before running a DELETE command without a condition.

4️⃣ Deleting Multiple Rows with a Condition

DELETE FROM Students
WHERE Age < 18;

This deletes all students who are younger than 18 years old.

5️⃣ Using Logical Conditions

You can combine multiple conditions with AND or OR.

DELETE FROM Students
WHERE Grade = 'F' OR Attendance < 60;

This deletes students who either failed (Grade = 'F') or have attendance below 60%.

6️⃣ Deleting All Data (Truncating a Table)

If you want to remove all rows but keep the table structure, use TRUNCATE instead of DELETE.

TRUNCATE TABLE Students;

This quickly removes all records from the table and resets any auto-increment values. However, unlike DELETE, you cannot use WHERE with TRUNCATE.

7️⃣ Viewing Remaining Data After Deletion

After deleting records, you can verify the remaining data:

SELECT * FROM Students;

This shows what’s left in the table after deletion.

8️⃣ Example Scenario

Imagine your Students table looks like this before deletion:

StudentIDNameAgeGrade
101John19A
102Mary18B
103Alex17F

Now run this command:

DELETE FROM Students
WHERE Grade = 'F';

After this, the table will look like:

StudentIDNameAgeGrade
101John19A
102Mary18B

9️⃣ Deleting Data from Related Tables

When using foreign keys, deleting data from one table may affect related tables.

Example: If you delete a student who has records in another table (like Grades), you may get an error due to foreign key constraints.

To handle this, databases often use ON DELETE CASCADE in relationships — this means that deleting a record in one table automatically deletes related records in another.

CREATE TABLE Grades (
  GradeID INT PRIMARY KEY,
  StudentID INT,
  FOREIGN KEY (StudentID) REFERENCES Students(StudentID) ON DELETE CASCADE
);

With this setup, if a student is deleted from the Students table, their grades will be deleted too.

🔟 Summary

  • DELETE FROM table WHERE condition; — deletes specific rows.
  • Always include WHERE to avoid removing everything.
  • TRUNCATE TABLE — deletes all rows quickly (no WHERE allowed).
  • Use SELECT to verify changes after deletion.
  • Be aware of relationships and foreign key constraints.

The DELETE command helps you clean up or manage data safely when used carefully. With this, you’ve learned all the basic Data Manipulation Language (DML) operations — INSERT, SELECT, UPDATE, and DELETE.
