Inserting Data (INSERT)

Now that you’ve learned how to create tables, it’s time to fill them with data! In SQL, the INSERT statement is used to add new records (rows) into a table.

1️⃣ What Is the INSERT Statement?

The INSERT command adds one or more new rows into a database table.

General syntax:

INSERT INTO table_name (column1, column2, column3, ...)
VALUES (value1, value2, value3, ...);

Example:

INSERT INTO Students (StudentID, Name, Age)
VALUES (1, 'Alice', 18);

This adds a new row to the Students table with the given values.

2️⃣ Inserting Data into All Columns

If you’re inserting values for all columns in the table, you can skip the column names (but the order must match the table definition):

INSERT INTO Students
VALUES (2, 'Bob', 20);

✅ Tip: Always include column names for clarity, especially in large tables.

3️⃣ Inserting Multiple Rows at Once

You can insert multiple records in a single statement to save time:

INSERT INTO Students (StudentID, Name, Age)
VALUES 
(3, 'Charlie', 19),
(4, 'Diana', 21),
(5, 'Ethan', 22);

This adds three new students in one go!

4️⃣ Handling DEFAULT Values

If a column has a DEFAULT value defined, you can omit it during insertion — it will automatically take the default.

CREATE TABLE Orders (
  OrderID INT PRIMARY KEY,
  Status VARCHAR(20) DEFAULT 'Pending'
);

INSERT INTO Orders (OrderID) VALUES (1);

The value for Status will automatically be set to 'Pending'.

5️⃣ Inserting NULL Values

If a column allows NULL, you can insert an empty value.

INSERT INTO Students (StudentID, Name, Age)
VALUES (6, 'Fiona', NULL);

Here, the Age value will be empty (NULL).

6️⃣ Common Mistakes to Avoid

  • ❌ Mismatching column count — number of values must match number of columns.
  • ❌ Violating constraints (e.g., inserting duplicate PRIMARY KEY).
  • ❌ Inserting wrong data type (e.g., text in numeric columns).

7️⃣ Using INSERT with SELECT

You can insert data from another table using the SELECT statement.

INSERT INTO Graduates (ID, Name)
SELECT StudentID, Name FROM Students WHERE Age > 20;

This copies all students older than 20 into the Graduates table.

8️⃣ Viewing Inserted Data

After inserting, use the SELECT command to check what’s inside your table:

SELECT * FROM Students;

This displays all rows in the Students table.

9️⃣ Example: Putting It All Together

CREATE TABLE Students (
  StudentID INT PRIMARY KEY,
  Name VARCHAR(50) NOT NULL,
  Age INT
);

INSERT INTO Students (StudentID, Name, Age)
VALUES 
(1, 'Alice', 18),
(2, 'Bob', 20),
(3, 'Charlie', 19);

SELECT * FROM Students;

Output:

+-----------+---------+-----+
| StudentID | Name    | Age |
+-----------+---------+-----+
| 1         | Alice   | 18  |
| 2         | Bob     | 20  |
| 3         | Charlie | 19  |
+-----------+---------+-----+

10️⃣ Summary

  • INSERT INTO ... VALUES (...); — adds new rows to a table.
  • Include column names for clarity.
  • Can insert multiple rows in one command.
  • Can use DEFAULT and NULL where applicable.
  • Use SELECT to verify inserted data.
