Reading Data (SELECT): 10-Part Beginner-Friendly Guide

1. Why Does This Topic Exist?

Every app or business—like Flipkart, SBI, or your college—needs to fetch and display data (orders, balances, marksheets). The SELECT command is how you read data from your database.

2. What Is This?

SELECT is an SQL command used to retrieve (read) data from one or more tables. It’s the most used SQL command for reports, dashboards, and analytics.

3. Syntax/Structure

-- Select specific columns
SELECT column1, column2 FROM table_name;

-- Select all columns
SELECT * FROM table_name;

-- Filter rows
SELECT * FROM table_name WHERE condition;

-- Sort results
SELECT * FROM table_name ORDER BY column1 DESC;

4. Step-by-Step: How It Works

  1. Decide what data you want to see (all columns or specific ones).
  2. Write the SELECT statement with the right columns and table name.
  3. Add WHERE to filter, ORDER BY to sort, or AS to rename columns as needed.
  4. Run the command and view the results.

5. When to Use

  • When you want to display data to users (e.g., show all students in a class).
  • When you need to analyze or report on data.

6. When Not to Use

  • When you want to add new data (use INSERT instead).
  • When you want to change or remove data (use UPDATE or DELETE instead).

7. Minimum to Remember

  • SELECT * FROM table_name; gets all data from a table.
  • Use WHERE to filter, ORDER BY to sort.
  • Use AS to rename columns in the result.

8. Common Mistakes

  • Misspelling table or column names.
  • Forgetting WHERE (can return too much data).
  • Using * when you only need a few columns (slows down big databases).

9. Interviewer’s Perspective

  • “How do you select only the names of all students?”
  • “What does SELECT * FROM table do?”
  • “How do you sort results in descending order?”

10. Practice Questions

  1. Write SQL to select all columns from the Employees table.
  2. How would you get only the names and ages of students from the Students table?
  3. Write a query to get all orders placed after 1st Jan 2026, sorted by amount (highest first).
SELECT * FROM Students ORDER BY Age;

This sorts students by Age in ascending order (smallest to largest). To sort in descending order, use DESC:

SELECT * FROM Students
ORDER BY Age DESC;

You can also sort by multiple columns:

SELECT * FROM Students
ORDER BY Age DESC, Name ASC;

7️⃣ Limiting Results (LIMIT and OFFSET)

When you have a large dataset, you might want to view only a small number of rows.

SELECT * FROM Students
LIMIT 5;

This retrieves only the first 5 rows.

To skip a few rows and then fetch the next few, use OFFSET:

SELECT * FROM Students
LIMIT 5 OFFSET 5;

This skips the first 5 records and then shows the next 5.

8️⃣ Filtering Unique Values (DISTINCT)

To get only unique (non-duplicate) values, use DISTINCT.

SELECT DISTINCT Age FROM Students;

This shows all unique age values from the table.

9️⃣ Example: Combining SELECT Features

Let’s combine everything in one example:

SELECT DISTINCT Name, Age
FROM Students
WHERE Age BETWEEN 18 AND 25
ORDER BY Age DESC
LIMIT 3;

Explanation:

  • Shows unique student names and ages.
  • Only includes students aged 18–25.
  • Sorts them from oldest to youngest.
  • Shows only the top 3 results.

10️⃣ Using SELECT with Calculations

You can perform simple calculations or combine columns directly in SELECT.

SELECT Name, Age + 1 AS "Age Next Year"
FROM Students;

This shows each student’s name and what their age will be next year.

11️⃣ Summary

  • SELECT * FROM table; — view all data.
  • SELECT column1, column2; — view specific columns.
  • WHERE — filter data based on conditions.
  • ORDER BY — sort results.
  • LIMIT and OFFSET — control how many rows are shown.
  • DISTINCT — remove duplicates.

The SELECT statement is the heart of SQL — it allows you to view, analyze, and make sense of your data. Next, we’ll learn how to update existing records in a table using the UPDATE command.
