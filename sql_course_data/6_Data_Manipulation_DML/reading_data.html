Reading Data (SELECT)

The SELECT statement is used to retrieve data from one or more tables in a database. It’s the most commonly used SQL command and the foundation of all data analysis and reporting.

1️⃣ Basic SELECT Syntax

SELECT column1, column2, ...
FROM table_name;

This command fetches specific columns from a table. To select all columns, use an asterisk (*).

SELECT * FROM Students;

This retrieves all columns and rows from the Students table.

2️⃣ Selecting Specific Columns

If you only need certain columns, specify them explicitly:

SELECT Name, Age FROM Students;

This will only show the Name and Age columns for all students.

3️⃣ Renaming Columns (Using AS)

You can rename a column in the result using the AS keyword — this is called an alias.

SELECT Name AS StudentName, Age AS StudentAge
FROM Students;

This doesn’t change the actual table — it just changes the display name in your result.

4️⃣ Filtering Results with WHERE

The WHERE clause is used to filter rows that meet specific conditions.

SELECT * FROM Students
WHERE Age > 18;

This shows only students older than 18.

You can use various operators inside WHERE:

  • = — equal to
  • > — greater than
  • < — less than
  • >=, <= — greater or less than or equal to
  • != or <> — not equal
  • LIKE — for pattern matching (e.g., names starting with “A”)
  • IN — check if a value exists in a list
  • BETWEEN — check if a value lies between two numbers
SELECT * FROM Students
WHERE Name LIKE 'A%';

This returns all students whose names start with “A”.

5️⃣ Combining Conditions with AND / OR

You can use AND and OR to combine multiple conditions.

SELECT * FROM Students
WHERE Age > 18 AND Name LIKE 'A%';

This shows students older than 18 whose names start with “A”.

6️⃣ Sorting Data with ORDER BY

To sort results, use ORDER BY.

SELECT * FROM Students
ORDER BY Age;

This sorts students by Age in ascending order (smallest to largest). To sort in descending order, use DESC:

SELECT * FROM Students
ORDER BY Age DESC;

You can also sort by multiple columns:

SELECT * FROM Students
ORDER BY Age DESC, Name ASC;

7️⃣ Limiting Results (LIMIT and OFFSET)

When you have a large dataset, you might want to view only a small number of rows.

SELECT * FROM Students
LIMIT 5;

This retrieves only the first 5 rows.

To skip a few rows and then fetch the next few, use OFFSET:

SELECT * FROM Students
LIMIT 5 OFFSET 5;

This skips the first 5 records and then shows the next 5.

8️⃣ Filtering Unique Values (DISTINCT)

To get only unique (non-duplicate) values, use DISTINCT.

SELECT DISTINCT Age FROM Students;

This shows all unique age values from the table.

9️⃣ Example: Combining SELECT Features

Let’s combine everything in one example:

SELECT DISTINCT Name, Age
FROM Students
WHERE Age BETWEEN 18 AND 25
ORDER BY Age DESC
LIMIT 3;

Explanation:

  • Shows unique student names and ages.
  • Only includes students aged 18–25.
  • Sorts them from oldest to youngest.
  • Shows only the top 3 results.

10️⃣ Using SELECT with Calculations

You can perform simple calculations or combine columns directly in SELECT.

SELECT Name, Age + 1 AS "Age Next Year"
FROM Students;

This shows each student’s name and what their age will be next year.

11️⃣ Summary

  • SELECT * FROM table; — view all data.
  • SELECT column1, column2; — view specific columns.
  • WHERE — filter data based on conditions.
  • ORDER BY — sort results.
  • LIMIT and OFFSET — control how many rows are shown.
  • DISTINCT — remove duplicates.

The SELECT statement is the heart of SQL — it allows you to view, analyze, and make sense of your data. Next, we’ll learn how to update existing records in a table using the UPDATE command.
