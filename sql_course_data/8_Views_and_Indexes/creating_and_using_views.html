Creating and Using Views in SQL

1️⃣ Why Does This Topic Exist?

Views help you simplify complex queries, secure sensitive data, and present information in a user-friendly way. In Indian companies, views are often used to give different teams access to only the data they need, such as showing only sales data to the sales team.

2️⃣ What Is a View?

A view is a virtual table created by saving a SQL query. It does not store data itself, but shows results from one or more tables as if it were a table. When you use a view, SQL runs the saved query and returns the result.

3️⃣ Syntax/Structure

CREATE VIEW view_name AS
SELECT column1, column2, ...
FROM table_name
WHERE condition;

Example: Create a view for IT employees:

CREATE VIEW IT_Employees AS
SELECT Name, Dept, Salary
FROM Employees
WHERE Dept = 'IT';

4️⃣ Step-by-Step Execution

  1. Write a SELECT query that gives the data you want.
  2. Use CREATE VIEW to save that query as a view.
  3. Query the view just like a table: SELECT * FROM view_name;
  4. To update the view, use CREATE OR REPLACE VIEW or ALTER VIEW.
  5. To remove a view, use DROP VIEW view_name;

5️⃣ When to Use

  • To simplify repeated complex queries (e.g., monthly sales reports).
  • To restrict access to sensitive columns (e.g., hiding salary in HR systems).
  • To present data in a specific format for different departments (e.g., sales, accounts).

6️⃣ When Not to Use

  • When you need to update data through the view and the view is based on joins or aggregations (these are not updatable).
  • If the underlying tables change very frequently, as views may become outdated.

7️⃣ Minimum to Remember

  • Views are virtual tables based on SELECT queries.
  • They help simplify queries and secure data.
  • Use CREATE VIEW and DROP VIEW to manage them.

8️⃣ Common Mistakes

  • Trying to update data through a view that is not updatable (e.g., views with GROUP BY or JOIN).
  • Forgetting to update the view when the underlying table structure changes.
  • Assuming views store data—they do not.

DROP VIEW IT_Employees;

Deleting a view does not remove any data from the original table — only the saved query (view definition) is deleted.

6️⃣ Using Views for Data Security

Views can hide sensitive information from users. For example, if you don’t want regular users to see salaries:

CREATE VIEW PublicEmployees AS
SELECT Name, Dept, Location
FROM Employees;

Now users can query PublicEmployees safely, without accessing salary data.

7️⃣ Joining Tables in a View

You can create a view that joins multiple tables. For example:

CREATE VIEW EmployeeProjects AS
SELECT e.Name, e.Dept, p.ProjectName
FROM Employees e
JOIN Projects p ON e.ID = p.EmployeeID;

This view combines employee and project data into one simple structure.

8️⃣ Renaming a View

SQL doesn’t have a direct RENAME VIEW command in all systems, but you can do it by recreating it:

CREATE VIEW NewViewName AS
SELECT * FROM OldViewName;

DROP VIEW OldViewName;

9️⃣ Viewing Existing Views

In most databases, you can see all views by checking system tables.

  • MySQL: SHOW FULL TABLES WHERE TABLE_TYPE = 'VIEW';
  • PostgreSQL: \dv
  • SQL Server: SELECT * FROM INFORMATION_SCHEMA.VIEWS;

🔟 Summary

  • A view is a virtual table based on a SQL query.
  • It simplifies complex queries and hides sensitive data.
  • You can create, modify, and delete views easily.
  • Views can be used for security, readability, and reusability.
