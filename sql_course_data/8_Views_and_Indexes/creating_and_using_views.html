Creating and Using Views in SQL

A view in SQL is a virtual table based on the result of a query. It doesn’t store data physically — instead, it provides a way to look at data stored in one or more tables as if it were a single table. Views make it easier to read, reuse, and secure data.

1️⃣ What is a View?

A view is like a saved SQL query that you can treat as a table. When you query a view, SQL runs the query defined in that view and returns the result.

Example: Suppose we have a table named Employees:

+----+----------+----------+------------+-----------+
| ID | Name     | Dept     | Salary     | Location  |
+----+----------+----------+------------+-----------+
| 1  | Alice    | HR       | 50000      | London    |
| 2  | Bob      | IT       | 70000      | New York  |
| 3  | Charlie  | Finance  | 60000      | Toronto   |
+----+----------+----------+------------+-----------+

If you frequently want to see only employees from the IT department, you can create a view like this:

CREATE VIEW IT_Employees AS
SELECT Name, Dept, Salary
FROM Employees
WHERE Dept = 'IT';

Now, instead of writing that query again, you can simply do:

SELECT * FROM IT_Employees;

This will show all employees from the IT department, using the saved query inside the view.

2️⃣ Why Use Views?

  • To simplify complex queries — you can reuse them easily.
  • To restrict access to sensitive data (e.g., hide salary columns).
  • To improve readability — users can work with meaningful data instead of raw tables.
  • To provide abstraction — users can interact with data without knowing how it’s structured.

3️⃣ Syntax for Creating a View

CREATE VIEW view_name AS
SELECT columns
FROM table_name
WHERE condition;

Example:

CREATE VIEW HighSalary AS
SELECT Name, Dept, Salary
FROM Employees
WHERE Salary > 60000;

Now you can use it like a table:

SELECT * FROM HighSalary;

4️⃣ Updating Data Through a View

In some cases, you can also update data using a view. However, not all views are updatable — it depends on how they are created.

Example:

UPDATE IT_Employees
SET Salary = 75000
WHERE Name = 'Bob';

This will update the underlying Employees table, since the view is based on it directly.

Note: A view is not updatable if it includes joins, aggregation, or GROUP BY.

5️⃣ Modifying and Deleting Views

  • To modify a view:
CREATE OR REPLACE VIEW IT_Employees AS
SELECT Name, Dept, Salary, Location
FROM Employees
WHERE Dept = 'IT';
  • To delete a view:
DROP VIEW IT_Employees;

Deleting a view does not remove any data from the original table — only the saved query (view definition) is deleted.

6️⃣ Using Views for Data Security

Views can hide sensitive information from users. For example, if you don’t want regular users to see salaries:

CREATE VIEW PublicEmployees AS
SELECT Name, Dept, Location
FROM Employees;

Now users can query PublicEmployees safely, without accessing salary data.

7️⃣ Joining Tables in a View

You can create a view that joins multiple tables. For example:

CREATE VIEW EmployeeProjects AS
SELECT e.Name, e.Dept, p.ProjectName
FROM Employees e
JOIN Projects p ON e.ID = p.EmployeeID;

This view combines employee and project data into one simple structure.

8️⃣ Renaming a View

SQL doesn’t have a direct RENAME VIEW command in all systems, but you can do it by recreating it:

CREATE VIEW NewViewName AS
SELECT * FROM OldViewName;

DROP VIEW OldViewName;

9️⃣ Viewing Existing Views

In most databases, you can see all views by checking system tables.

  • MySQL: SHOW FULL TABLES WHERE TABLE_TYPE = 'VIEW';
  • PostgreSQL: \dv
  • SQL Server: SELECT * FROM INFORMATION_SCHEMA.VIEWS;

🔟 Summary

  • A view is a virtual table based on a SQL query.
  • It simplifies complex queries and hides sensitive data.
  • You can create, modify, and delete views easily.
  • Views can be used for security, readability, and reusability.
