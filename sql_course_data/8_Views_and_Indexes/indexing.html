Indexing in Databases

When databases grow large, searching for specific information can become slow. To make searches faster, databases use something called an index — a special data structure that helps the system find data quickly without scanning every row.

1️⃣ What is an Index?

An index is like the index section in a book — it helps you jump directly to the page you need instead of reading the whole book.

In databases, an index works the same way. It allows the database engine to find the required rows faster.

Example: Suppose you have a table named Students:

+----+----------+----------+
| ID | Name     | City     |
+----+----------+----------+
| 1  | Alice    | London   |
| 2  | Bob      | Paris    |
| 3  | Charlie  | New York |
| 4  | David    | Paris    |
+----+----------+----------+

If you frequently search by City — e.g. SELECT * FROM Students WHERE City = 'Paris'; — creating an index on City will make those searches much faster.

CREATE INDEX idx_city ON Students(City);

2️⃣ How Indexes Work (In Simple Terms)

Indexes are usually implemented as special data structures such as B-trees or hash maps. They store key values (like “City”) in sorted order with pointers to the actual rows in the table.

So instead of scanning every row, the database looks through the sorted index — like using a dictionary — and jumps directly to the matching records.

3️⃣ Benefits of Indexing

  • ✅ Faster searches — quickly locate rows without scanning the entire table.
  • ✅ Better sorting and filtering — queries with ORDER BY and WHERE run faster.
  • ✅ Improved join performance — indexes help speed up joins between related tables.

4️⃣ Drawbacks of Indexing

  • ⚠️ Slower inserts and updates — when data changes, the index must also be updated.
  • ⚠️ Extra storage space — indexes take up additional disk space.
  • ⚠️ Too many indexes can slow down data modification operations.

Tip: Only create indexes on columns that are frequently used for searching or sorting.

5️⃣ Types of Indexes

👉 Primary Index

A primary index is automatically created when you define a PRIMARY KEY in a table. It ensures that each row can be uniquely identified.

CREATE TABLE Employees (
    ID INT PRIMARY KEY,
    Name VARCHAR(50),
    Department VARCHAR(50)
);

Here, an index is automatically created on the ID column.

👉 Unique Index

A unique index ensures that no two rows have the same value in the indexed column.

CREATE UNIQUE INDEX idx_email ON Users(Email);

This prevents duplicate email addresses in the Users table.

👉 Composite Index

A composite index is an index on two or more columns. It’s useful when you often search using multiple conditions together.

CREATE INDEX idx_name_dept ON Employees(Name, Department);

1️⃣ Why Does This Topic Exist?


Indexes make searching for data in large tables much faster. In Indian banks, e-commerce, and government databases, indexes are used to quickly find customer records, transactions, or Aadhaar details without scanning every row.



2️⃣ What Is an Index?


An index is a special data structure that helps the database find rows quickly, like the index in a book. It does not store data, but points to where the data is stored.
