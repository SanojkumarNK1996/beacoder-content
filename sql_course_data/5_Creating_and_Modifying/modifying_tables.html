Modifying Tables in SQL

Once a table is created, you might later realize that you need to make changes — for example, add a new column, rename an existing one, or delete a column you no longer need.

SQL lets you do all these changes using the ALTER TABLE command.

1️⃣ What is ALTER TABLE?

The ALTER TABLE statement is used to modify an existing table’s structure without deleting the data inside it.

Modifying Tables: 10-Part Beginner-Friendly Guide

1. Why Does This Topic Exist?

As your app or business grows, you may need to change your data structure—add a new field, remove an old one, or fix a mistake. The ALTER TABLE command lets you update tables without losing your data.

2. What Is This?

ALTER TABLE is an SQL command used to change the structure of an existing table. You can add, remove, or modify columns, or even rename the table or its columns.

3. Syntax/Structure

-- Add a column
ALTER TABLE Students ADD COLUMN Address VARCHAR(100);

-- Modify a column
ALTER TABLE Students MODIFY COLUMN Name VARCHAR(100);

-- Drop a column
ALTER TABLE Students DROP COLUMN Age;

-- Rename a table
ALTER TABLE Students RENAME TO Learners;

-- Rename a column (MySQL 8+)
ALTER TABLE Students RENAME COLUMN Email TO EmailID;

4. Step-by-Step: How It Works

  1. Decide what change you need (add, remove, modify, rename).
  2. Write the appropriate ALTER TABLE command.
  3. Run the command in your SQL tool (like MySQL Workbench, DBeaver, or command line).
  4. Check the table structure with DESCRIBE table_name; to confirm the change.

5. When to Use

  • When your requirements change (e.g., you need to store students' addresses).
  • When you want to fix a mistake in the table structure.
  • When you want to rename tables or columns for clarity.

6. When Not to Use

  • When you want to change data (use UPDATE for that, not ALTER TABLE).
  • When you want to delete all data (use DROP TABLE or TRUNCATE instead).

7. Minimum to Remember

  • ALTER TABLE changes the structure, not the data.
  • Use ADD COLUMN, MODIFY COLUMN, DROP COLUMN, RENAME TO, and RENAME COLUMN as needed.

8. Common Mistakes

  • Forgetting to check if the column already exists or is used elsewhere.
  • Misspelling column or table names.
  • Trying to drop a column that is part of a PRIMARY KEY or FOREIGN KEY without updating constraints.

9. Interviewer’s Perspective

  • “How do you add a new column to an existing table in SQL?”
  • “What happens if you drop a column that is used in a foreign key?”
  • “How do you rename a table in SQL?”

10. Practice Questions

  1. Add a column PhoneNumber to the Students table.
  2. Change the Name column to allow 150 characters.
  3. Remove the Address column from the table.
  4. Rename the Students table to Pupils.
  5. Rename the EmailID column to EmailAddress.
ALTER TABLE Students
DROP COLUMN PhoneNumber;

This completely removes the column and its data. ⚠️ Warning: Once a column is dropped, you cannot recover its data unless you have a backup!

7️⃣ Renaming a Table

Sometimes, you may want to rename the table itself (for example, from Students to StudentRecords).

RENAME TABLE Students TO StudentRecords;

After this, all queries must use the new name.

8️⃣ Changing the Default Value of a Column

SQL allows you to modify the default value assigned to a column when new data is inserted.

ALTER TABLE Students
ALTER COLUMN Age SET DEFAULT 18;

Now, if you insert a student without specifying an age, it will automatically use 18.

9️⃣ Adding Constraints to an Existing Table

Constraints are rules that define what kind of data can go into a table. We’ll cover them in detail in the “Constraints in SQL” topic, but here’s a quick look.

Example: Add a UNIQUE constraint to the Email column:

ALTER TABLE Students
ADD CONSTRAINT unique_email UNIQUE (Email);

This ensures no two students can have the same email address.

🔟 Dropping Constraints

If you need to remove a constraint, you can use:

ALTER TABLE Students
DROP CONSTRAINT unique_email;

(Constraint names vary by database system — MySQL, PostgreSQL, SQL Server all have slightly different syntaxes.)

Summary

  • ALTER TABLE — used to modify existing table structure.
  • ADD — to add new columns.
  • MODIFY / ALTER COLUMN — to change column data types or defaults.
  • CHANGE / RENAME COLUMN — to rename columns.
  • DROP COLUMN — to delete a column.
  • RENAME TABLE — to rename a table.

Making structural changes to tables is common during database design. Always make sure you have a backup before running ALTER commands on a live database.
