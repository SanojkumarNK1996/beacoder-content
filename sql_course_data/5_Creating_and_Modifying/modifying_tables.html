Modifying Tables in SQL

Once a table is created, you might later realize that you need to make changes — for example, add a new column, rename an existing one, or delete a column you no longer need.

SQL lets you do all these changes using the ALTER TABLE command.

1️⃣ What is ALTER TABLE?

The ALTER TABLE statement is used to modify an existing table’s structure without deleting the data inside it.

You can use it to:

  • Add new columns
  • Delete (drop) columns
  • Rename columns
  • Change data types
  • Add or remove constraints

2️⃣ Adding a New Column

Suppose we have a Students table and now we want to store each student’s phone number.

ALTER TABLE Students
ADD PhoneNumber VARCHAR(15);

Explanation:

  • ALTER TABLE Students — means we’re going to change the Students table.
  • ADD PhoneNumber VARCHAR(15) — adds a new column named PhoneNumber that stores up to 15 characters.

Now if you run DESCRIBE Students;, you’ll see the new column added at the end.

3️⃣ Adding Multiple Columns

You can add more than one column at a time:

ALTER TABLE Students
ADD (Address VARCHAR(100), City VARCHAR(50));

This will add both Address and City columns in a single command.

4️⃣ Modifying a Column’s Data Type

If you want to change the data type of an existing column, use MODIFY (or ALTER COLUMN depending on the SQL version).

Example: Suppose you initially set Age as INT but you want to make it SMALLINT to save space.

ALTER TABLE Students
MODIFY Age SMALLINT;

This changes the column’s data type while keeping existing data intact (as long as it’s compatible).

5️⃣ Renaming a Column

To rename a column, use the CHANGE or RENAME COLUMN syntax (varies by database).

Example (MySQL):

ALTER TABLE Students
CHANGE Name FullName VARCHAR(50);

This renames the column Name to FullName and keeps the same data type.

Example (PostgreSQL / SQL Server):

ALTER TABLE Students
RENAME COLUMN Name TO FullName;

6️⃣ Deleting (Dropping) a Column

If you no longer need a column, you can remove it with DROP COLUMN.

ALTER TABLE Students
DROP COLUMN PhoneNumber;

This completely removes the column and its data. ⚠️ Warning: Once a column is dropped, you cannot recover its data unless you have a backup!

7️⃣ Renaming a Table

Sometimes, you may want to rename the table itself (for example, from Students to StudentRecords).

RENAME TABLE Students TO StudentRecords;

After this, all queries must use the new name.

8️⃣ Changing the Default Value of a Column

SQL allows you to modify the default value assigned to a column when new data is inserted.

ALTER TABLE Students
ALTER COLUMN Age SET DEFAULT 18;

Now, if you insert a student without specifying an age, it will automatically use 18.

9️⃣ Adding Constraints to an Existing Table

Constraints are rules that define what kind of data can go into a table. We’ll cover them in detail in the “Constraints in SQL” topic, but here’s a quick look.

Example: Add a UNIQUE constraint to the Email column:

ALTER TABLE Students
ADD CONSTRAINT unique_email UNIQUE (Email);

This ensures no two students can have the same email address.

🔟 Dropping Constraints

If you need to remove a constraint, you can use:

ALTER TABLE Students
DROP CONSTRAINT unique_email;

(Constraint names vary by database system — MySQL, PostgreSQL, SQL Server all have slightly different syntaxes.)

Summary

  • ALTER TABLE — used to modify existing table structure.
  • ADD — to add new columns.
  • MODIFY / ALTER COLUMN — to change column data types or defaults.
  • CHANGE / RENAME COLUMN — to rename columns.
  • DROP COLUMN — to delete a column.
  • RENAME TABLE — to rename a table.

Making structural changes to tables is common during database design. Always make sure you have a backup before running ALTER commands on a live database.
