Creating Databases and Tables

Before we can store any data, we need a database — a structured place to hold information. Once a database exists, we can create tables inside it to organize data into rows and columns.

1️⃣ What is a Database?

A database is a container that holds tables, views, stored procedures, and other objects. You can think of a database as a “folder” that stores all the related data for an application.

For example, a School Database might include:

  • A table for students
  • A table for teachers
  • A table for courses
  • A table for grades

2️⃣ Creating a Database

To create a new database in SQL, use the CREATE DATABASE command.

CREATE DATABASE SchoolDB;

Explanation:

  • CREATE DATABASE — tells SQL you want to make a new database.
  • SchoolDB — is the name of your database (you can choose any name).

After creating: You can start using the database with:

USE SchoolDB;

This tells SQL to start working inside that database.

3️⃣ What is a Table?

A table is like a spreadsheet — it stores information in rows and columns.

Each column represents a data field (like name, age, or email), and each row represents a record (a single person or item).

Example: A Students table might look like this:

+-----------+-------------+-------+-----------------------+
| StudentID | Name        | Age   | Email                 |
+-----------+-------------+-------+-----------------------+
| 1         | Riya Singh  | 18    | riya@example.com      |
| 2         | Arjun Mehta | 19    | arjun@example.com     |
| 3         | Neha Patel  | 20    | neha@example.com      |
+-----------+-------------+-------+-----------------------+

4️⃣ Creating a Table

Once you’re inside a database, you can create a table using the CREATE TABLE statement.

CREATE TABLE Students (
  StudentID INT PRIMARY KEY,
  Name VARCHAR(50),
  Age INT,
  Email VARCHAR(100)
);

Let’s break this down:

  • CREATE TABLE Students — creates a new table named “Students”.
  • StudentID INT PRIMARY KEY — creates a column for a numeric student ID and marks it as the unique identifier.
  • Name VARCHAR(50) — a text column for the student’s name (up to 50 characters).
  • Age INT — stores the student’s age as a number.
  • Email VARCHAR(100) — stores their email address.

5️⃣ Verifying the Table

You can see all the tables in your current database using:

SHOW TABLES;

To see the structure of a particular table:

DESCRIBE Students;

This will display each column, its data type, and whether it allows NULL (empty) values.

6️⃣ Inserting Some Data (Just as a Test)

After creating the table, you can add data using the INSERT INTO statement:

INSERT INTO Students (StudentID, Name, Age, Email)
VALUES (1, 'Riya Singh', 18, 'riya@example.com');

To check if it worked, use:

SELECT * FROM Students;

This will show the data you just added.
