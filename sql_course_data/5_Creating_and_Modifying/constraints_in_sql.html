Constraints in SQL

When you create tables in a database, you often want to make sure that the data entered is valid and meaningful. For example, you wouldn’t want a student record without a name, or two users with the same ID. This is where constraints come in.

1️⃣ What Are Constraints?

Constraints are rules that you apply to columns in a table to control what kind of data can be stored there. They ensure that your data stays accurate and consistent over time.

Think of them as “guardrails” for your database — they prevent mistakes before they happen.

2️⃣ Why Are Constraints Important?

  • ✅ They maintain data accuracy.
  • ✅ They prevent invalid data from being inserted.
  • ✅ They enforce relationships between tables.
  • ✅ They protect data integrity and consistency.

3️⃣ Common Types of Constraints

SQL provides several types of constraints. Let’s explore each one with simple examples.

🔹 NOT NULL Constraint

This ensures that a column cannot have a NULL (empty) value.

CREATE TABLE Students (
  StudentID INT,
  Name VARCHAR(50) NOT NULL,
  Age INT
);

In this example, every student must have a name. If you try to insert a record without a name, SQL will reject it.

INSERT INTO Students (StudentID, Age) VALUES (1, 18);
-- ❌ Error: Column 'Name' cannot be null

🔹 UNIQUE Constraint

The UNIQUE constraint makes sure that all values in a column are different.

CREATE TABLE Users (
  UserID INT,
  Email VARCHAR(100) UNIQUE
);

This ensures that no two users can have the same email address.

INSERT INTO Users VALUES (1, 'john@example.com');
INSERT INTO Users VALUES (2, 'john@example.com');
-- ❌ Error: Duplicate entry 'john@example.com'

🔹 PRIMARY KEY Constraint

The PRIMARY KEY uniquely identifies each record in a table. It automatically combines the behavior of NOT NULL and UNIQUE.

CREATE TABLE Employees (
  EmpID INT PRIMARY KEY,
  Name VARCHAR(50)
);

This ensures that:

  • Each employee has a unique ID.
  • No employee can have a NULL (empty) ID.

🔹 FOREIGN KEY Constraint

The FOREIGN KEY constraint is used to link two tables together. It ensures that the value in one table matches a valid value in another table.

CREATE TABLE Departments (
  DeptID INT PRIMARY KEY,
  DeptName VARCHAR(50)
);

CREATE TABLE Employees (
  EmpID INT PRIMARY KEY,
  Name VARCHAR(50),
  DeptID INT,
  FOREIGN KEY (DeptID) REFERENCES Departments(DeptID)
);

This ensures that every employee’s DeptID must exist in the Departments table.

If you try to insert an employee with a non-existent department, SQL will block it.

🔹 CHECK Constraint

The CHECK constraint ensures that values in a column meet a specific condition.

CREATE TABLE Products (
  ProductID INT PRIMARY KEY,
  Price DECIMAL(10,2) CHECK (Price > 0)
);

This ensures that the Price value must always be positive.

INSERT INTO Products VALUES (1, -10.00);
-- ❌ Error: CHECK constraint failed

🔹 DEFAULT Constraint

The DEFAULT constraint automatically assigns a value to a column if no value is provided.

CREATE TABLE Orders (
  OrderID INT PRIMARY KEY,
  Status VARCHAR(20) DEFAULT 'Pending'
);

If you insert a record without specifying Status, it will automatically take the value 'Pending'.

INSERT INTO Orders (OrderID) VALUES (1);

-- Result:
-- | OrderID | Status  |
-- |    1    | Pending |

4️⃣ Combining Multiple Constraints

You can apply multiple constraints to the same table for stronger validation.

CREATE TABLE Customers (
  CustomerID INT PRIMARY KEY,
  Name VARCHAR(50) NOT NULL,
  Email VARCHAR(100) UNIQUE,
  Age INT CHECK (Age >= 18),
  Country VARCHAR(50) DEFAULT 'India'
);

Here’s what happens:

  • Each customer must have an ID (PRIMARY KEY).
  • The name cannot be blank (NOT NULL).
  • Email must be unique (UNIQUE).
  • Age must be 18 or older (CHECK).
  • If no country is given, it defaults to “India” (DEFAULT).

5️⃣ Viewing Constraints in a Table

You can use SQL commands to check what constraints exist on a table. For example, in MySQL:

SHOW CREATE TABLE Customers;

This will show the table structure including all constraints applied.

6️⃣ Modifying or Dropping Constraints

You can also add or remove constraints using the ALTER TABLE command.

-- Add a new constraint
ALTER TABLE Employees
ADD CONSTRAINT chk_age CHECK (Age >= 18);

-- Drop an existing constraint (syntax may vary by database)
ALTER TABLE Employees
DROP CONSTRAINT chk_age;

Always be careful when dropping constraints, as it removes important data protections.

7️⃣ Summary

  • NOT NULL — Prevents empty values.
  • UNIQUE — Prevents duplicate values.
  • PRIMARY KEY — Uniquely identifies each row.
  • FOREIGN KEY — Links tables together.
  • CHECK — Ensures valid conditions.
  • DEFAULT — Sets a default value if none is given.

Constraints are like the “rules of the road” for your database — they keep your data clean, valid, and trustworthy.
