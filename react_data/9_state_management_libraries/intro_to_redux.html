🔄 Introduction to Redux

🎯 What is Redux?

Redux is a predictable state container for JavaScript apps. State lives in a single store, updated only via actions (plain objects) processed by pure reducer functions.

📋 Core Concepts

  • Store: Single source of truth for all app state
  • Action: Plain object describing what happened { type: 'ADD_ITEM', payload: item }
  • Reducer: Pure function: (state, action) => newState
  • Dispatch: Send an action to the store
  • Selector: Read data from the store

🔄 Unidirectional Data Flow

UI event → dispatch(action) → reducer(currentState, action) → newState → UI re-renders

📌 Traditional Redux (Verbose)

const ADD = 'cart/add';
const addItem = (item) => ({ type: ADD, payload: item });

function cartReducer(state = [], action) {
  switch (action.type) {
    case ADD: return [...state, action.payload];
    default: return state;
  }
}

🎯 Summary

Classic Redux required a lot of boilerplate. Modern apps use Redux Toolkit which dramatically simplifies this. Understanding classic Redux is important to debug and understand library internals.
