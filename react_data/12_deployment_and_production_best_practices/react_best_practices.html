✅ React Best Practices

📁 Project Structure

src/
├── components/    # Shared, reusable UI components
├── pages/         # Route-level components
├── hooks/         # Custom hooks
├── services/      # API calls (api.js, userService.js)
├── store/         # State management (Redux/Zustand)
├── utils/         # Pure utility functions
├── types/         # TypeScript interfaces (if using TS)
└── assets/        # Images, fonts

📋 Key Best Practices

  • Composition over inheritance: Build complex UIs by composing small components
  • Single responsibility: Each component does one thing well
  • Colocate related code: Component + its CSS + its tests in the same folder
  • Use TypeScript: Catches errors at compile time, great DX, better refactoring
  • Named exports over default exports: Easier to refactor, better autocomplete
  • Avoid deeply nested state: Normalize complex state (separate arrays/objects)
  • Don't store derived state: Compute from state on render, not in additional state
  • Consistent naming: PascalCase for components, camelCase for functions, SCREAMING_SNAKE for constants

🎯 Summary

These practices accumulate into a codebase that's easy to read, test, and scale. The biggest wins: TypeScript, React Query for server state, and colocating related code.
