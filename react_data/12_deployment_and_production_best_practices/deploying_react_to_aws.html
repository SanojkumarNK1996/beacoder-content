☁️ Deploying React to AWS

🎯 Option 1: S3 + CloudFront (Static Hosting)

Host the React build on S3, serve globally via CloudFront CDN with HTTPS.

# Build the app
npm run build

# Upload to S3 (AWS CLI)
aws s3 sync dist/ s3://my-react-app-bucket --delete

# S3 bucket settings:
# - Enable "Static website hosting"
# - Set index.html as index document AND error document (SPA routing fix)
# - Bucket policy: allow public read

🔧 CloudFront Configuration

  • Origin: S3 bucket website endpoint
  • Add custom error response: 404 → /index.html → 200 (SPA routing fix)
  • Enable HTTPS with ACM certificate
  • Set cache-control headers for assets vs index.html

🚀 Option 2: AWS Amplify

npm install -g @aws-amplify/cli
amplify init
amplify add hosting
amplify publish

Amplify provides Vercel-like DX with full AWS integration — auto-deploys, preview environments, custom domains, and direct access to other AWS services.

🎯 Summary

S3 + CloudFront for full control and lowest cost. AWS Amplify for a managed experience like Vercel/Netlify with deeper AWS integration.
