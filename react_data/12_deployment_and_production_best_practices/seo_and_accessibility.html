🔎 SEO and Accessibility in React SPAs

🎯 SEO Challenges with SPAs

Traditional React SPAs render content with JavaScript. Search engine crawlers see an empty HTML shell initially — bad for SEO.

🔧 Solutions

  • Server-Side Rendering (SSR): Use Next.js — renders HTML on the server, ideal for SEO
  • Pre-rendering: Generate static HTML at build time for known routes
  • SSG: Static Site Generation with Next.js for content-heavy sites

🏷️ Meta Tags with React Helmet

import { Helmet } from 'react-helmet-async';

function ProductPage({ product }) {
  return (
    <>
      <Helmet>
        <title>{product.name} | MyShop</title>
        <meta name="description" content={product.description} />
        <meta property="og:image" content={product.image} />
      </Helmet>
      <ProductDetails product={product} />
    </>
  );
}

♿ Accessibility (a11y) Basics

  • Use semantic HTML: <button> not <div onClick>
  • Add alt text to all images
  • Ensure sufficient color contrast (WCAG AA: 4.5:1 ratio)
  • All interactive elements reachable by keyboard (Tab key)
  • Use ARIA labels for icon-only buttons: <button aria-label="Close">

🎯 Summary

For SEO-critical apps, use Next.js instead of CRA/Vite. For accessibility — semantic HTML, keyboard navigation, and proper ARIA labels cover 80% of requirements.
