🔐 Environment Variables in React

🎯 What are Environment Variables?

Configuration values that change between environments (development, staging, production): API URLs, API keys, feature flags.

🔧 Vite Environment Variables

# .env (defaults — commit to git, no secrets)
VITE_API_URL=http://localhost:4000

# .env.local (local overrides — in .gitignore!)
VITE_API_URL=http://localhost:3000

# .env.production (production values — no secrets!)
VITE_API_URL=https://api.myapp.com

# Access in code (only VITE_ prefix vars are exposed to browser):
const apiUrl = import.meta.env.VITE_API_URL;
const isProd = import.meta.env.PROD;   // Built-in Vite variable

⚠️ Security Critical Rule

Never put secrets (private API keys, DB passwords) in frontend env vars. All Vite/CRA env vars are embedded in the JS bundle — anyone can read them in the browser devtools.

  • Frontend: public API keys (publishable Stripe key, public Firebase config) ✅
  • Frontend: API URL, feature flags ✅
  • Frontend: private API keys, OAuth secrets, database URLs ❌

🎯 Summary

Prefix env vars with VITE_ (Vite) or REACT_APP_ (CRA) to expose them to the browser. Never put secrets in frontend env vars — they end up in the compiled JS.
