🏗️ Building React Apps for Production

🎯 Production Build

# Vite (recommended)
npm run build        # Creates /dist folder with optimized assets
npm run preview      # Preview the production build locally

# Create React App
npm run build        # Creates /build folder

🔧 What the Build Process Does

  • Minification: Removes whitespace, shortens variable names
  • Tree shaking: Removes unused code from bundles
  • Code splitting: Splits into multiple chunks for parallel loading
  • Asset hashing: main.abc123.js — enables long cache TTLs
  • Source maps: Map minified code back to source for debugging

📋 Pre-Deploy Checklist

  • All .env variables are set in deployment platform
  • API base URL points to production server
  • Console.log statements removed (or use a logger with levels)
  • Error boundary in place for the entire app
  • 404 route configured for SPA (redirect all paths to index.html)

🎯 Summary

npm run build creates an optimized production bundle. The output is static files (HTML, JS, CSS, assets) that can be hosted anywhere — CDN, S3, Vercel, Netlify.
