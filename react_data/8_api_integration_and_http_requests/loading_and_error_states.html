⏳ Handling Loading and Error States

🎯 The Three States Pattern

Every async operation has three states: loading, success, error. Always handle all three.

function UserProfile({ userId }) {
  const [user, setUser] = useState(null);
  const [status, setStatus] = useState('idle'); // idle | loading | success | error
  const [error, setError] = useState(null);

  useEffect(() => {
    setStatus('loading');
    api.getUser(userId)
      .then(data => { setUser(data); setStatus('success'); })
      .catch(err => { setError(err.message); setStatus('error'); });
  }, [userId]);

  if (status === 'loading') return <Skeleton />;
  if (status === 'error') return <ErrorMessage message={error} retry={reload} />;
  if (!user) return null;
  return <UserCard user={user} />;
}

📋 UX Best Practices

  • Show skeleton screens instead of spinners for content (less jarring)
  • Always show a retry button on error states
  • Preserve stale data during re-fetching (don't blank the UI)
  • Debounce rapidly triggered fetches (search inputs)

🎯 Summary

Model async state with status enum (idle/loading/success/error) rather than separate booleans to avoid impossible states like isLoading=true && isError=true.
