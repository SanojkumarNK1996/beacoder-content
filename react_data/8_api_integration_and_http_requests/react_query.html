🔍 React Query for Data Fetching

🎯 What is React Query?

TanStack Query (React Query) is the gold standard for server state management in React. It handles caching, background refetching, loading/error states, and pagination automatically.

npm install @tanstack/react-query

🔧 Setup

// main.jsx
import { QueryClient, QueryClientProvider } from '@tanstack/react-query';
const queryClient = new QueryClient();

// Wrap your app:
<QueryClientProvider client={queryClient}>
  <App />
</QueryClientProvider>

📋 useQuery and useMutation

// Fetching data
function UserList() {
  const { data: users, isLoading, error } = useQuery({
    queryKey: ['users'],
    queryFn: () => api.get('/users').then(r => r.data),
    staleTime: 5 * 60 * 1000,  // Cache for 5 minutes
  });
  if (isLoading) return <Spinner />;
  if (error) return <Error />;
  return <ul>{users.map(u => <li key={u.id}>{u.name}</li>)}</ul>;
}

// Mutating data
const { mutate: createUser } = useMutation({
  mutationFn: (data) => api.post('/users', data),
  onSuccess: () => queryClient.invalidateQueries(['users']),
});

🎯 Summary

React Query eliminates 80% of data-fetching boilerplate. It handles caching, deduplication, background sync, and refetch on focus automatically. Use it for any server state.
