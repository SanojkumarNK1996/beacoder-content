⚡ React Performance Basics

🎯 Understanding React Re-renders

React re-renders a component when: its state changes, its props change, or its parent re-renders. Most performance issues come from excessive re-renders.

🔍 When to Optimize

Don't optimize prematurely. First, measure with tools:

  • React DevTools Profiler: See which components re-render and why
  • Chrome Performance Tab: Full timeline of browser rendering
  • Lighthouse: Core Web Vitals metrics

📋 Key Metrics (Core Web Vitals)

  • LCP (Largest Contentful Paint): How fast is the main content visible? Target < 2.5s
  • FID/INP (Interaction to Next Paint): Responsiveness. Target < 200ms
  • CLS (Cumulative Layout Shift): Visual stability. Target < 0.1

🔧 Quick Wins

  • Avoid creating objects/arrays in render (new reference every time = child re-renders)
  • Move expensive calculations to useMemo
  • Use React.memo for pure components that re-render often
  • Virtualize long lists (react-window)

🎯 Summary

Measure before you optimize. The Profiler in React DevTools shows "why did this render?" — fix the root cause, not symptoms.
