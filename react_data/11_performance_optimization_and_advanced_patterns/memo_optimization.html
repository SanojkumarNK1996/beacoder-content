🧠 React.memo and Memoization

🎯 React.memo

React.memo is a Higher Order Component that prevents re-rendering if props haven't changed (shallow comparison).

// Without memo: re-renders every time parent re-renders
function UserCard({ user }) {
  return <div>{user.name}</div>;
}

// With memo: only re-renders when user prop changes
const UserCard = React.memo(function UserCard({ user }) {
  return <div>{user.name}</div>;
});

⚠️ Pitfall: New Objects Break Memoization

// BAD: new object created every render → memo is useless
<UserCard style={{ color: 'red' }} />  

// GOOD: stable reference
const style = useMemo(() => ({ color: 'red' }), []);
<UserCard style={style} />

// BAD: new function every render
<UserCard onClick={() => handleClick(id)} />  

// GOOD: stable function
const handleClick = useCallback(() => doSomething(id), [id]);
<UserCard onClick={handleClick} />

🎯 Summary

React.memo works only when combined with stable props (useMemo + useCallback). Use it for expensive components that receive the same props often.
