⌨️ Keyboard and Input Events

📋 Input Change Events

function SearchBar() {
  const [query, setQuery] = useState('');

  const handleChange = (e) => {
    setQuery(e.target.value);  // e.target.value is the input's current value
  };

  return <input value={query} onChange={handleChange} placeholder="Search..." />;
}

⌨️ Keyboard Events

function SearchBar() {
  const [query, setQuery] = useState('');

  const handleKeyDown = (e) => {
    if (e.key === 'Enter') submitSearch(query);
    if (e.key === 'Escape') setQuery('');
  };

  return (
    <input
      value={query}
      onChange={e => setQuery(e.target.value)}
      onKeyDown={handleKeyDown}
    />
  );
}

📌 Common Input Events

  • onChange: Fires every time the value changes
  • onFocus: When input gains focus
  • onBlur: When input loses focus (great for validation)
  • onKeyDown/onKeyUp/onKeyPress: Keyboard events

🎯 Summary

onChange + onKeyDown cover 90% of input interactions. Use onBlur for "touched" validation (show error after user leaves the field).
