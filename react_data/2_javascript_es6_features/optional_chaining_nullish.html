❓ Optional Chaining and Nullish Coalescing

🎯 Optional Chaining (?.)

Safely access deeply nested properties without throwing "Cannot read property of undefined".

const user = { profile: { address: { city: 'London' } } };

// Old way (verbose):
const city = user && user.profile && user.profile.address && user.profile.address.city;

// Optional chaining:
const city = user?.profile?.address?.city; // 'London' (or undefined safely)

// With methods:
const name = user?.getName?.(); // Calls only if method exists

🌊 Nullish Coalescing (??)

Returns the right-hand side only if the left side is null or undefined (not 0 or empty string).

const name = user?.name ?? 'Anonymous'; // 'Anonymous' if null/undefined
const count = data?.count ?? 0;         // 0 if null/undefined

// vs || (breaks for falsy values like 0, '', false)
const price = product?.price ?? 0;  // Correct: 0 is a valid price
const price2 = product?.price || 0; // Wrong: treats 0 as falsy!

🎯 Summary

Use ?. to safely traverse API responses with nullable fields. Use ?? instead of || for default values to avoid bugs with falsy values like 0 or empty string.
