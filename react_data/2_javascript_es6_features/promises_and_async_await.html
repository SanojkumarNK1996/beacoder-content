⏳ Promises and Async/Await

🎯 The Problem: Async Code

Fetching data, reading files, delays — these are asynchronous. JavaScript handles async code with Promises and async/await.

📋 Promises

fetch('/api/users')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error(error));

✨ Async/Await (cleaner syntax)

async function fetchUsers() {
  try {
    const response = await fetch('/api/users');
    const data = await response.json();
    return data;
  } catch (error) {
    console.error('Failed to fetch:', error);
  }
}

// In React useEffect:
useEffect(() => {
  const load = async () => {
    const users = await fetchUsers();
    setUsers(users);
  };
  load();
}, []);

🎯 Summary

Use async/await in React for all API calls — it reads like synchronous code and is easier to reason about. Always wrap in try/catch for error handling.
