📦 ES6 Modules: Import and Export

📤 Named Exports

// utils.js
export const add = (a, b) => a + b;
export const PI = 3.14159;
export function greet(name) { return `Hello, ${name}`; }

// consumer.js
import { add, PI } from './utils';
import { add as sum } from './utils';  // Alias

📦 Default Exports

// Button.jsx
export default function Button({ children, onClick }) {
  return <button onClick={onClick}>{children}</button>;
}

// App.jsx
import Button from './Button';   // Any name works
import MyBtn from './Button';    // Also valid

📌 Naming Convention

  • React components: default export, PascalCase file name (Button.jsx)
  • Utilities, hooks, constants: named exports

🎯 Summary

React components use default exports. Hooks and utilities use named exports. VS Code auto-imports handle most of this automatically.
