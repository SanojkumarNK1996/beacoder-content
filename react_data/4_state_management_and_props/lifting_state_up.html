⬆️ Lifting State Up

🎯 The Problem

When two sibling components need to share state, neither can directly access the other's state. The solution: lift state up to their common parent.

// ❌ WRONG: state in child, sibling can't access
function Child1() {
  const [value, setValue] = useState('');
  return <input value={value} onChange={e => setValue(e.target.value)} />;
}
function Child2() { /* can't see Child1's value */ }

// ✅ Lift state to parent
function Parent() {
  const [value, setValue] = useState('');
  return (
    <>
      <Child1 value={value} onChange={setValue} />
      <Child2 value={value} />      {/* Both can see it */}
    </>
  );
}
function Child1({ value, onChange }) {
  return <input value={value} onChange={e => onChange(e.target.value)} />;
}
function Child2({ value }) {
  return <p>You typed: {value}</p>;
}

🎯 Summary

If two components need shared data, lift the state to their nearest common ancestor. Pass state down as props, update state via callback props.
