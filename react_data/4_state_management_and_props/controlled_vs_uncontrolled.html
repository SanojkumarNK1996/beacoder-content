🎛️ Controlled vs Uncontrolled Components

✅ Controlled Components

React controls the form element's value via state. The state is the "single source of truth."

function ControlledInput() {
  const [value, setValue] = useState('');
  return (
    <input
      value={value}
      onChange={e => setValue(e.target.value)}
    />
  );
}
  • Full control: validate on change, transform input, disable submit based on value
  • More code but more predictable

🔓 Uncontrolled Components

DOM manages the input value; React reads it via a ref when needed.

function UncontrolledInput() {
  const inputRef = useRef();
  const handleSubmit = () => console.log(inputRef.current.value);
  return <input ref={inputRef} defaultValue="initial" />;
}
  • Less code; good for simple forms or file inputs
  • Harder to validate in real time

🎯 Summary

Use controlled components for most forms — they give you real-time validation and predictable behavior. Uncontrolled components are acceptable for simple read-once forms.
