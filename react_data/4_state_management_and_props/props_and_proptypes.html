🎭 Props and PropTypes

🎯 Passing Props

// Parent passes props
<UserCard
  name="Alice"
  age={30}
  isAdmin={true}
  hobbies={['reading', 'coding']}
  onDelete={handleDelete}
/>

// Child receives props
function UserCard({ name, age, isAdmin, hobbies, onDelete }) {
  return (
    <div>
      <h2>{name} {isAdmin && '(Admin)'}</h2>
      <p>Age: {age}</p>
      <ul>{hobbies.map(h => <li key={h}>{h}</li>)}</ul>
      <button onClick={onDelete}>Delete</button>
    </div>
  );
}

🔍 PropTypes (Runtime Type Checking)

import PropTypes from 'prop-types';

UserCard.propTypes = {
  name: PropTypes.string.isRequired,
  age: PropTypes.number.isRequired,
  isAdmin: PropTypes.bool,
  onDelete: PropTypes.func.isRequired,
};

UserCard.defaultProps = { isAdmin: false };

📌 TypeScript Alternative

PropTypes are runtime checks. TypeScript interfaces give compile-time safety — preferred for larger projects.

🎯 Summary

Props are read-only data passed from parent. Use PropTypes or TypeScript interfaces to document expected prop types and catch bugs early.
