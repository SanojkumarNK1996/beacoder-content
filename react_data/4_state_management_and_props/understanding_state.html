💡 Understanding State in React

🎯 What is State?

State is data that can change over time and, when it changes, causes the component to re-render to reflect the new data in the UI.

📋 State vs Props

FeaturePropsState
Owned byParent componentComponent itself
MutableNo (read-only)Yes (via setState)
Used forPassing data downComponent's own data
Triggers re-renderWhen parent re-rendersWhen state changes

📌 What Should Be State?

  • User input (form fields)
  • Toggle values (isOpen, isLoading)
  • API response data
  • Current page / selected item

⚠️ What Should NOT Be State?

  • Derived values (compute from existing state instead)
  • Props (don't copy props into state)
  • Constant values

🎯 Summary

State = the component's memory for changing data. Only put things in state that actually need to trigger a re-render and that can't be derived from existing state or props.
