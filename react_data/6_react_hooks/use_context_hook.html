🌐 useContext Hook

🎯 The Problem: Prop Drilling

When data needs to pass through many nested components that don't need it (just pass it along), prop drilling becomes painful. React Context solves this.

📋 Creating and Using Context

// 1. Create context
const ThemeContext = createContext('light');

// 2. Provide context (wrap tree)
function App() {
  const [theme, setTheme] = useState('dark');
  return (
    <ThemeContext.Provider value={{ theme, setTheme }}>
      <Layout />
    </ThemeContext.Provider>
  );
}

// 3. Consume anywhere in the tree (no prop drilling!)
function DeepChild() {
  const { theme, setTheme } = useContext(ThemeContext);
  return <button onClick={() => setTheme(t => t === 'dark' ? 'light' : 'dark')}>
    Current: {theme}
  </button>;
}

⚠️ When NOT to Use Context

Context re-renders all consumers when the value changes. Don't use it for high-frequency updates (e.g., mouse position, every keystroke). Use state management libraries (Zustand, Redux) for those cases.

🎯 Summary

Context for cross-cutting concerns: theme, auth, locale, user preferences. Not a full state management solution — use Zustand/Redux for complex shared state.
