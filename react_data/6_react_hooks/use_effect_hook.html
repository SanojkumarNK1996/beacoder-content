🔄 useEffect Hook

🎯 What it does

useEffect handles side effects — operations that interact with the outside world: API calls, subscriptions, timers, DOM manipulation.

useEffect(() => {
  // Side effect code here
  return () => {
    // Cleanup (called before next effect or unmount)
  };
}, [dependencies]);  // Re-runs when dependencies change

📋 Common Patterns

// On mount only (empty deps):
useEffect(() => {
  fetchData();
}, []);

// On every render (no deps):
useEffect(() => { document.title = 'Updated'; });

// When a value changes:
useEffect(() => {
  fetchUserById(userId);
}, [userId]);   // Re-runs when userId changes

// Cleanup (subscriptions, timers):
useEffect(() => {
  const timer = setInterval(() => tick(), 1000);
  return () => clearInterval(timer);  // Cleanup
}, []);

⚠️ Pitfall: Infinite Loop

If you update state inside useEffect without proper dependencies, you'll cause an infinite re-render loop. Always specify the dependency array.

🎯 Summary

useEffect is the Swiss Army knife for side effects. The dependency array controls when it runs. Always return a cleanup function for subscriptions/timers.
