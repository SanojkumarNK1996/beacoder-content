🔒 Protected Routes and Auth Guards

🎯 What are Protected Routes?

Routes that require authentication — redirect to login if user is not logged in.

import { Navigate, Outlet } from 'react-router-dom';

// ProtectedRoute component
function ProtectedRoute() {
  const { user } = useAuth();  // From AuthContext
  if (!user) return <Navigate to="/login" replace />;
  return <Outlet />;           // Render children if authenticated
}

// Usage in routes:
<Routes>
  <Route path="/login" element={<LoginPage />} />

  {/* Protected routes */}
  <Route element={<ProtectedRoute />}>
    <Route path="/dashboard" element={<Dashboard />} />
    <Route path="/profile" element={<Profile />} />
  </Route>
</Routes>

📌 Role-Based Protection

function AdminRoute() {
  const { user } = useAuth();
  if (!user) return <Navigate to="/login" replace />;
  if (user.role !== 'admin') return <Navigate to="/forbidden" replace />;
  return <Outlet />;
}

🎯 Summary

Create a ProtectedRoute wrapper component checking auth status. Wrap private routes in it. Use <Navigate replace> to redirect cleanly (no broken back-button history).
