🧪 Testing Fundamentals for React

🎯 Why Test?

Tests catch regressions, document behavior, and enable confident refactoring. Good tests focus on user behavior, not implementation details.

📋 Types of Tests

  • Unit tests: Test a single function or component in isolation (fastest)
  • Integration tests: Test how components interact (main sweet spot)
  • End-to-end (E2E) tests: Test full user flows in a real browser (slowest, highest confidence)

🔧 Testing Stack

  • Vitest / Jest: Test runner (execute tests, assertions)
  • React Testing Library (RTL): Render components, simulate user interactions
  • Cypress / Playwright: E2E testing in real browser

📌 Test Structure: AAA Pattern

it('should increment counter on click', () => {
  // Arrange
  render(<Counter />);
  // Act
  fireEvent.click(screen.getByRole('button', { name: /increment/i }));
  // Assert
  expect(screen.getByText('Count: 1')).toBeInTheDocument();
});

🎯 Summary

Focus on integration tests as your core test suite. They give the best ROI — test how the user actually interacts without being brittle to implementation changes.
