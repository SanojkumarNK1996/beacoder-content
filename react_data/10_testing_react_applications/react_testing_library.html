🔬 React Testing Library (RTL)

🎯 Philosophy

RTL's guiding principle: "The more your tests resemble the way your software is used, the more confidence they can give you." Query elements by accessible roles, text, and labels — not by CSS classes or component internals.

npm install -D @testing-library/react @testing-library/jest-dom @testing-library/user-event

🔧 Setup and Queries

import { render, screen } from '@testing-library/react';
import userEvent from '@testing-library/user-event';

test('shows welcome message after login', async () => {
  const user = userEvent.setup();
  render(<LoginPage />);

  await user.type(screen.getByLabelText(/email/i), 'test@example.com');
  await user.type(screen.getByLabelText(/password/i), 'password123');
  await user.click(screen.getByRole('button', { name: /login/i }));

  expect(await screen.findByText(/welcome/i)).toBeInTheDocument();
});

📌 Query Priority Order

Prefer queries in this order (most accessible to least):

  1. getByRole — accessibility roles (button, input, heading)
  2. getByLabelText — form fields with labels
  3. getByPlaceholderText
  4. getByText — text content
  5. getByTestId — last resort

🎯 Summary

RTL makes you write tests from the user's perspective. Use userEvent over fireEvent for more realistic interactions. Use findBy (async) for elements that appear after async operations.
