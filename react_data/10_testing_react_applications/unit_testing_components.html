🧪 Unit Testing React Components

🎯 What to Unit Test

  • Pure utility functions (formatDate, calculateTotal)
  • Custom hooks in isolation
  • Simple presentational components with fixed outputs
// utils.test.js
import { formatCurrency } from './utils';
test('formats number as USD', () => {
  expect(formatCurrency(15.5)).toBe('$15.50');
  expect(formatCurrency(0)).toBe('$0.00');
});

// Component test
import { render, screen } from '@testing-library/react';
import Badge from './Badge';

test('renders with correct text and color', () => {
  render(<Badge variant="success">Active</Badge>);
  const badge = screen.getByText('Active');
  expect(badge).toBeInTheDocument();
  expect(badge).toHaveClass('badge-success');
});

// Custom hook test with renderHook:
import { renderHook, act } from '@testing-library/react';
import useCounter from './useCounter';

test('increments counter', () => {
  const { result } = renderHook(() => useCounter(0));
  act(() => result.current.increment());
  expect(result.current.count).toBe(1);
});

🎯 Summary

Unit tests are fastest and most precise but test in isolation (miss integration bugs). Use them for utilities and custom hooks. Use integration tests for components with user interactions.
