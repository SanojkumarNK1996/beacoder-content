🔮 Mocking and Integration Testing

🎯 Why Mock?

Tests should be fast, deterministic, and isolated. Mock external dependencies (APIs, timers, modules) to avoid real network calls.

🔧 Mocking API Calls with MSW

Mock Service Worker (MSW) is the best approach — intercepts fetch/axios requests at the network level.

// handlers.js - define mock API responses
import { http, HttpResponse } from 'msw';
export const handlers = [
  http.get('/api/users', () => HttpResponse.json([{ id: 1, name: 'Alice' }])),
  http.post('/api/users', async ({ request }) => {
    const body = await request.json();
    return HttpResponse.json({ id: 2, ...body }, { status: 201 });
  }),
];

// Setup in tests:
import { setupServer } from 'msw/node';
const server = setupServer(...handlers);
beforeAll(() => server.listen());
afterEach(() => server.resetHandlers());
afterAll(() => server.close());

📌 Integration Test Example

test('loads and displays users', async () => {
  render(<UserList />);
  expect(screen.getByText(/loading/i)).toBeInTheDocument();
  // After API resolves:
  expect(await screen.findByText('Alice')).toBeInTheDocument();
});

🎯 Summary

MSW is the gold standard for API mocking — tests work identically with real fetch calls, the mock just intercepts network requests. More realistic than mocking modules.
