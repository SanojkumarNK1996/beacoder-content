🏗️ Three-Tier Architecture on AWS

🎯 What is Three-Tier Architecture?

The standard web application pattern: Presentation → Logic → Data tiers, each independently scalable and replaceable.

🗺️ AWS Implementation

Internet → Route 53 (DNS)
         → CloudFront (CDN)
         → ALB (Application Load Balancer)  [Public Subnet]
         → EC2 / ECS (App Servers)          [Private Subnet]
         → RDS / Aurora (Database)          [Private Subnet]
         +→ ElastiCache (Cache Layer)        [Private Subnet]

📋 Tier Responsibilities

  • Presentation: React/Angular SPA served via S3 + CloudFront. Or ALB → EC2 web tier.
  • Application: EC2 (Auto Scaling), ECS containers, or Lambda. Business logic.
  • Data: RDS/Aurora (relational), DynamoDB (NoSQL), S3 (objects), ElastiCache (cache).

🎯 Summary

Three-tier is the starting point for most web apps on AWS. Keep tiers in separate subnets, use security groups to enforce tier isolation.
