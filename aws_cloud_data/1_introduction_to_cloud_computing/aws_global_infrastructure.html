🌍 AWS Global Infrastructure

📍 Regions

AWS operates 30+ Regions worldwide (e.g., us-east-1, eu-west-1, ap-southeast-1). Each region is an independent geographic area with full AWS services. Choose your region based on latency, compliance, and cost.

🏢 Availability Zones (AZs)

Each region has 2–6 Availability Zones. An AZ is one or more discrete data centers with independent power, cooling, and networking. Design apps to span multiple AZs for high availability.

🌐 Edge Locations

AWS has 400+ edge locations globally for CloudFront CDN and Route 53 DNS. Serve cached content closer to end users for low latency.

🔗 Local Zones and Outposts

  • Local Zones: AWS compute/storage in major cities not yet a full region
  • AWS Outposts: AWS hardware in your own on-premises data center

🎯 Summary

Regions → AZs → Edge Locations is the hierarchy. Always deploy across multiple AZs for fault tolerance.
