🪣 S3 Buckets and Objects

🪣 Buckets

A bucket is a container for objects. Each bucket has a globally unique name and belongs to a specific region. Naming rules: lowercase, 3–63 chars, no underscores or uppercase.

📄 Objects

An object is a file + metadata stored in a bucket. Identified by a key (the full path in the bucket).

# S3 URI format
s3://my-bucket/folder/subfolder/file.txt
# Key = folder/subfolder/file.txt

🔒 Access Control

  • Bucket Policy: JSON policy on the bucket (recommended)
  • IAM Policies: Attach to users/roles to grant S3 access
  • ACLs: Legacy per-object access control (avoid for new setups)
  • Block Public Access: Account-level protection against accidental public exposure

🎯 Summary

Buckets organize objects. Always enable "Block Public Access" by default — make things public intentionally via bucket policy, not by accident.
