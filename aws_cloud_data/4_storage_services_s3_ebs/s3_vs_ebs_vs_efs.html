🆚 S3 vs EBS vs EFS

📋 Comparison Table

FeatureS3EBSEFS
TypeObject storageBlock storageFile storage (NFS)
Attached toAny service (API)One EC2 instanceMultiple EC2 instances
PersistencePermanentPermanentPermanent
CapacityUnlimitedUp to 64 TBAuto-scales
AccessHTTP REST APIBlock device (like HDD)POSIX file system
Best forFiles, media, backupsOS, databasesShared files across EC2

📌 Decision Guide

  • Need to store files accessed by many services → S3
  • EC2 OS disk or database storage → EBS
  • Shared file system for multiple EC2 instances (e.g., CMS, HPC) → EFS

🎯 Summary

Each storage type has its niche. Most apps use S3 + EBS combination. EFS is for workloads needing shared POSIX filesystem access.
