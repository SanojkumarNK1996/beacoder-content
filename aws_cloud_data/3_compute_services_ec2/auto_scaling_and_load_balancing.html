📈 Auto Scaling and Load Balancing

⚖️ Elastic Load Balancer (ELB)

Distributes incoming traffic across multiple EC2 instances. Types:

  • ALB (Application Load Balancer): HTTP/HTTPS, path-based routing, best for web apps and microservices
  • NLB (Network Load Balancer): TCP/UDP, ultra low latency, millions of requests/second
  • CLB (Classic Load Balancer): Legacy, not recommended for new apps

📊 EC2 Auto Scaling

Automatically adjust the number of EC2 instances based on demand.

  • Scaling out: Add instances when CPU > 70%
  • Scaling in: Remove instances when CPU < 30%
  • Launch Template: Defines instance configuration for Auto Scaling
  • Min/Max/Desired capacity: Control the fleet size bounds

🎯 Summary

ALB + Auto Scaling = highly available, elastic architecture. Your app handles any load and only pays for what it uses.
