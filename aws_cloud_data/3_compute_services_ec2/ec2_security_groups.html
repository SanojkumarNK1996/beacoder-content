🔒 EC2 Security Groups and Key Pairs

🛡️ Security Groups

A Security Group acts as a virtual firewall for your EC2 instance. It controls inbound and outbound traffic.

  • Stateful: If you allow inbound traffic in, the response is automatically allowed out
  • Default rule: Deny all inbound, allow all outbound
  • You add allow rules only (no explicit deny rules)
Common Security Group Rules:
- SSH: Port 22, Source: My IP (for admin access)
- HTTP: Port 80, Source: 0.0.0.0/0 (public web)
- HTTPS: Port 443, Source: 0.0.0.0/0 (public web)
- Custom: Port 8080, Source: Another security group (internal)

🔑 Key Pairs

Used to SSH into Linux instances or decrypt the Windows admin password.

  • AWS stores the public key on the instance
  • You download the private key (.pem) — store it securely, can't retrieve it again

🎯 Summary

Security groups are your first line of defense. Follow least privilege: only open ports you need, only to IPs that need access.
