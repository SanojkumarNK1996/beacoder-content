📊 EC2 Instance Types

🎯 Instance Families

  • General Purpose (t3, m6i): Balanced compute, memory, networking. Good for web servers, development.
  • Compute Optimized (c6i, c7g): High CPU. Good for batch processing, gaming, HPC.
  • Memory Optimized (r6i, x2idn): Large RAM. Good for in-memory databases (Redis, SAP HANA).
  • Storage Optimized (i3, d2): High I/O. Good for NoSQL databases, data warehousing.
  • Accelerated Computing (p4, g5): GPU/FPGA. Good for ML training, video rendering.

📐 Naming Convention

m6i.xlarge: m = instance family, 6 = generation, i = Intel, xlarge = size (2x = 2× resources of large)

🎯 Summary

Choose instance type based on your workload's bottleneck: CPU, memory, storage I/O, or GPU. Start with t3.micro for learning (free tier eligible).
