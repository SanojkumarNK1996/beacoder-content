🔔 Lambda Triggers and Event Sources

🎯 Lambda is Event-Driven

Lambda executes in response to events from various AWS services or direct invocations.

📋 Common Trigger Sources

  • API Gateway: HTTP requests trigger Lambda (serverless REST API)
  • S3: Object upload/delete triggers Lambda (image resizing, data processing)
  • DynamoDB Streams: React to table changes in real time
  • SQS: Process messages from a queue (decoupled architecture)
  • SNS: Fan-out notifications trigger lambda
  • EventBridge: Scheduled or event-driven triggers (cron jobs)
  • Kinesis: Real-time data stream processing
  • ALB: Load balancer target for Lambda

🎯 Summary

Lambda's power comes from its integration with the AWS ecosystem. One function can react to S3 uploads, DynamoDB changes, HTTP requests, or scheduled events — all without infrastructure management.
