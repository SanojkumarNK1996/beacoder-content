⚡ Introduction to AWS Lambda

🎯 What is Lambda?

AWS Lambda lets you run code without provisioning or managing servers. You upload code; Lambda runs it on demand and scales automatically.

  • Serverless: No servers to patch, scale, or manage
  • Pay per use: Billed for exact duration (100ms increments) × requests
  • Scales automatically: From 0 to thousands of concurrent executions in seconds

📋 Supported Runtimes

Node.js, Python, Java, Go, .NET (C#), Ruby, and custom runtimes via Lambda Layers.

📌 Lambda Limits

  • Timeout: Max 15 minutes per invocation
  • Memory: 128 MB to 10 GB
  • Package size: 50 MB zipped, 250 MB unzipped

🎯 Summary

Lambda = event-driven functions. Great for APIs, data processing, automation. Not suitable for long-running jobs (>15min) or always-on servers.
