📅 Reserved Instances and Savings Plans

🎯 Why Commit?

On-Demand pricing is flexible but expensive. By committing to 1 or 3-year usage, AWS gives significant discounts (up to 75%).

📦 Reserved Instances (RI)

  • Commit to a specific instance type in a specific region
  • Payment options: All Upfront (best discount), Partial Upfront, No Upfront
  • Use for: Steady-state EC2 (production app servers, databases)

💡 Savings Plans

More flexible than RIs — commit to an amount of compute usage ($/hour) regardless of instance type, region, or OS.

  • Compute Savings Plans: Apply to EC2, Lambda, Fargate across all regions. Most flexible.
  • EC2 Instance Savings Plans: Locked to an instance family + region. Deeper discount.

⚖️ RI vs Savings Plans

Savings Plans are generally preferred now — more flexible, apply automatically as your usage evolves.

🎯 Summary

If a resource runs 24/7 for >1 year, buy a Savings Plan or RI. Analyze with Cost Explorer's RI/Savings Plan recommendations.
