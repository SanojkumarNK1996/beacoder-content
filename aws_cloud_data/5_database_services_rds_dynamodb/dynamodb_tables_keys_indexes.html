🔑 DynamoDB Tables, Keys, and Indexes

📋 Table Structure

  • Table: Collection of items (like a database table)
  • Item: A record (like a row), up to 400 KB
  • Attribute: Data field on an item (like a column, but flexible)

🔑 Primary Key Types

  • Partition Key (Hash Key): Single unique attribute. DynamoDB hashes it to determine the partition.
  • Composite Key (Partition + Sort Key): Two-attribute key. Enables multiple items with same partition key (different sort keys). Most flexible.

📌 Indexes

  • GSI (Global Secondary Index): Query on any non-primary-key attribute. Has its own capacity units. Can project any attributes.
  • LSI (Local Secondary Index): Alternative sort key for the same partition key. Must be created at table creation time.

🎯 Summary

Design your access patterns first, then choose partition key + sort key to support them. Add GSIs for additional query patterns.
