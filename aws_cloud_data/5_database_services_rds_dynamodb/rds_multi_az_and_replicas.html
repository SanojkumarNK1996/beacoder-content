🔄 RDS Multi-AZ and Read Replicas

🏢 Multi-AZ Deployment

AWS maintains a standby replica of your database in a different AZ, synchronously replicated.

  • Automatic failover: If primary fails, DNS automatically points to standby (1–2 min)
  • Standby is NOT readable — for HA only
  • Use for production databases requiring high availability

📖 Read Replicas

Asynchronously replicated copies of the primary database. Readable, can be promoted to standalone DB.

  • Up to 15 read replicas for Aurora, 5 for MySQL/PostgreSQL
  • Can span multiple AZs or even multiple regions
  • Use to scale read-heavy workloads (reporting, analytics)

⚖️ Multi-AZ vs Read Replica

  • Multi-AZ = high availability, automatic failover, NOT readable
  • Read Replica = scale reads, eventual consistency, manually fail over

🎯 Summary

Use Multi-AZ for production HA. Add Read Replicas to offload reporting queries and reduce load on the primary.
