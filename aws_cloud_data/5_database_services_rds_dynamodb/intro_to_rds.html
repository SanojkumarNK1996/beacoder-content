🗄️ Introduction to AWS RDS

🎯 What is RDS?

Amazon RDS (Relational Database Service) is a managed relational database service. AWS handles provisioning, patching, backups, monitoring, and scaling — you focus on your database and schema.

📋 Key Features

  • Automated backups: Point-in-time recovery up to 35 days
  • Database snapshots: Manual backups retained indefinitely
  • Multi-AZ deployment: Automatic failover for high availability
  • Read Replicas: Scale read traffic across copies
  • Encryption at rest and in transit

📌 When to Use RDS

  • You need a relational data model (tables, joins, ACID transactions)
  • You want managed infrastructure (no DBA needed for patching/backups)
  • Workload fits traditional relational patterns

🎯 Summary

RDS removes the heavy lifting of running relational databases. Choose it over self-managing on EC2 for most standard relational workloads.
