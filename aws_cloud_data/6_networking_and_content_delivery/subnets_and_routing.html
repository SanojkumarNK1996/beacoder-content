🔀 Subnets, Route Tables, and Internet Gateway

📌 Public vs Private Subnets

  • Public Subnet: Has a route to the Internet Gateway. Resources can have public IPs. (Web servers, bastion hosts)
  • Private Subnet: No direct internet route. (Databases, app servers) — outbound internet via NAT Gateway

🗺️ Route Tables

Each subnet is associated with a route table. Rules determine where traffic goes.

Public Route Table routes:
  10.0.0.0/16  →  local (within VPC)
  0.0.0.0/0    →  igw-xxxxx (Internet Gateway)

Private Route Table routes:
  10.0.0.0/16  →  local
  0.0.0.0/0    →  nat-xxxxx (NAT Gateway)

🌐 Internet Gateway (IGW)

Allows VPC resources to communicate with the internet. Attach one IGW per VPC.

🔒 NAT Gateway

Allows private subnet resources to initiate outbound internet traffic (e.g., download updates) without being reachable from the internet. Deployed in a public subnet.

🎯 Summary

Standard pattern: public subnets for load balancers, private subnets for app servers and databases. NAT Gateway for private subnet outbound access.
