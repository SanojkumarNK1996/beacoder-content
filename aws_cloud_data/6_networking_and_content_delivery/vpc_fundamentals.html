🌐 AWS VPC Fundamentals

🎯 What is a VPC?

A Virtual Private Cloud (VPC) is your own private network in AWS. Logically isolated section of the AWS cloud where you launch resources with full control over networking: IP ranges, subnets, route tables, gateways.

📋 Key Concepts

  • CIDR Block: IP range for your VPC (e.g., 10.0.0.0/16 = 65,536 IPs)
  • Default VPC: Each region has a pre-created default VPC (10.0.0.0/16 equiv.)
  • Tenancy: Shared (default) or Dedicated (single-tenant hardware)

🔧 VPC Components

  • Subnets (public/private)
  • Route Tables
  • Internet Gateway
  • NAT Gateway
  • Security Groups (stateful firewall)
  • Network ACLs (stateless firewall at subnet level)

🎯 Summary

Every AWS resource lives in a VPC. Master VPC networking — it underpins every production architecture.
