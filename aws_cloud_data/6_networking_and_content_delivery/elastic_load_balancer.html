⚖️ Elastic Load Balancer

🎯 What is ELB?

Elastic Load Balancing (ELB) automatically distributes incoming application traffic across multiple targets (EC2 instances, containers, IPs, Lambda) in one or more AZs.

📋 ELB Types

  • ALB (Application Load Balancer): Layer 7 (HTTP/HTTPS). Path-based and host-based routing. Best for microservices and web apps.
  • NLB (Network Load Balancer): Layer 4 (TCP/UDP/TLS). Ultra-low latency, millions of RPS. Best for real-time apps, WebSockets, VoIP.
  • GLB (Gateway Load Balancer): Deploy third-party virtual network appliances (firewalls, IDS/IPS).

🔧 Key Features

  • Health checks on targets
  • SSL/TLS termination
  • Stickiness (session affinity)
  • Cross-zone load balancing

🎯 Summary

Use ALB for HTTP/HTTPS web apps. Use NLB when you need extreme performance or non-HTTP protocols. Always use ELB with Auto Scaling for elastic, HA architectures.
