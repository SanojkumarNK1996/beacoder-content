✅ AWS Config and Compliance

🎯 What is AWS Config?

AWS Config continuously tracks the configuration of your AWS resources and evaluates them against desired configurations. It answers: "was this S3 bucket public at any time in the last 30 days?"

📋 Key Concepts

  • Configuration Item: Snapshot of a resource's configuration at a point in time
  • Configuration History: Timeline of all changes to a resource
  • Config Rules: Evaluate resources against policies (e.g., "all S3 buckets must have encryption enabled")
  • Conformance Packs: Collections of Config rules for compliance frameworks (PCI DSS, HIPAA, CIS)

📌 Config vs CloudTrail

  • Config: WHAT changed (resource state history)
  • CloudTrail: WHO changed it (API audit log)
  • Together: Complete change audit

🎯 Summary

Enable AWS Config for continuous compliance monitoring. Use Config Rules to auto-detect misconfigurations before they become security incidents.
