🔍 AWS CloudTrail

🎯 What is CloudTrail?

AWS CloudTrail records all API calls made in your AWS account — who did what, from where, and when. It's your audit log for governance, compliance, and security analysis.

📋 What CloudTrail Records

  • Every AWS API call (console, CLI, SDK): CreateBucket, RunInstances, PutObject
  • Who made the call (IAM user, role, root)
  • Source IP address
  • Timestamp
  • Request parameters and response

📌 CloudTrail vs CloudWatch

  • CloudTrail: WHO did WHAT to which resource (API audit log)
  • CloudWatch: HOW IS the resource performing (metrics and logs)

🔒 Best Practices

  • Enable CloudTrail in ALL regions
  • Store logs in S3 with integrity validation
  • Set CloudTrail log alerts for sensitive actions (root login, IAM changes)

🎯 Summary

CloudTrail is mandatory for security compliance. Enable it immediately after account creation — it answers "who deleted that bucket?"
