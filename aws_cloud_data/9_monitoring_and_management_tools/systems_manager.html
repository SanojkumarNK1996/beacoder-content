🔧 AWS Systems Manager

🎯 What is Systems Manager?

AWS Systems Manager (SSM) gives you operational visibility and control over AWS infrastructure. Manage EC2 and on-premises servers at scale without SSH.

📋 Key SSM Features

  • Session Manager: Browser-based or CLI SSH to EC2 without opening port 22. No key pairs needed. All sessions logged to CloudWatch/S3.
  • Parameter Store: Centralized config store for strings, secrets, encrypted values (free tier for standard parameters)
  • Run Command: Execute scripts across a fleet of instances simultaneously without SSH
  • Patch Manager: Automate OS patching across EC2 fleet
  • Inventory: Collect metadata about installed software, OS, configuration

🔒 Session Manager vs SSH

Session Manager is preferred for production: no bastion host needed, no open ports, full audit trail of all commands.

🎯 Summary

SSM replaces SSH for EC2 management. Use Parameter Store for app config, Session Manager for secure server access, Run Command for fleet-wide automation.
