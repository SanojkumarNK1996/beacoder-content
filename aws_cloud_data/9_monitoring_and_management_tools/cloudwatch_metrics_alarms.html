📈 CloudWatch Metrics, Alarms, and Logs

📊 Metrics

A metric is a time-ordered set of data points (e.g., EC2 CPUUtilization measured every 5 minutes). Namespace organizes metrics: AWS/EC2, AWS/RDS, etc.

🔔 Alarms

Set alarms on metrics to trigger notifications or automated actions.

Example Alarm:
- Metric: EC2 CPUUtilization
- Threshold: > 80% for 5 consecutive minutes
- Action: Send SNS notification → Email alert
- Action: EC2 Auto Scaling → Add instances

📝 CloudWatch Logs

  • Log Group: Container for log streams (e.g., /aws/lambda/my-function)
  • Log Stream: Sequence of log events from one source
  • Log Insights: Query language to search and analyze logs
  • Metric Filters: Extract metrics from log data (count ERROR occurrences)

🎯 Summary

Set billing alarms first (avoid surprises). Then add CPU, error rate, and latency alarms for all production services.
