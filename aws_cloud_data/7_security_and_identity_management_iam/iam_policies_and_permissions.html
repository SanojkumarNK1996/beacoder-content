📋 IAM Policies and Permissions

🎯 What is an IAM Policy?

An IAM policy is a JSON document defining permissions. It specifies what actions are allowed or denied on which resources under what conditions.

{
  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Allow",
    "Action": ["s3:GetObject", "s3:PutObject"],
    "Resource": "arn:aws:s3:::my-bucket/*"
  }]
}

📋 Policy Types

  • AWS Managed Policies: Pre-built by AWS (e.g., AmazonS3ReadOnlyAccess)
  • Customer Managed Policies: You create and manage them
  • Inline Policies: Embedded directly in a user/role (not reusable — avoid)

🔒 Policy Evaluation Logic

Explicit Deny ALWAYS wins. Then Allow if a matching allow policy exists. Default is Implicit Deny for everything.

🎯 Summary

Master IAM policy JSON. Use AWS Managed Policies for common access patterns, Customer Managed for fine-grained control. Explicit deny overrides everything.
