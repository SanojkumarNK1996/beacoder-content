🔐 AWS KMS and Secrets Manager

🔑 AWS KMS (Key Management Service)

Create, manage, and control encryption keys used to encrypt your data. Fully managed, integrated with S3, EBS, RDS, DynamoDB, Lambda, and more.

  • CMK (Customer Master Key): Your encryption key, managed by you in KMS
  • Envelope encryption: KMS encrypts a data key, which then encrypts your data
  • All key usage logged in AWS CloudTrail for audit

🔒 AWS Secrets Manager

Securely store, rotate, and retrieve secrets (database passwords, API keys, OAuth tokens).

  • Automatic rotation: Rotate RDS credentials automatically without touching your code
  • Application retrieves secrets via API at runtime — never hardcoded
  • Integrates with IAM for fine-grained access control

📌 Secrets Manager vs Parameter Store

Secrets Manager: For secrets (auto-rotation, higher cost). AWS Systems Manager Parameter Store: For config values and less sensitive data (cheaper, no auto-rotation).

🎯 Summary

Never store secrets in code or environment variables. Use Secrets Manager for credentials, KMS for encryption at rest.
