🔐 Introduction to IAM

🎯 What is IAM?

AWS IAM (Identity and Access Management) lets you control who (authentication) can do what (authorization) on AWS resources. It's the security foundation of every AWS account.

📋 IAM is Global

IAM is not region-specific — users, roles, and policies apply globally across all regions in your account.

🔑 IAM Principles

  • Least Privilege: Grant only the permissions needed, nothing more
  • Zero Trust: Verify every access explicitly; never assume trust

📌 Root vs IAM User

  • Root user: Full unrestricted access. Never use for daily tasks. Enable MFA immediately.
  • IAM users: Named identities with specific permissions. Use for all regular operations.

🎯 Summary

IAM is the gatekeeper to all AWS services. Mastering IAM = mastering AWS security. Everything else builds on it.
