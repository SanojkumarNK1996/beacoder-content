🔀 Conditional Statements in Java

1. 🎯 Why This Exists (Indian Real-Life Example)

  • In a UPI payment:
    • If PIN is correct → payment succeeds.
    • If PIN is wrong → show error message.
  • In IRCTC booking:
    • If seats are available → confirm ticket.
    • Else → show waiting list / RAC.
  • These "if this happens, do that" rules in real life are written in Java using conditional statements.

2. 📖 What Are Conditional Statements?

  • They let your program take decisions.
  • Based on a condition (true/false), Java chooses which block of code to execute.
  • Main types: if, if-else, if-else-if, and switch.

3. 🧾 Syntax / Basic Structure ⭐

// if
if (condition) {
    // code runs if condition is true
}

// if-else
if (condition) {
    // true block
} else {
    // false block
}

// if-else-if ladder
if (cond1) {
    
} else if (cond2) {
    
} else {
    
}

// switch
switch (expression) {
    case value1:
        // code
        break;
    case value2:
        // code
        break;
    default:
        // code
}

4. 🔍 How It Works (Step-by-Step Execution) ⭐

public class ConditionalDemo {
    public static void main(String[] args) {
        int amount = 600;          // 1

        if (amount >= 500) {      // 2
            System.out.println("Free delivery");
        } else {                  // 3
            System.out.println("Delivery charge: 40");
        }

        int day = 3;              // 4
        switch (day) {            // 5
            case 1:
                System.out.println("Monday");
                break;
            case 2:
                System.out.println("Tuesday");
                break;
            case 3:
                System.out.println("Wednesday");
                break;
            default:
                System.out.println("Invalid day");
        }
    }
}
  1. Line 3 – We set amount to 600.
  2. Lines 5–9 – If amount ≥ 500, print "Free delivery"; otherwise print delivery charge.
  3. Line 11 – day is 3.
  4. Lines 12–24 – switch checks each case until it matches 3 and prints "Wednesday".

5. ✅ When to Use Which

  • Use simple if when you only care about the true case.
  • Use if-else when you want exactly one of two paths.
  • Use if-else-if ladder when checking ranges or multiple conditions (marks, age groups, etc.).
  • Use switch when comparing one variable against many fixed values (menu options, days, choices).

6. ⛔ When NOT to Use / Things to Avoid

  • Don’t write very long if-else-if chains when switch is clearer.
  • Don’t put complex calculations directly inside conditions; it reduces readability.
  • Avoid overlapping conditions in an if-else-if ladder.

7. 🧠 Minimum You MUST Remember

  • Conditions should result in boolean (true/false).
  • Only one block in an if-else-if ladder executes.
  • switch compares one value with many cases; use break to stop falling through.

8. ⚠️ Common Mistakes

  • Using = instead of == inside conditions.
  • Forgetting break in switch, causing multiple cases to execute.
  • Writing conditions like if (marks = 40) instead of if (marks == 40).
