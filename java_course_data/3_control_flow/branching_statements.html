🔁 Branching Statements in Java

1. 🎯 Why This Exists (Indian Real-Life Example)

  • Imagine checking tokens in a bank queue:
    • If token is "Lunch Break" → stop serving (like break).
    • If token belongs to another counter → skip and move to next (like continue).
    • When work is done → go home (exit the system, like return from method).
  • Branching statements help change the normal flow of execution like this.

2. 📖 What Are Branching Statements?

  • They are statements that change the normal flow of a program.
  • Main ones in Java:
    • break – exit loop/switch.
    • continue – skip current loop iteration.
    • return – exit a method (optionally with a value).

3. 🧾 Syntax / Basic Structure ⭐

// break inside loop or switch
break;

// continue inside loop
continue;

// return in a method
return;        // in void method
return value;  // in non-void method

4. 🔍 How It Works (Step-by-Step Execution) ⭐

break Example

for (int i = 1; i <= 10; i++) {
    if (i == 5) {
        break;              // 1
    }
    System.out.println(i); // 2
}
  1. When i becomes 5, break executes and loop stops.
  2. Output: 1 2 3 4.

continue Example

for (int i = 1; i <= 5; i++) {
    if (i == 3) {
        continue;           // 1
    }
    System.out.println(i); // 2
}
  1. When i == 3, continue skips that iteration.
  2. Output: 1 2 4 5 (3 is skipped).

return Example

public class ReturnDemo {
    static int add(int a, int b) {
        return a + b;   // 1
    }

    public static void main(String[] args) {
        int sum = add(5, 3);  // 2
        System.out.println(sum);
    }
}
  1. return a + b; sends result back and exits the method.
  2. main method receives the value and prints it.

5. ✅ When to Use Each

  • break – when you want to stop a loop early (e.g., found required element).
  • continue – when you want to skip a particular case but still continue loop.
  • return – when method has finished its work and should go back to caller.

6. ⛔ When NOT to Use / Be Careful

  • Don’t overuse break and continue; they can make loops harder to understand.
  • Avoid using return from multiple places in long methods unless really needed.

7. 🧠 Minimum You MUST Remember

  • break – exits loop/switch completely.
  • continue – jumps to next loop iteration.
  • return – exits method and can send a value back.

8. ⚠️ Common Mistakes

  • Using break outside a loop or switch (compile-time error).
  • Forgetting that continue does not exit the loop, only skips one iteration.
  • Not returning a value from all paths in a non-void method.

  1. Very Easy: Write a loop from 1 to 10 and stop when number becomes 7 using break.
