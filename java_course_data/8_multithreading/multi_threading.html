🧵 Multithreading in Java (Beginner View)

1. 🎯 Why This Exists (Indian Real-Life Example)

  • In a Swiggy app:
    • One part updates live order status.
    • Another part listens for new offers.
    • Another part tracks your GPS location.
  • All these run together, not one after another slowly.
  • Similarly, in Java, multithreading lets your program do multiple tasks in parallel (or seemingly parallel).

2. 📖 What Is Multithreading?

  • A thread is a small unit of execution inside a program.
  • Multithreading means a Java program can run many threads at the same time.
  • Each thread can run its own task, but all share the same memory of the program.

3. 🧾 Syntax / Basic Structure ⭐

Two common ways to create a thread:

// 1) Extending Thread class
class MyThread extends Thread {
    @Override
    public void run() {
        System.out.println("Running in: " + Thread.currentThread().getName());
    }
}

// 2) Implementing Runnable interface
class MyTask implements Runnable {
    @Override
    public void run() {
        System.out.println("Task in: " + Thread.currentThread().getName());
    }
}

4. 🔍 How It Works (Step-by-Step Execution) ⭐

public class ThreadDemo {
    public static void main(String[] args) {
        // Using Thread subclass
        MyThread t1 = new MyThread();        // 1
        t1.setName("Worker-1");             // 2

        // Using Runnable
        Thread t2 = new Thread(new MyTask()); // 3
        t2.setName("Worker-2");             // 4

        t1.start(); // 5
        t2.start(); // 6

        System.out.println("Main thread: " + Thread.currentThread().getName()); // 7
    }
}
  1. Create a thread object t1 from MyThread.
  2. Give it a name "Worker-1" (for easy identification in logs).
  3. Create another thread t2 using new Thread(new MyTask()).
  4. Name it "Worker-2".
  5. Calling t1.start() asks JVM to start a new thread, which then calls run() internally.
  6. t2.start() does the same for the second thread.
  7. The main method itself runs in a separate thread called main.

5. ✅ When to Use Multithreading

  • When tasks are independent and can run in parallel (downloading files, processing orders).
  • When you want to keep UI or main flow responsive while doing background work.
  • When you need to handle many users/requests at the same time (server-side Java).

6. ⛔ When NOT to Use / Be Careful

  • For very simple console programs – multithreading may only add confusion.
  • When you are not handling shared data carefully → can cause wrong results.
  • Too many threads can slow down the system instead of speeding it up.

7. 🧠 Minimum You MUST Remember

  • Override run(), but start a thread using start(), not run() directly.
  • Shared data must be protected (e.g., using synchronized).
  • Threads share memory → powerful but risky if misused.

8. ⚠️ Common Mistakes

  • Calling run() instead of start() – then it behaves like a normal method, not a thread.
  • Updating shared variables from multiple threads without synchronization → inconsistent results.
  • Ignoring exceptions in threads, making bugs hard to see.

9. 🎤 Interviewer’s Perspective

  • "What is a thread?"

  • "Why do we call start(), not run()?"
  • "What is synchronization? Why is it needed?" (for slightly advanced fresher).

10. 📝 Practice Questions ⭐

  1. Very Easy:

    Create a thread that prints numbers 1 to 5 with the thread name.

  2. Easy:

    Create two threads that both print their names 3 times. Observe interleaving output.

  3. Medium:

    Create a shared Counter class with a count variable and a synchronized increment() method. Run 2 threads, each incrementing 1000 times, and verify the final count.
