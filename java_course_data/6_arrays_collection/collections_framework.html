📦 Collections Framework (Lists, Sets, Maps)

1. 🎯 Why This Exists (Indian Real-Life Example)

  • Swiggy/Zomato:
    • List of restaurants, list of items in cart, set of cuisines, map of item → price.
  • IRCTC:
    • List of trains, set of stations, map of PNR → booking details.
  • Arrays are fixed size; real apps need growable, flexible structures → Java Collections Framework.

2. 📖 What Is the Collections Framework?

  • A set of interfaces and classes to store and manage groups of objects.
  • Main groups:
    • List – ordered, allows duplicates (e.g., ArrayList).
    • Set – unique elements, no duplicates (e.g., HashSet).
    • Map – key–value pairs (e.g., HashMap).

3. 🧾 Syntax / Basic Structure ⭐

// List (ArrayList)
import java.util.ArrayList;

ArrayList<String> names = new ArrayList<>();
names.add("Rahul");
names.add("Anjali");

// Set (HashSet)
import java.util.HashSet;

HashSet<String> cities = new HashSet<>();
cities.add("Kochi");
cities.add("Kochi"); // duplicate ignored

// Map (HashMap)
import java.util.HashMap;

HashMap<String, Integer> marks = new HashMap<>();
marks.put("Maths", 90);
marks.put("Science", 85);

4. 🔍 How It Works (Step-by-Step Execution) ⭐

import java.util.*;

public class CollectionsDemo {
    public static void main(String[] args) {
        // 1. List example
        List<String> cart = new ArrayList<>();
        cart.add("Pizza");
        cart.add("Burger");
        cart.add("Pizza");          // duplicates allowed

        System.out.println(cart);    // [Pizza, Burger, Pizza]

        // 2. Set example
        Set<String> uniqueCities = new HashSet<>();
        uniqueCities.add("Kochi");
        uniqueCities.add("Kochi");  // ignored
        uniqueCities.add("Chennai");

        System.out.println(uniqueCities); // order not guaranteed

        // 3. Map example
        Map<String, Integer> itemToPrice = new HashMap<>();
        itemToPrice.put("Pizza", 250);
        itemToPrice.put("Burger", 150);

        System.out.println(itemToPrice.get("Pizza")); // 250
    }
}
  1. List keeps insertion order and allows duplicates.
  2. Set stores only unique values; duplicates are ignored.
  3. Map links keys (like item name) to values (like price).

5. ✅ When to Use Which

  • List – when order matters and duplicates are allowed (cart items, message history).
  • Set – when you need unique values (unique phone numbers, unique cities).
  • Map – when you need key → value lookup (userId → details, PNR → booking).

6. ⛔ When NOT to Use / Caution

  • Don’t use a List when you require uniqueness – use a Set instead.
  • Don’t use a Map when you only need a simple list.
  • Don’t choose a collection blindly – think about order, duplicates, lookup needs.

7. 🧠 Minimum You MUST Remember

  • List → ordered, duplicates allowed.
  • Set → unordered (mostly), unique elements.
  • Map → key–value pairs.
  • Use generics: List<String>, Set<Integer>, Map<String, Integer>.

8. ⚠️ Common Mistakes

  • Forgetting to use generics (List<String>) and ending up with Object casting issues.
  • Expecting HashSet or HashMap to maintain insertion order (they don’t).
  • Using wrong key type in Map (e.g., using String when an Integer ID is better).
