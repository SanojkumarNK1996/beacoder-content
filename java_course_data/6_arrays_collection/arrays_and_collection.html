📚 Arrays in Java

1. 🎯 Why This Exists (Indian Real-Life Example)

  • In a college marks system, you store marks of 5 subjects for 1 student.
  • In IRCTC, you store all seat numbers in a coach.
  • In Swiggy, you store all item prices in the current cart.
  • Instead of creating marks1, marks2, marks3..., Java gives arrays to store multiple values of the same type together.

2. 📖 What Is an Array?

  • An array is a fixed-size, ordered collection of elements of the same data type.
  • Each element is stored at an index (position) starting from 0.
  • Think of it like a row of boxes in memory: each box holds one value, and you can access it by position.

3. 🧾 Syntax / Basic Structure ⭐

Declaration + Creation + Initialization

// 1. Declare
int[] marks;

// 2. Create array of size 5
marks = new int[5];

// 3. Assign values
marks[0] = 85;
marks[1] = 90;

// Short form (declare + create + initialize)
int[] scores = {85, 90, 78, 92};

Accessing elements

System.out.println(scores[0]);      // first element
System.out.println(scores[3]);      // fourth element
System.out.println(scores.length);  // size of array

4. 🔍 How It Works (Step-by-Step Execution) ⭐

public class ArrayDemo {
    public static void main(String[] args) {
        int[] marks = {80, 85, 90};   // 1

        System.out.println(marks[0]); // 2

        marks[1] = 88;                // 3

        for (int i = 0; i < marks.length; i++) { // 4
            System.out.println("Subject " + (i + 1) + ": " + marks[i]);
        }
    }
}
  1. Line 3 – Creates an array of size 3 with values 80, 85, 90.
  2. Line 5 – Accesses element at index 0 → prints 80.
  3. Line 7 – Updates element at index 1 from 85 to 88.
  4. Lines 9–11 – Loop from 0 to marks.length - 1 and print each value.

5. ✅ When to Use Arrays

  • When the number of items is fixed (e.g., 12 months, 7 days, 5 subjects).
  • When all elements are of the same type (all int, all double, etc.).
  • When you need fast index-based access, like marks[2].

6. ⛔ When NOT to Use Arrays

  • When the number of elements changes frequently → prefer collections like ArrayList.
  • When you need to insert/delete in the middle often – arrays are costly to resize.
  • When you need to store different types together (e.g., name + age + salary) – use a class instead.

7. 🧠 Minimum You MUST Remember

  • Indexes start from 0 and go till length - 1.
  • Syntax: type[] name = new type[size]; or type[] name = {values};.
  • array.length gives the total number of elements.
  • Arrays are fixed size once created.

8. ⚠️ Common Mistakes

  • Index out of bounds: accessing marks[3] when size is 3 → ArrayIndexOutOfBoundsException.
  • Confusing length (for arrays) with length() (for strings).
  • Forgetting that all elements get default values if not set (e.g., 0 for int, null for objects).
