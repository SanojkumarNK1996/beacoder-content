🔁 Iterating Collections in Java

1. 🎯 Why This Exists (Indian Real-Life Example)

  • In Swiggy, to show your cart, app must go through each item and print name, price, quantity.
  • In IRCTC, to show all available trains between two stations, it must loop through a list of trains.
  • In Java, this "going through each element" is called iteration.

2. 📖 What Is Iteration?

  • Iteration means visiting each element one by one in a collection or array.
  • Java provides several ways: for loop, for-each loop, and Iterator.

3. 🧾 Syntax / Basic Structure ⭐

For loop (index-based)

for (int i = 0; i < list.size(); i++) {
    System.out.println(list.get(i));
}

Enhanced for-each loop

for (Type element : collection) {
    System.out.println(element);
}

Using Iterator

Iterator<Type> it = collection.iterator();
while (it.hasNext()) {
    Type value = it.next();
}

4. 🔍 How It Works (Step-by-Step Execution) ⭐

import java.util.*;

public class IterationDemo {
    public static void main(String[] args) {
        List<String> cart = new ArrayList<>();
        cart.add("Pizza");
        cart.add("Burger");
        cart.add("Juice");

        // 1. For loop
        for (int i = 0; i < cart.size(); i++) {
            System.out.println("Item " + i + ": " + cart.get(i));
        }

        // 2. For-each
        for (String item : cart) {
            System.out.println("For-each: " + item);
        }

        // 3. Iterator
        Iterator<String> it = cart.iterator();
        while (it.hasNext()) {
            String item = it.next();
            System.out.println("Iterator: " + item);
        }
    }
}
  1. For loop – uses index to access each element.
  2. For-each – automatically picks each element, easier to read.
  3. Iterator – uses hasNext() and next() to traverse.

5. ✅ When to Use Which

  • For loop – when you need the index (position), or to modify elements by index.
  • For-each loop – when you only need to read each element; simplest for beginners.
  • Iterator – when you may need to remove elements while iterating.

6. ⛔ When NOT to Use / Be Careful

  • Don’t modify a collection (add/remove) inside a for-each loop directly – may cause ConcurrentModificationException.
  • Don’t ignore hasNext() with Iterator – calling next() without it can throw an exception.

7. 🧠 Minimum You MUST Remember

  • For-each syntax: for (Type var : collection).
  • Iterator has two main methods: hasNext() and next().
  • Use Iterator when removing elements during iteration.

8. ⚠️ Common Mistakes

  • Using wrong loop condition (e.g., i <= list.size()) causing IndexOutOfBoundsException.
  • Modifying list size inside a normal for loop without adjusting the index.
  • Forgetting that Set/Map do not have direct index access like list.get(i).
