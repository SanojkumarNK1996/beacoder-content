📝 Strings in Java

1. 🎯 Why This Exists (Indian Real-Life Example)

  • Every real app uses text: names, cities, messages, OTPs.
  • IRCTC: passenger name, train name, source, destination.
  • Swiggy: restaurant name, item names, delivery address.
  • All this text is handled in Java using String.

2. 📖 What Is a String?

  • A String is a sequence of characters (text) like "Kochi", "Hello", "UPI123".
  • In Java, String is a class, not a primitive type.
  • Strings are immutable – once created, their value cannot be changed; operations create new objects.

3. 🧾 Syntax / Basic Structure ⭐

// Two common ways to create strings
String city = "Bengaluru";              // string literal
String greeting = new String("Hello"); // using constructor

Basic string operations

String text = "Hello World";

int len = text.length();                 // length of string
String upper = text.toUpperCase();       // HELLO WORLD
char ch = text.charAt(1);                // 'e'
String part = text.substring(0, 5);      // "Hello"
boolean hasWorld = text.contains("World");

4. 🔍 How It Works (Step-by-Step Execution) ⭐

public class StringDemo {
    public static void main(String[] args) {
        String name = "Rahul";                 // 1
        System.out.println(name);               // 2

        String welcome = "Welcome " + name;    // 3
        System.out.println(welcome);            // 4

        System.out.println(name.length());      // 5
        System.out.println(name.toUpperCase()); // 6
    }
}
  1. Line 3 – A String object with value "Rahul" is created and name refers to it.
  2. Line 4 – Prints "Rahul".
  3. Line 6 – "Welcome " + name creates a new string "Welcome Rahul".
  4. Line 7 – Prints "Welcome Rahul".
  5. Line 9 – length() returns number of characters in "Rahul".
  6. Line 10 – toUpperCase() returns "RAHUL" (original name is unchanged).

5. ✅ When to Use Strings

  • Whenever you work with text: names, messages, IDs, codes.
  • For building messages to show users or logs.
  • For processing input from the user (console, forms, files).

6. ⛔ When NOT to Use Plain String

  • When doing heavy modifications in a loop (e.g., concatenating in large loops) – prefer StringBuilder.
  • When you need binary data (images, PDFs) – use byte arrays/streams, not String.

7. 🧠 Minimum You MUST Remember

  • Strings are immutable.
  • Use equals() to compare string values, not ==.
  • Common methods: length(), toUpperCase(), toLowerCase(), substring(), contains(), replace().

8. ⚠️ Common Mistakes

  • Using == for comparison: if(name == "Rahul") – may fail; use name.equals("Rahul").
  • Ignoring case when comparing ("rahul" vs "Rahul"); consider equalsIgnoreCase().
  • Assuming toUpperCase() changes original string – it returns a new one.
