📦 Variables in Java

1. 🎯 Why This Exists (Indian Real-Life Example)

  • Think of an ATM transaction:
    • The machine needs to hold your account balance, withdrawal amount, remaining balance etc.
  • Or a Swiggy order:
    • It keeps track of item price, delivery fee, discount, total bill.
  • In Java, these changing pieces of information are stored using variables.

2. 📖 What Is a Variable?

  • A variable is a named container in memory that stores a value.
  • You can read it, change it, and use it in calculations.
  • Example idea: "age of user", "totalBill", "upiPinValid" – all are good candidates for variables.

3. 🧾 Syntax / Basic Structure ⭐

General form:

dataType variableName = value;   // declaration + initialization

Examples:

int age = 25;
double billAmount = 499.50;
String customerName = "Rahul";
  • dataType – type of data (int, double, String, etc.).
  • variableName – name you choose.
  • value – initial data stored.

4. 🔍 How It Works (Step-by-Step Execution) ⭐

public class VariableDemo {
  public static void main(String[] args) {
    int age = 22;              // 1
    double balance = 1500.75;  // 2
    String name = "Anjali";   // 3

    System.out.println(name + " is " + age + " years old."); // 4
    System.out.println("Balance: " + balance);                  // 5
  }
}
  1. Line 3 – Java creates a box named age to store an integer (22).
  2. Line 4 – Creates balance to store 1500.75 (a decimal number).
  3. Line 5 – Creates name to store the text "Anjali".
  4. Line 7 – Combines name and age into a message and prints it.
  5. Line 8 – Prints the account balance.

5. ✅ When to Use Variables

  • Whenever a value can change during program execution (e.g., remaining balance, count of items).
  • To store inputs from user forms, ATM screens, or console.
  • To hold intermediate results in calculations (e.g., discount, GST, total amount).

6. ⛔ When NOT to Use a Normal Variable

  • For values that should never change (like GST rate, constant PI) – use constants instead (with final).
  • When the data is shared across many classes and must stay consistent – consider proper design (e.g., configuration or constants class).

7. 🧠 Minimum You MUST Remember

  • Syntax: dataType variableName = value;
  • Good names use camelCase and are meaningful (e.g., accountBalance).
  • You must declare a variable before using it.
  • Variables are case-sensitive: age ≠ Age.

8. ⚠️ Common Mistakes

  • Using confusing names like a, b, c everywhere.
  • Trying to use a variable before declaring it.
  • Mismatching data type and value (e.g., int age = 20.5;).
  • Starting variable names with numbers (e.g., 1age – not allowed).
