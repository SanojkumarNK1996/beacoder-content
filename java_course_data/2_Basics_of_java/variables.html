Variables in Java

In Java, a variable is a name given to a memory location that stores a value. Think of it like a container where you can put data, use it, and even change it later in your program.

1. What Is a Variable?

Every program works with data — numbers, text, or other information. A variable helps you store and manage this data while your program runs.

In simple words, a variable is like a label for a value. For example, if you want to store someone’s age, you can create a variable named age and assign it a value.

int age = 25;

Here:

  • int → the data type (integer)
  • age → the variable name
  • 25 → the value stored

2. Declaration and Initialization

You must declare a variable before using it. Declaration means telling Java what type of data the variable will hold.

Declaration Example:

int number; // declares a variable named number of type int

To give the variable an initial value, you initialize it.

Initialization Example:

number = 10; // assigns a value to the variable

You can also do both in a single line:

int number = 10; // declare and initialize together

Multiple Variables:

You can declare and initialize multiple variables of the same type in one line:

int x = 5, y = 10, z = 15;

Example Program:

public class VariableExample {
    public static void main(String[] args) {
        int age = 25;
        double salary = 55000.75;
        String name = "John";

        System.out.println("Name: " + name);
        System.out.println("Age: " + age);
        System.out.println("Salary: " + salary);
    }
}

3. Naming Conventions

Choosing good variable names makes your code easy to read and understand. Java has some naming rules and best practices you should follow.

Rules (Must Follow):

  • Variable names can contain letters, numbers, _, and $.
  • They cannot start with a number.
  • They are case-sensitive — age and Age are different.
  • No spaces or special characters are allowed.
  • They cannot be a Java keyword (like class or public).

Conventions (Good Practices):

  • Use camelCase for variable names — e.g., studentName, totalMarks.
  • Use meaningful names that describe the data — not short or random letters.
  • Use UPPER_CASE for constants — e.g., PI, MAX_SPEED.
  • Avoid single-character names unless used for short loops (like i, j).

Examples:

// Good variable names
int studentAge = 20;
double accountBalance = 1050.75;
String fullName = "Alice Johnson";

// Bad variable names
int a = 20;
double xyz = 10.5;
String n = "Bob";

4. Summary

  • A variable stores data that can change during program execution.
  • You must declare it with a type before use.
  • Use clear and meaningful names following Java’s naming conventions.

Understanding variables is a basic but very important step — every Java program you write will use them!
