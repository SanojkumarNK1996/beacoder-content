➕ Operators in Java

1. 🎯 Why This Exists (Indian Real-Life Example)

  • In a shopping app (like Flipkart):
    • Add item prices → total amount (arithmetic).
    • Check if total > 500 → free delivery (relational).
    • Check if user is Prime OR cart > 1000 (logical).
  • Java operators allow you to calculate, compare, and decide in code, similar to these business rules.

2. 📖 What Are Operators?

  • Operators are symbols that tell Java to perform a specific operation.
  • Example: + (add), > (compare), && (AND condition).
  • They work with variables and values to produce new results.

3. 🧾 Syntax / Basic Structure ⭐

int a = 10;
int b = 5;

int sum = a + b;          // arithmetic
boolean isGreater = a > b; // relational
boolean result = (a > 5) && (b < 10); // logical

4. 🔍 How It Works (Step-by-Step Execution) ⭐

Example Program Using Different Operators

public class OperatorDemo {
    public static void main(String[] args) {
        int itemPrice = 450;              // 1
        int deliveryCharge = 50;          // 2

        int total = itemPrice + deliveryCharge;   // 3 (arithmetic)
        boolean freeDelivery = total > 500;       // 4 (relational)

        boolean isPrimeUser = true;               // 5
        boolean finalFreeDelivery = freeDelivery || isPrimeUser; // 6 (logical)

        System.out.println("Total: " + total);                 // 7
        System.out.println("Free Delivery? " + finalFreeDelivery); // 8
    }
}
  1. Lines 3–4 – Use + and > to compute and compare.
  2. Lines 5–6 – Use || (OR) to combine conditions.
  3. Lines 7–8 – Print results.

5. ✅ When to Use Different Operator Types

  • Arithmetic (+ - * / %) – for calculations (totals, averages, remainders).
  • Relational (>, <, ==, !=, >=, <=) – when you need comparisons.
  • Logical (&& || !) – when combining multiple conditions in if / while.
  • Assignment (=, +=, -= ...) – when updating variable values.
  • Increment/Decrement (++ --) – in loops and counters.

6. ⛔ When NOT to Use Certain Operators

  • Don’t use == to compare String content – use .equals() instead.
  • Be careful with / with integers – it does integer division (no decimal part).
  • Avoid overusing ++/-- inside complex expressions – it confuses beginners.

7. 🧠 Minimum You MUST Remember

  • Arithmetic: + - * / %.
  • Relational: > < == != >= <= → always give boolean.
  • Logical: && (AND), || (OR), ! (NOT).
  • Assignment shortcuts: +=, -=, *= etc.

8. ⚠️ Common Mistakes

  • Using = instead of == in conditions.
  • Dividing integers and expecting decimal result (e.g., 5/2 → 2, not 2.5).
  • Misunderstanding ++a vs a++ (pre/post increment).
  • Using & and | (bitwise) instead of && and || by mistake.
