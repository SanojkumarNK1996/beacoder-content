🔢 Primitive Data Types in Java

1. 🎯 Why This Exists (Indian Real-Life Example)

  • In a railway reservation system, you need to store:
    • Seat number → whole number (e.g., 23).
    • Ticket price → decimal (e.g., 523.75).
    • Coach code → a character (e.g., 'S').
    • Payment status → true/false.
  • Primitive data types let Java store these basic pieces of data efficiently.

2. 📖 What Are Primitive Data Types?

  • They are the basic building blocks of data in Java.
  • They store simple values like numbers, characters, and true/false.
  • There are 8 primitive types in Java.

3. 🧾 Syntax / Basic Structure ⭐

General form:

dataType variableName = value;

Examples of each primitive type:

byte branchCode = 10;          // small integer
short students = 3000;         // medium integer
int ticketCount = 120000;      // normal integer
long aadhaar = 123456789012L;  // very large integer (note 'L')

float gstRate = 18.5f;         // decimal with 'f'
double ticketPrice = 523.75;   // decimal (default)

char coach = 'S';              // single character
boolean paymentDone = true;    // true or false

4. 🔍 How It Works (Step-by-Step Execution) ⭐

public class PrimitiveDemo {
    public static void main(String[] args) {
        int age = 22;                 // 1
        double salary = 35000.50;     // 2
        char grade = 'A';             // 3
        boolean isEmployed = true;    // 4

        System.out.println(age);      // 5
        System.out.println(salary);   // 6
        System.out.println(grade);    // 7
        System.out.println(isEmployed); // 8
    }
}
  1. Line 3 – Creates an int variable age and stores 22.
  2. Line 4 – Creates a double variable salary for decimal values.
  3. Line 5 – Creates a char variable grade.
  4. Line 6 – Creates a boolean variable representing true/false.
  5. Lines 8–11 – Print each value to the screen.

5. ✅ When to Use Primitive Types

  • For simple values like counts, prices, statuses, characters.
  • When you care about performance and memory.
  • In low-level parts of code (like loops, calculations, flags).

6. ⛔ When NOT to Use Primitive Types

  • When you need objects with methods (e.g., String, collections).
  • When you need to store multiple values together (use arrays, classes).
  • When you need to use them with generics/collections – then wrapper classes (Integer, Double, etc.) are used instead.

7. 🧠 Minimum You MUST Remember

  • There are 8 primitive types: byte, short, int, long, float, double, char, boolean.
  • Use int for most whole numbers, double for decimals.
  • Add L at the end of long and f at the end of float values.
  • char stores one character inside single quotes ('A').

8. ⚠️ Common Mistakes

  • Forgetting f for float (e.g., float x = 10.5; – error, should be 10.5f).
  • Using long without L for very large numbers.
  • Using char with double quotes instead of single quotes.
  • Storing large values in small types and causing overflow (e.g., value > 127 in byte).
