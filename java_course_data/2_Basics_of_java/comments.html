Comments in Java

In Java, comments are lines in your code that are ignored by the compiler. They are used to explain code, make it more readable, and help others (and your future self) understand what the code does.

Comments do not affect how a program runs — they are purely for humans to read.

Why Use Comments?

  • To describe what a piece of code does.
  • To temporarily disable a part of the code during debugging.
  • To provide information about the author, version, or usage of a class or method.

Types of Comments in Java

Java supports three types of comments:

  1. Single-line comments
  2. Multi-line comments
  3. Documentation comments

1. Single-line Comments

Single-line comments start with //. Everything written after // on that line is ignored by the compiler.

Example:

public class Main {
    public static void main(String[] args) {
        // This is a single-line comment
        System.out.println("Hello, World!"); // This prints text to the console
    }
}

💡 Use single-line comments for short notes or quick explanations.

2. Multi-line Comments

Multi-line comments start with /* and end with */. Everything between these symbols is ignored by the compiler — even if it spans multiple lines.

Example:

public class Main {
    public static void main(String[] args) {
        /* This is a multi-line comment.
           It can cover several lines.
           Use it for detailed explanations. */
        System.out.println("Java is fun!");
    }
}

💡 Use multi-line comments when you need to explain a large section of code or document logic flow.

3. Documentation Comments

Documentation comments start with /** and end with */. They are used to describe classes, methods, and fields in detail and can be extracted using the Javadoc tool to generate HTML documentation.

Example:

/**
 * The HelloWorld class prints a greeting message.
 * 
 * @author  John Doe
 * @version 1.0
 */
public class HelloWorld {
    
    /**
     * This method prints "Hello, World!" to the console.
     * @param args Unused
     */
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

💡 Use documentation comments for public classes and methods — they make your code professional and easier to understand for others.

Summary

  • Single-line comments: start with // — good for short explanations.
  • Multi-line comments: enclosed between /* and */ — used for longer notes.
  • Documentation comments: start with /** — used with Javadoc to generate documentation.

Comments are one of the simplest yet most important parts of writing clean and maintainable Java code. Always document your code well!
