🧩 Non-Primitive (Reference) Data Types

1. 🎯 Why This Exists (Indian Real-Life Example)

  • In a Swiggy order, you don’t just store one value:
    • Restaurant name (text)
    • List of items
    • Each item has name, price, quantity
  • Primitive types alone (int, double, etc.) are not enough to model this.
  • We need non-primitive types like String, arrays, and classes/objects to represent real-world data.

2. 📖 What Are Non-Primitive Data Types?

  • Also called reference types.
  • They are created from classes.
  • They can store multiple values and have methods.
  • Examples: String, arrays, classes, interfaces, objects.

3. 🧾 Syntax / Basic Structure ⭐

Examples:

// String
String city = "Bengaluru";

// Array of integers
int[] marks = {85, 90, 78, 92};

// Custom class object (imagine Student class exists)
Student s = new Student();

4. 🔍 How It Works (Step-by-Step Execution) ⭐

Example with String

public class StringExample {
  public static void main(String[] args) {
    String course = "Java Programming";      // 1
    System.out.println(course);              // 2
    System.out.println(course.length());     // 3
    System.out.println(course.toUpperCase()); // 4
  }
}
  1. Line 3 – String variable course is created.
  2. Line 4 – Prints the text as it is.
  3. Line 5 – Calls method length() to get number of characters.
  4. Line 6 – Calls toUpperCase() to convert text to uppercase.

Example with Array

public class ArrayExample {
  public static void main(String[] args) {
    int[] marks = {85, 90, 78};        // 1
    System.out.println(marks[0]);     // 2

    for (int mark : marks) {          // 3
      System.out.println(mark);     // 4
    }
  }
}
  1. Line 3 – Creates an array of three integers.
  2. Line 4 – Accesses the first element using index 0.
  3. Lines 6–8 – Loop through the array and print each mark.

5. ✅ When to Use Non-Primitive Types

  • When you need to store text (names, cities, email) → String.
  • When you have a list of similar values (marks, ticket numbers) → arrays.
  • When you model real-world entities (Student, Order, Account) → classes/objects.

6. ⛔ When NOT to Use Non-Primitive Types

  • For very small, simple values like counters, flags, or tiny calculations – primitives are enough.
  • When performance is ultra-critical and object overhead is unnecessary.

7. 🧠 Minimum You MUST Remember

  • Non-primitive = reference types (they refer to objects in memory).
  • String and arrays are the most commonly used non-primitive types for beginners.
  • They have methods (like length(), toUpperCase()).

8. ⚠️ Common Mistakes

  • Confusing String with a primitive type (it is a class).
  • Forgetting that array indices start from 0 (not 1).
  • Trying to access an index outside the array range → ArrayIndexOutOfBoundsException.
