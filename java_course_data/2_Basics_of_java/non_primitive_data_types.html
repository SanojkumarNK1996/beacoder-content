Non-Primitive Data Types

Non-primitive types (also known as reference types) are more complex. They are created from classes and can store multiple values or objects. Unlike primitive types, non-primitive types have methods (functions) you can use to perform operations.

1. Strings

A String is used to store text. It is actually a class in Java, not a primitive type.

public class StringExample {
    public static void main(String[] args) {
        String name = "Java Programming";
        System.out.println("Welcome to " + name);
        System.out.println("Length: " + name.length());
        System.out.println("Uppercase: " + name.toUpperCase());
    }
}

Strings can be combined, compared, and modified using built-in methods like length(), toUpperCase(), and concat().

2. Arrays

An Array is a collection of elements of the same type, stored in a single variable. For example, you can store multiple numbers or names together.

public class ArrayExample {
    public static void main(String[] args) {
        int[] numbers = {10, 20, 30, 40, 50};
        
        System.out.println("First number: " + numbers[0]);
        System.out.println("All numbers:");
        
        for (int num : numbers) {
            System.out.println(num);
        }
    }
}

Arrays make it easy to manage multiple values using indexes. The first element starts at position 0.

3. Summary

Category Type Example

Understanding data types is essential in Java. They decide what kind of data you can store, how much memory it uses, and what operations you can perform. Always choose the type that best fits your data for efficient and clean code.
