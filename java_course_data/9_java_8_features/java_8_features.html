⚡ Java 8+ Features (Beginner Overview)

1. 🎯 Why This Exists (Indian Real-Life Example)

  • Modern apps like Swiggy, PhonePe, IRCTC handle huge data – lists of orders, transactions, trains.
  • We often want to filter, sort, transform this data in a clean way.
  • Before Java 8, this needed a lot of boilerplate loops.
  • Java 8+ features like lambdas, streams, and Optional make code shorter, clearer, and safer.

2. 📖 What Are Java 8+ Features (for You)?

  • Lambda expressions – small functions you can pass around as values.
  • Streams API – pipeline style processing of collections (filter, map, collect).
  • Optional – a wrapper to safely handle null values.
  • Functional interfaces – interfaces with exactly one abstract method, used with lambdas.

3. 🧾 Syntax / Basic Structure ⭐

Lambda basic form

(parameters) -> expression
// or
(parameters) -> { statements }

Simple example: print all names

import java.util.*;

public class LambdaBasic {
    public static void main(String[] args) {
        List<String> names = Arrays.asList("Rahul", "Anjali", "Vikas");

        // Using lambda with forEach
        names.forEach(n -> System.out.println("Hello, " + n));
    }
}

4. 🔍 How It Works (Step-by-Step Execution) ⭐

  1. Create list – names holds three strings.
  2. Call forEach – for each element, Java calls the lambda n -> ....
  3. Lambda runs – n takes each value ("Rahul", then "Anjali", then "Vikas").
  4. Print – prints "Hello, Rahul" and so on.

Streams small example – filter even numbers

import java.util.*;
import java.util.stream.*;

public class StreamBasic {
    public static void main(String[] args) {
        List<Integer> nums = Arrays.asList(1, 2, 3, 4, 5, 6);

        List<Integer> even = nums.stream()          // 1
                                  .filter(n -> n % 2 == 0) // 2
                                  .collect(Collectors.toList()); // 3

        System.out.println("Even: " + even);
    }
}
  1. nums.stream() – converts list to a stream (pipeline).
  2. filter(...) – keeps only numbers where condition is true.
  3. collect(...) – converts the stream back to a list.

5. ✅ When to Use These Features

  • When processing lists/sets/maps of data (orders, users, transactions).
  • When you want clean, readable code instead of nested loops.
  • When you want to avoid NullPointerException using Optional.

6. ⛔ When NOT to Overuse

  • Don’t use streams + lambdas just to look "fancy" in very simple loops.
  • Don’t chain too many operations in a single line – it becomes hard to read.
  • Don’t wrap everything in Optional; it’s mainly for values that can be genuinely missing.

7. 🧠 Minimum You MUST Remember

  • Lambda = (args) -> { body }.
  • Basic stream flow: source → stream() → operations (filter/map) → terminal (collect/forEach).
  • Optional helps you say "value may be absent" and handle it safely.

8. ⚠️ Common Mistakes

  • Using == to compare strings inside lambdas instead of equals().
  • Doing heavy work inside filter or map making streams slow.
  • Calling get() on Optional without checking presence – defeats its purpose.
