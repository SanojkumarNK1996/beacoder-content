🔧 Methods (Functions) in Java

1. 🎯 Why This Exists (Indian Real-Life Example)

  • In a bank, the same interest calculation is done for thousands of accounts.
  • In a Swiggy app, the same logic calculates total bill for every order.
  • Instead of writing that logic again and again, we put it in a method and reuse it.

2. 📖 What Is a Method?

  • A method is a named block of code that performs a specific task.
  • You write it once and call it whenever needed.
  • It can take inputs (parameters) and may give an output (return value).

3. 🧾 Syntax / Basic Structure ⭐

returnType methodName(parameterList) {
    // method body
}

// Example: no input, no return
public static void greet() {
    System.out.println("Hello, welcome to Java!");
}

4. 🔍 How It Works (Step-by-Step Execution) ⭐

public class MethodDemo {
    // 1. Method definition
    public static int add(int a, int b) {
        int sum = a + b;      // 2
        return sum;           // 3
    }

    public static void main(String[] args) {
        int result = add(5, 10);           // 4
        System.out.println("Sum = " + result); // 5
    }
}
  1. Method add is defined with 2 parameters and int return type.
  2. Inside method, we calculate sum.
  3. return sum; sends result back to caller.
  4. main calls add(5, 10); control jumps to method, executes, and comes back.
  5. Result is printed.

5. ✅ When to Use Methods

  • Whenever a piece of logic is reused in multiple places.
  • To break a large program into small readable parts.
  • To avoid code duplication and make maintenance easier.

6. ⛔ When NOT to Use / Be Careful

  • Don’t create unnecessary tiny methods that are called only once and reduce readability.
  • Be careful with recursion; always have a base case to stop.

7. 🧠 Minimum You MUST Remember

  • General form: returnType name(parameters) { body }.
  • void means method does not return anything.
  • Parameters are inputs; return type is output.

8. ⚠️ Common Mistakes

  • Forgetting to return a value in non-void methods.
  • Mismatch between parameter types in method definition and method call.
  • Writing all code in main instead of separate methods, making programs messy.
