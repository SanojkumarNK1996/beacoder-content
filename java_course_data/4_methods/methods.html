Methods (Functions) in Java

Methods, also called functions, are blocks of code that perform a specific task. They help make programs more organized, reusable, and easier to understand. Using methods is part of modular programming — breaking a large program into smaller, manageable pieces.

1. Defining Methods

A method is defined using a specific structure that includes its return type, name, and parameters. The general syntax looks like this: 


returnType methodName(parameters) {
    // code to execute
}

Example: 


// Example: a simple method
public static void greet() {
    System.out.println("Hello, welcome to Java!");
}

Here, the method greet() has no parameters and no return value. You can call it from the main() method like this: 


public static void main(String[] args) {
    greet(); // method call
}

2. Method Parameters and Return Types

A method can take inputs (called parameters) and can send back an output (called a return value). 


// Example: method with parameters and return type
public static int addNumbers(int a, int b) {
    return a + b; // returns the sum
}

public static void main(String[] args) {
    int result = addNumbers(5, 10);
    System.out.println("Sum: " + result);
}

Here, addNumbers() takes two integers and returns their sum. The int before the method name means the method will return an integer value.

3. Method Overloading

Method overloading means having multiple methods with the same name but different parameters. This helps you perform similar operations in different ways. 


// Example: method overloading
public static int add(int a, int b) {
    return a + b;
}

public static double add(double a, double b) {
    return a + b;
}

public static void main(String[] args) {
    System.out.println(add(5, 10));      // calls first method
    System.out.println(add(5.5, 2.3));   // calls second method
}

The compiler decides which method to call based on the arguments you pass. This is called compile-time polymorphism.

4. Recursion Basics

Recursion is when a method calls itself to solve a smaller part of a problem. It’s commonly used in tasks like calculating factorials or traversing trees. 


// Example: factorial using recursion
public static int factorial(int n) {
    if (n == 1) {
        return 1; // base case
    }
    return n * factorial(n - 1); // recursive call
}

public static void main(String[] args) {
    System.out.println("Factorial of 5: " + factorial(5));
}

In this example, factorial(5) calls itself repeatedly until it reaches the base case (n == 1). Recursion must always have a base condition to stop, or it will loop forever.

Summary

  • Method – a reusable block of code that performs a task.
  • Parameters – values passed into methods.
  • Return type – defines what a method sends back.
  • Overloading – same method name, different parameter types.
  • Recursion – a method calling itself to solve smaller problems.

Using methods makes your Java programs cleaner, modular, and easier to maintain.
