🧠 Abstraction in Java

1. 🎯 Why This Exists (Indian Real-Life Example)

  • When you use a UPI app (PhonePe, GPay), you just click "Pay".
  • You don’t see how NPCI, banks, encryption, and servers work behind the scenes.
  • This hiding of internal complexity and showing only what you need is exactly what abstraction does in Java.

2. 📖 What Is Abstraction?

  • Abstraction means showing only essential details and hiding the internal working.
  • In Java, it is mainly achieved using abstract classes and interfaces.
  • You focus on what an object can do, not how it does it.

3. 🧾 Syntax / Basic Structure ⭐

// Abstract class
abstract class Animal {
    abstract void makeSound();  // abstract method (no body)

    void sleep() {              // normal method
        System.out.println("This animal is sleeping");
    }
}

// Interface
interface Vehicle {
    void start();
    void stop();
}

4. 🔍 How It Works (Step-by-Step Execution) ⭐

abstract class Bank {
    abstract void transferMoney();     // 1

    void showBalance() {              // 2
        System.out.println("Showing balance");
    }
}

class SBIBank extends Bank {          // 3
    void transferMoney() {
        System.out.println("Transferring via SBI servers");
    }
}

public class AbstractionDemo {
    public static void main(String[] args) {
        Bank bank = new SBIBank();    // 4
        bank.transferMoney();         // 5
        bank.showBalance();           // 6
    }
}
  1. Bank declares an abstract method transferMoney() – only "what", not "how".
  2. showBalance() is a normal method with full code.
  3. SBIBank extends Bank and provides actual implementation.
  4. In main, we use reference type Bank but object type SBIBank.
  5. Call to transferMoney() runs SBI-specific code, but main method does not know internal logic.

5. ✅ When to Use Abstraction

  • When many classes share a common idea but differ in details (e.g., different banks, vehicles, payment methods).
  • When you want to define a contract for other developers (via interfaces).
  • When you want to hide complex logic and expose only simple methods to the outside world.

6. ⛔ When NOT to Use / Overuse

  • For very small, simple programs where abstraction will make code harder to read.
  • Don’t create abstract classes/interfaces without real need – it can overcomplicate beginner-level code.

7. 🧠 Minimum You MUST Remember

  • abstract class → cannot create objects directly.
  • Abstract methods have no body and must be implemented in child classes.
  • Interface → all methods are abstract by default (Java 8+ adds default/static methods, but that’s later).
  • Abstraction = focus on what, hide how.

8. ⚠️ Common Mistakes

  • Trying to create an object of an abstract class: new Animal() → not allowed.
  • Forgetting to implement all interface methods in the implementing class.
  • Mixing up "abstraction" with "encapsulation" – abstraction hides implementation ideas; encapsulation hides data.

9. 🎤 Interviewer’s Perspective

  • "What is abstraction? Give a real-life example."
  • "How is abstraction achieved in Java?" (Expected: abstract classes and interfaces.)
  • "Difference between abstract class and interface?" (basic points only for fresher).

10. 📝 Practice Questions ⭐

  1. Very Easy: Create an abstract class Shape with abstract method draw(). Create two subclasses Circle and Rectangle and print different messages in draw().
  2. Easy: Define an interface Payment with methods pay() and refund(). Implement it for UPIPayment and CardPayment.
  3. Medium: Write down at least 5 differences between abstract class and interface in your own words.
  4. Thinking: Pick any app you use daily (Swiggy, IRCTC, Paytm). List what actions it exposes to you and imagine what internal details it might be hiding – relate that to abstraction.
