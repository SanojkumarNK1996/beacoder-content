Inheritance in Java

Inheritance is an important concept in Object-Oriented Programming (OOP) that allows one class to acquire the properties and behaviors of another class. It helps to reuse existing code and build new features easily.

In Java, inheritance is implemented using the extends keyword.

What is Inheritance?

When a class (called the child class or subclass) inherits from another class (called the parent class or superclass), it automatically gets access to all the fields and methods of the parent class.

Example:

class Animal {
    void eat() {
        System.out.println("This animal eats food");
    }
}

// Dog inherits from Animal
class Dog extends Animal {
    void bark() {
        System.out.println("The dog barks");
    }
}

public class Main {
    public static void main(String[] args) {
        Dog d = new Dog();
        d.eat();  // Inherited method
        d.bark(); // Child class method
    }
}

Here, the Dog class inherits from the Animal class using extends. This means the Dog class can use the eat() method defined in Animal.

The extends Keyword

The extends keyword is used to create a subclass from a superclass. It establishes an “is-a” relationship between classes.

Syntax:

class SubClass extends SuperClass {
    // additional fields and methods
}

Example:

class Vehicle {
    void start() {
        System.out.println("Vehicle started");
    }
}

class Car extends Vehicle {
    void playMusic() {
        System.out.println("Music is playing");
    }
}

public class Main {
    public static void main(String[] args) {
        Car c = new Car();
        c.start();     // Inherited from Vehicle
        c.playMusic(); // Defined in Car
    }
}

The super Keyword

The super keyword refers to the parent class. It can be used in two ways:

  1. To call the parent class constructor.
  2. To access parent class methods or variables.

Example 1 – Calling Parent Constructor:

class Animal {
    Animal() {
        System.out.println("Animal created");
    }
}

class Dog extends Animal {
    Dog() {
        super();  // calls Animal constructor
        System.out.println("Dog created");
    }
}

public class Main {
    public static void main(String[] args) {
        Dog d = new Dog();
    }
}

Output:

Animal created
Dog created

Example 2 – Accessing Parent Method:

class Animal {
    void eat() {
        System.out.println("Animal eats");
    }
}

class Dog extends Animal {
    void eat() {
        System.out.println("Dog eats");
    }

    void showEating() {
        super.eat(); // calls parent method
        eat();       // calls child method
    }
}

public class Main {
    public static void main(String[] args) {
        Dog d = new Dog();
        d.showEating();
    }
}

Output:

Animal eats
Dog eats

Method Overriding

Method overriding means redefining a method in the child class that already exists in the parent class with the same name, parameters, and return type.

It allows a subclass to provide its own version of a method to modify or extend the parent behavior.

Example:

class Animal {
    void makeSound() {
        System.out.println("Some generic animal sound");
    }
}

class Dog extends Animal {
    @Override
    void makeSound() {
        System.out.println("Dog barks");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal a = new Animal();
        a.makeSound(); // Parent method

        Dog d = new Dog();
        d.makeSound(); // Overridden method
    }
}

Output:

Some generic animal sound
Dog barks

Here, the Dog class overrides the makeSound() method from the Animal class. The @Override annotation is used to indicate that the method is being overridden.

Key Points

  • Inheritance allows a class to reuse code from another class.
  • extends is used to create a subclass from a superclass.
  • super is used to call the parent class constructor or method.
  • Method overriding lets the subclass change or improve parent class behavior.

Summary

  • extends → Establishes inheritance relationship.
  • super → Refers to the parent class.
  • Overriding → Redefining parent class methods in the subclass.

Inheritance promotes code reusability, extensibility, and a clear class hierarchy in Java programs.
