🧬 Inheritance in Java

1. 🎯 Why This Exists (Indian Real-Life Example)

  • Think of a vehicle company like Tata Motors.
  • They have a base design for a "vehicle": engine, wheels, brakes.
  • From that, they create different models like car, truck, bus which all share common features but add their own.
  • In Java, this "base design" is the parent class, and the specific models are child classes using inheritance.

2. 📖 What Is Inheritance?

  • Inheritance lets one class (child/subclass) reuse fields and methods of another class (parent/superclass).
  • In Java, this is done using the extends keyword.
  • It represents an "is-a" relationship (Dog is an Animal, Car is a Vehicle).

3. 🧾 Syntax / Basic Structure ⭐

class Animal {            // parent class
    void eat() {
        System.out.println("This animal eats food");
    }
}

class Dog extends Animal {   // child class
    void bark() {
        System.out.println("The dog barks");
    }
}

public class Main {
    public static void main(String[] args) {
        Dog d = new Dog();
        d.eat();   // inherited from Animal
        d.bark();  // Dog's own method
    }
}

4. 🔍 How It Works (Step-by-Step Execution) ⭐

  1. Dog is defined with extends Animal → Dog becomes a subclass.
  2. Dog automatically gets access to eat() from Animal.
  3. When we create new Dog(), memory is allocated for both Animal part and Dog part.
  4. Calling d.eat() runs the parent method; d.bark() runs the child method.

5. ✅ When to Use Inheritance

  • When classes share common properties/behaviour (e.g., all vehicles, all employees).
  • When you want to reduce duplicate code.
  • When you want a clear hierarchy (e.g., Animal → Dog, Cat; Employee → Manager, Developer).

6. ⛔ When NOT to Use / Overuse

  • Don’t use inheritance just to reuse one or two methods – consider composition instead.
  • If the relationship is not truly "is-a" (e.g., Bank is not a Customer), avoid inheritance.

7. 🧠 Minimum You MUST Remember

  • extends → used to create a subclass from a superclass.
  • super → used to refer to the parent class (constructor or method).
  • Child class inherits all non-private fields and methods of parent.

8. ⚠️ Common Related Concepts (Quick View)

super Keyword

class Animal {
    Animal() {
        System.out.println("Animal created");
    }
}

class Dog extends Animal {
    Dog() {
        super();                    // calls Animal constructor
        System.out.println("Dog created");
    }
}

Method Overriding (Brief)

class Animal {
    void makeSound() {
        System.out.println("Some animal sound");
    }
}

class Dog extends Animal {
    @Override
    void makeSound() {
        System.out.println("Dog barks");
    }
}
