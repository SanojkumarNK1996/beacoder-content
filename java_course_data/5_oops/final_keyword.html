The final Keyword in Java

In Java, the final keyword is used to make something unchangeable or constant. It can be applied to variables, methods, and classes. Once something is declared as final, it cannot be modified in the way it normally would.

1. final Variable

When a variable is declared as final, its value cannot be changed once it has been assigned. This makes it act like a constant.

Example:

class Example {
    final int SPEED_LIMIT = 100; // constant

    void show() {
        // SPEED_LIMIT = 120; // ❌ Error: cannot assign a value to final variable
        System.out.println("Speed Limit: " + SPEED_LIMIT);
    }
}

public class Main {
    public static void main(String[] args) {
        Example e = new Example();
        e.show();
    }
}

Output:

Speed Limit: 100

Once a final variable is assigned, you cannot reassign it. Final variables are often written in uppercase letters to show they are constants (e.g., MAX_SIZE).

Blank Final Variables

A blank final variable is a final variable that is not initialized at the time of declaration. It must be initialized in the constructor.

Example:

class Student {
    final String name; // blank final variable

    Student(String n) {
        name = n; // initialized in constructor
    }

    void display() {
        System.out.println("Student name: " + name);
    }
}

public class Main {
    public static void main(String[] args) {
        Student s1 = new Student("Alice");
        s1.display();
    }
}

Output:

Student name: Alice

A blank final variable allows each object to have its own final value.

Final Static Variables (Constants)

When a variable is declared as final and static, it becomes a constant for the entire class. Such constants are usually defined in uppercase and shared by all objects.

Example:

class MathUtils {
    static final double PI = 3.14159;
}

public class Main {
    public static void main(String[] args) {
        System.out.println("Value of PI: " + MathUtils.PI);
    }
}

Output:

Value of PI: 3.14159

2. final Method

When a method is declared as final, it cannot be overridden by subclasses. This ensures that the behavior of that method remains the same for all inherited classes.

Example:

class Vehicle {
    final void displayType() {
        System.out.println("This is a vehicle");
    }
}

class Car extends Vehicle {
    // void displayType() {  // ❌ Error: cannot override final method
    //     System.out.println("This is a car");
    // }
}

Here, the displayType() method in Vehicle is final, so it cannot be redefined in the Car class.

3. final Class

When a class is declared as final, it cannot be inherited. No other class can extend it.

Example:

final class Vehicle {
    void display() {
        System.out.println("This is a vehicle");
    }
}

// class Car extends Vehicle {  // ❌ Error: cannot subclass a final class
// }

Final classes are useful when you want to prevent inheritance or ensure that a class’s behavior is not changed.

For example, many core Java classes like String and Math are declared as final.

Summary of final Keyword

Usage Effect
final variable Value cannot be changed once assigned.
final method Cannot be overridden in subclasses.
final class Cannot be extended or subclassed.

Key Points

  • final means “unchangeable.”
  • Used with variables, methods, and classes.
  • final variable → constant value.
  • final method → prevents overriding.
  • final class → prevents inheritance.
  • final and abstract cannot be used together (they conflict).

Summary

  • Use final for constants, secure methods, and unextendable classes.
  • Helps make your code more reliable and predictable.

The final keyword gives you control over how classes, methods, and variables behave — ensuring that critical parts of your program stay safe and unchanged.
