🔒 Encapsulation in Java

1. 🎯 Why This Exists (Indian Real-Life Example)

  • Think of your bank account in SBI/HDFC.
  • You cannot directly open the bank’s database and change your balance.
  • You can only use controlled options like ATM, UPI, cheque, branch deposit.
  • This protection of data with controlled access is exactly what encapsulation does in Java.

2. 📖 What Is Encapsulation?

  • Encapsulation = wrapping data and methods inside a class.
  • It also means hiding the data and allowing access only via methods.
  • We generally make fields private and expose public getters/setters.

3. 🧾 Syntax / Basic Structure ⭐

class Person {
    private String name;   // hidden data
    private int age;       // hidden data

    public void setName(String n) {   // setter
        name = n;
    }

    public String getName() {         // getter
        return name;
    }
}

4. 🔍 How It Works (Step-by-Step Execution) ⭐

class Person {
    private String name;
    private int age;

    public void setName(String n) {
        name = n;                      // 2
    }

    public String getName() {
        return name;                   // 4
    }

    public void setAge(int a) {
        if (a > 0) {                   // 3 (validation)
            age = a;
        } else {
            System.out.println("Age must be positive!");
        }
    }
}

public class Main {
    public static void main(String[] args) {
        Person p = new Person();       // 1
        p.setName("Alice");
        p.setAge(25);
        System.out.println(p.getName());
    }
}
  1. Create object p of Person.
  2. setName() assigns the value to the private field.
  3. setAge() checks if age is valid before setting.
  4. getName() safely returns the stored value.

5. ✅ When to Use Encapsulation

  • Whenever you have sensitive data (salary, balance, password, marks, etc.).
  • When you want to validate data before saving it.
  • When you want to change internal implementation later without breaking other code.

6. ⛔ When NOT to Overuse

  • For very small demo programs, you don’t need getters/setters for everything.
  • Don’t create unnecessary getters/setters if the field is never used outside the class.

7. 🧠 Minimum You MUST Remember

  • Make fields private to hide data.
  • Use public getters and setters to provide controlled access.
  • You can add validation logic inside setters.

8. ⚠️ Common Mistakes

  • Keeping fields public and allowing anyone to change them.
  • Writing getters/setters but still directly accessing fields from outside.
  • Not adding validation where it is needed (like negative age, invalid marks, etc.).
