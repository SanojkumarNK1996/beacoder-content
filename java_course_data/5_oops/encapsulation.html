Encapsulation in Java

Encapsulation is one of the main principles of Object-Oriented Programming (OOP). It means wrapping data (fields) and methods (functions) together into a single unit — the class — and restricting direct access to the internal data.

In simple words, encapsulation hides how things work internally and only exposes what is necessary. It keeps data safe and controlled.

Why Encapsulation?

  • To protect data from unauthorized access.
  • To make code easier to maintain and modify.
  • To control how fields are set or retrieved.

Access Modifiers

Access modifiers define who can access a class, method, or variable. They are keywords that control the visibility of class members.

Modifier Access Level Description
private Within the same class only Used to hide data. Other classes cannot access it directly.
public Accessible from anywhere Used for methods or classes that should be available globally.
protected Accessible in the same package and by subclasses Used when you want to share data with inherited classes but not with everyone.
default (no keyword) Accessible only within the same package Used when you don’t need to expose data outside the package.

Example Without Encapsulation

class Person {
    String name;
    int age;
}

public class Main {
    public static void main(String[] args) {
        Person p = new Person();
        p.name = "John";  // Direct access to fields
        p.age = -5;       // Invalid data!
        System.out.println(p.name + " is " + p.age + " years old.");
    }
}

Here, data is not protected — anyone can change it directly, even with invalid values.

Encapsulated Version (Using Getters and Setters)

To protect fields, we make them private and provide public methods to access or modify them — these are called getters and setters. 

class Person {
    private String name;
    private int age;

    // Setter method
    public void setName(String n) {
        name = n;
    }

    // Getter method
    public String getName() {
        return name;
    }

    // Setter with validation
    public void setAge(int a) {
        if (a > 0) {
            age = a;
        } else {
            System.out.println("Age must be positive!");
        }
    }

    // Getter
    public int getAge() {
        return age;
    }
}

public class Main {
    public static void main(String[] args) {
        Person p = new Person();
        p.setName("Alice");
        p.setAge(25);

        System.out.println(p.getName() + " is " + p.getAge() + " years old.");
    }
}

In this example, the data is protected. You can only access it through the methods, which can check or limit what data is allowed.

Key Points

  • Use private to hide data fields.
  • Provide public getters and setters to access or modify data safely.
  • Add validation in setters to ensure correct data.
  • Encapsulation improves security, flexibility, and maintainability of your code.

Summary

  • Encapsulation hides internal details and exposes only what’s needed.
  • Access modifiers control visibility of members.
  • Getters and Setters provide controlled access to private fields.

Encapsulation helps you write clean, secure, and reliable Java programs by protecting data from unwanted changes.
