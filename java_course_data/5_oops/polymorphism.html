🎭 Polymorphism in Java

1. 🎯 Why This Exists (Indian Real-Life Example)

  • Think of a payment button in an app: "Pay".
  • Whether you use UPI, debit card, or net banking, you still click the same button.
  • The action name is the same (Pay), but the actual processing changes based on method.
  • This "same name, different behavior" is exactly what polymorphism provides in Java.

2. 📖 What Is Polymorphism?

  • Polymorphism literally means "many forms".
  • In Java, it allows a single method name to behave differently based on parameters or object type.
  • Two main types:
    • Compile-time polymorphism → method overloading.
    • Runtime polymorphism → method overriding.

3. 🧾 Syntax / Basic Structure ⭐

Compile-time (Overloading)

class MathUtils {
    int add(int a, int b) {          // 2 int
        return a + b;
    }

    int add(int a, int b, int c) {   // 3 int
        return a + b + c;
    }
}

Runtime (Overriding)

class Animal {
    void sound() {
        System.out.println("Some animal sound");
    }
}

class Dog extends Animal {
    @Override
    void sound() {
        System.out.println("Dog barks");
    }
}

4. 🔍 How It Works (Step-by-Step Execution) ⭐

A. Compile-time Polymorphism (Method Overloading)

class MathUtils {
    int add(int a, int b) {
        return a + b;
    }

    int add(int a, int b, int c) {
        return a + b + c;
    }
}

public class Main {
    public static void main(String[] args) {
        MathUtils m = new MathUtils();
        System.out.println(m.add(5, 10));      // 1
        System.out.println(m.add(5, 10, 15));  // 2
    }
}
  1. The compiler looks at add(5, 10) → 2 int parameters → picks 2-arg method.
  2. For add(5, 10, 15) → 3 int parameters → picks 3-arg method.

B. Runtime Polymorphism (Method Overriding)

class Animal {
    void sound() {
        System.out.println("Some animal sound");
    }
}

class Dog extends Animal {
    @Override
    void sound() {
        System.out.println("Dog barks");
    }
}

class Cat extends Animal {
    @Override
    void sound() {
        System.out.println("Cat meows");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal a1 = new Dog();  // 1
        Animal a2 = new Cat();  // 2

        a1.sound();             // 3
        a2.sound();             // 4
    }
}
  1. a1 reference type is Animal, object type is Dog.
  2. a2 reference type is Animal, object type is Cat.
  3. At runtime, a1.sound() calls Dog’s version.
  4. Similarly, a2.sound() calls Cat’s version.

5. ✅ When to Use Polymorphism

  • When you want a common interface but different implementations (e.g., different payment methods, different notification types).
  • When you want to write code that works on parent type but can handle different child objects.
  • When you want to reduce if-else chains and use overriding instead.

6. ⛔ When NOT to Overuse

  • Don’t overload methods with confusing parameter combinations that make code hard to read.
  • Don’t create inheritance just to show polymorphism in real projects – design should be natural.

7. 🧠 Minimum You MUST Remember

  • Overloading → same method name, different parameters, happens in same class.
  • Overriding → same method signature in parent & child, decided at runtime.
  • Polymorphism lets one reference type (parent) point to many object types (children).

8. ⚠️ Common Mistakes

  • Changing return type only and thinking it is overloading (it is not).
  • Misspelling method name in child class → overriding doesn’t actually happen.
  • Forgetting @Override annotation – then mistakes are harder to catch.
