Polymorphism in Java

The word Polymorphism comes from Greek, meaning "many forms". In Java, polymorphism allows a single action or method to behave differently based on the object or context.

It is one of the main features of Object-Oriented Programming (OOP), making code more flexible and reusable.

Types of Polymorphism in Java

  • Compile-time Polymorphism (Method Overloading)
  • Runtime Polymorphism (Method Overriding)

1. Compile-time Polymorphism (Method Overloading)

Compile-time polymorphism happens when multiple methods in the same class have the same name but different parameters (type, number, or order). This is known as method overloading.

The compiler decides which method to call at compile time based on the arguments passed.

Example:

class MathUtils {
    // Method with one parameter
    int add(int a, int b) {
        return a + b;
    }

    // Overloaded method with three parameters
    int add(int a, int b, int c) {
        return a + b + c;
    }

    // Overloaded method with different types
    double add(double a, double b) {
        return a + b;
    }
}

public class Main {
    public static void main(String[] args) {
        MathUtils math = new MathUtils();

        System.out.println(math.add(5, 10));        // Calls 2-int version
        System.out.println(math.add(5, 10, 15));    // Calls 3-int version
        System.out.println(math.add(5.5, 4.5));     // Calls double version
    }
}

Output:

15
30
10.0

Here, the add() method behaves differently based on the number and type of parameters. This is compile-time polymorphism because the method call is resolved during compilation.

2. Runtime Polymorphism (Method Overriding)

Runtime polymorphism occurs when a child class overrides a method of its parent class. The method that gets executed is determined at runtime based on the object’s actual type.

This is achieved using method overriding and is the foundation of dynamic method dispatch in Java.

Example:

class Animal {
    void sound() {
        System.out.println("Some generic animal sound");
    }
}

class Dog extends Animal {
    @Override
    void sound() {
        System.out.println("Dog barks");
    }
}

class Cat extends Animal {
    @Override
    void sound() {
        System.out.println("Cat meows");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal a1 = new Dog(); // Reference of parent, object of child
        Animal a2 = new Cat();

        a1.sound(); // Dog’s version called
        a2.sound(); // Cat’s version called
    }
}

Output:

Dog barks
Cat meows

Here, the parent reference variable (Animal) points to different child objects (Dog and Cat). At runtime, Java decides which method to execute based on the actual object type.

Key Differences

Feature Compile-time Polymorphism Runtime Polymorphism
Definition Same method name, different parameters Same method name and parameters, but defined in child class
Method Type Method Overloading Method Overriding
Decision Time During compile time During runtime
Inheritance Required No Yes
Performance Faster Slightly slower

Key Points

  • Polymorphism allows one method name to perform different actions.
  • Overloading → Happens within the same class (compile time).
  • Overriding → Happens across parent and child classes (runtime).
  • It improves code reusability and flexibility.

Summary

  • Compile-time Polymorphism → Achieved by method overloading.
  • Runtime Polymorphism → Achieved by method overriding.
  • Polymorphism helps Java programs adapt behavior dynamically and handle objects more generically.

By mastering polymorphism, you can write cleaner, more extensible, and more dynamic Java programs.
