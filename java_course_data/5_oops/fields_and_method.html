Fields and Methods in Java

In Java, fields and methods are the main parts of a class. They define what an object knows (data) and what it does (behavior).

Fields

A field (also called a member variable) is a variable that belongs to a class or an object. It stores data or state information about an object.

Example:

class Car {
    String color;  // field
    int speed;     // field
}

Here, color and speed are fields that describe the car.

Methods

A method is a block of code inside a class that performs a specific task. Methods are used to define the behavior of objects.

Example:

class Car {
    String color;
    int speed;

    void drive() {
        System.out.println("The car is moving at " + speed + " km/h");
    }

    void stop() {
        System.out.println("The car has stopped");
    }
}

Here, drive() and stop() are methods that describe what the car can do.

Instance Members

An instance member belongs to an individual object of a class. Every object has its own copy of instance fields and methods.

Example:

class Car {
    String color; // instance field

    void showColor() { // instance method
        System.out.println("Color: " + color);
    }
}

public class Main {
    public static void main(String[] args) {
        Car car1 = new Car();
        Car car2 = new Car();

        car1.color = "Red";
        car2.color = "Blue";

        car1.showColor(); // Output: Color: Red
        car2.showColor(); // Output: Color: Blue
    }
}

Each car object has its own color. Changing one does not affect the other.

Static Members

A static member belongs to the class itself, not to any specific object. It is shared by all objects of that class. You can access static members using the class name directly.

Example:

class Car {
    static int wheels = 4; // static field
    String color;

    static void showWheels() { // static method
        System.out.println("All cars have " + wheels + " wheels");
    }
}

public class Main {
    public static void main(String[] args) {
        Car.showWheels(); // Access static method directly

        Car car1 = new Car();
        car1.color = "Black";

        System.out.println("Car color: " + car1.color);
    }
}

Here, wheels and showWheels() are static members shared across all car objects.

Key Differences

Feature Instance Member Static Member
Belongs to Individual object Class itself
Memory Separate copy for each object One shared copy for all objects
Accessed by Using object reference Using class name
Use case When data varies per object When data is common for all objects

Summary

  • Fields store object data.
  • Methods define object behavior.
  • Instance members belong to each object separately.
  • Static members are shared among all objects of a class.

Understanding the difference between instance and static members helps you design cleaner and more efficient Java programs.
