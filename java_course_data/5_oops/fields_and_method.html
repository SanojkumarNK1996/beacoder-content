🏗️ Fields and Methods in Java

1. 🎯 Why This Exists (Indian Real-Life Example)

  • Think of a Swiggy order.
  • Each order has some data: items, price, address, status.
  • And some actions: place order, cancel order, track delivery.
  • In Java, fields store data and methods perform actions – exactly like this.

2. 📖 What Are Fields and Methods?

  • Field (member variable) → stores information about the object (data/state).
  • Method → block of code that performs some operation (behaviour).
  • They together define what an object has and what it does.

3. 🧾 Syntax / Basic Structure ⭐

class Car {
    String color;   // field
    int speed;      // field

    void drive() {  // method
        System.out.println("The car is moving at " + speed + " km/h");
    }

    void stop() {   // method
        System.out.println("The car has stopped");
    }
}

4. 🔍 How It Works (Instance vs Static) ⭐

A. Instance Members

class Car {
    String color;           // instance field

    void showColor() {      // instance method
        System.out.println("Color: " + color);
    }
}

public class Main {
    public static void main(String[] args) {
        Car car1 = new Car();
        Car car2 = new Car();

        car1.color = "Red";
        car2.color = "Blue";

        car1.showColor();   // Color: Red
        car2.showColor();   // Color: Blue
    }
}

Each object keeps its own copy of instance fields.

B. Static Members

class Car {
    static int wheels = 4;      // static field
    String color;

    static void showWheels() {  // static method
        System.out.println("All cars have " + wheels + " wheels");
    }
}

public class Main {
    public static void main(String[] args) {
        Car.showWheels();       // 1

        Car car1 = new Car();
        car1.color = "Black";  // 2
    }
}
  1. Static members are accessed using class name.
  2. Static data is shared across all objects.

5. ✅ When to Use Instance vs Static

  • Use instance fields when value is different for each object (student name, account balance).
  • Use static fields when value is same for all objects (bank IFSC code, number of wheels, company name).

6. ⛔ When NOT to Use Static

  • Do not make everything static just to avoid creating objects – it leads to bad design.
  • Avoid static when data is specific to each object; otherwise values may be accidentally shared.

7. 🧠 Minimum You MUST Remember

  • Field = variable in class; Method = function in class.
  • Instance → per object; static → per class (shared).
  • Static members are accessed via ClassName.member.

8. ⚠️ Common Mistakes

  • Trying to access instance fields from static methods without an object.
  • Using object reference to access static members (works, but not recommended style).
  • Making fields public instead of using encapsulation (private + getters/setters).
