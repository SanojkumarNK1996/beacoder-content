Packages and Importing in Java

In Java, a package is a way to organize related classes and interfaces together. It helps keep code neat, avoids naming conflicts, and makes large projects easier to manage.

You can think of a package as a folder inside your project that contains related files.

What is a Package?

A package is a namespace that groups related Java classes, interfaces, and sub-packages. It helps developers organize code logically, similar to how folders organize files on your computer.

Example:

package myapp.models;

public class Student {
    public void displayInfo() {
        System.out.println("This is a student class");
    }
}

Here, the class Student belongs to the package myapp.models. The first line package myapp.models; defines the package name.

Types of Packages

  • Built-in Packages – These are provided by Java itself, such as java.util, java.io, java.lang, etc.
  • User-defined Packages – These are created by programmers to organize their own classes.

Using Built-in Packages

Java provides many built-in packages that contain ready-to-use classes and methods. You can use them by importing the required package.

Example:

import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);  // using class from java.util package
        System.out.print("Enter your name: ");
        String name = sc.nextLine();
        System.out.println("Hello, " + name + "!");
    }
}

Here, we import the Scanner class from the java.util package to read user input.

Creating and Using User-defined Packages

Step 1: Create a Package

// File: mypack/Message.java
package mypack;

public class Message {
    public void show() {
        System.out.println("Hello from mypack package!");
    }
}

Step 2: Use the Package in Another Class

// File: Main.java
import mypack.Message;

public class Main {
    public static void main(String[] args) {
        Message msg = new Message();
        msg.show();
    }
}

When you compile, make sure both files are in proper folder structure: 

project-folder/
├── mypack/
│   └── Message.java
└── Main.java

Compile and run commands: 

javac mypack/Message.java
javac Main.java
java Main

Output:

Hello from mypack package!

Importing Packages

You can import classes or entire packages using the import keyword.

Type Syntax Description
Single class import import packageName.ClassName; Imports one specific class.
Whole package import import packageName.*; Imports all classes in a package.
Static import import static packageName.ClassName.member; Allows access to static members without using the class name.

Example of Static Import

import static java.lang.Math.*;

public class Main {
    public static void main(String[] args) {
        System.out.println("Square root of 25: " + sqrt(25));
        System.out.println("Value of PI: " + PI);
    }
}

Output:

Square root of 25: 5.0
Value of PI: 3.141592653589793

Here, sqrt() and PI are used directly without writing Math.sqrt() or Math.PI because of static import.

Key Points

  • Packages group related classes together.
  • They help in code organization and name conflict prevention.
  • Use the package keyword to define your own package.
  • Use the import keyword to access classes from other packages.
  • Static import lets you access static members without prefixing class names.

Summary

  • Package → Container for classes and interfaces.
  • Import → Used to access classes from other packages.
  • Static Import → Allows direct access to static members.

Packages keep your Java code organized, reusable, and easy to maintain.
