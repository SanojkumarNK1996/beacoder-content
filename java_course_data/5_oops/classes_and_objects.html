🚗 Classes and Objects in Java

1. 🎯 Why This Exists (Indian Real-Life Example)

  • Think of a Car showroom in India (Maruti, Tata, Hyundai).
  • The company first designs a blueprint of a car model (e.g., Nexon, Baleno).
  • From this one design, many actual cars are manufactured and sold.
  • In Java: the class is like the blueprint, and the objects are the actual cars you use.

2. 📖 What Are Classes and Objects?

  • Class → A blueprint/template that defines properties (fields) and behaviour (methods).
  • Object → A real instance created from a class, stored in memory.
  • Java programs are mostly about creating classes and then creating objects from them.

3. 🧾 Syntax / Basic Structure ⭐

// Class definition
class Car {
    String color;   // field
    int speed;      // field

    void drive() {  // method
        System.out.println("The car is moving");
    }
}

// Creating an object
Car myCar = new Car();
myCar.color = "Red";
myCar.speed = 120;
myCar.drive();

4. 🔍 How It Works (Step-by-Step Execution) ⭐

class Car {
    String color;
    int speed;

    void drive() {
        System.out.println("The car is moving");
    }
}

public class Main {
    public static void main(String[] args) {
        Car myCar = new Car();   // 1
        myCar.color = "Blue";   // 2
        myCar.speed = 150;       // 3
        myCar.drive();           // 4
    }
}
  1. new Car() creates an object in memory.
  2. myCar.color stores the car’s color.
  3. myCar.speed stores the car’s speed.
  4. myCar.drive() calls the method, which prints the message.

5. ✅ Constructors (Special Methods)

A constructor is used to initialize the object when it is created.

class Car {
    String color;
    int speed;

    // Constructor
    Car(String c, int s) {
        color = c;
        speed = s;
    }

    void displayInfo() {
        System.out.println("Color: " + color + ", Speed: " + speed);
    }
}

public class Main {
    public static void main(String[] args) {
        Car myCar = new Car("Blue", 150);   // calls constructor
        myCar.displayInfo();
    }
}

When you write new Car("Blue", 150), Java automatically calls the constructor and sets values.

6. ⛔ When NOT to Overcomplicate

  • For very small programs (like printing "Hello World"), you don’t need multiple classes and objects.
  • Don’t add too many fields and methods in one class initially – keep it simple and focused.

7. 🧠 Minimum You MUST Remember

  • Class → blueprint/template.
  • Object → real instance created using new.
  • Constructor → special method used to initialize objects; same name as class; no return type.
  • You access fields and methods using objectName.field and objectName.method().

8. ⚠️ Common Mistakes

  • Forgetting to use new while creating objects.
  • Trying to access fields before creating the object (will cause NullPointerException if reference is null).
  • Giving constructor a return type (then it becomes a normal method, not a constructor).
