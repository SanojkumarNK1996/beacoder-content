Media Queries

Why this exists

  • Users in India access websites from mobile phones, tablets, and desktops with very different screen sizes.
  • Media queries allow layouts to adapt to these devices.

What is this

  • CSS media queries apply different styles based on device characteristics like width, height, or orientation.

Syntax / Basic usage

/* Default styles */
.container {
  width: 800px;
}

/* Small screens */
@media (max-width: 600px) {
  .container {
    width: 100%;
  }
}

How it works

  1. Write base styles for general use.
  2. Add @media rules to override styles when conditions (like max-width) are met.

Minimum you must remember

  • Common breakpoints like 320px, 768px, 1024px are widely used but not mandatory.
  • Mobile-first approach: start with small screens and use min-width queries.

Common mistakes

  • Writing too many specific breakpoints leading to hard-to-maintain code.
  • Designing only for desktop and ignoring mobile.

Interviewer perspective

  • Explain mobile-first vs desktop-first approach.
  • How would you make a layout responsive without a framework?

Practice questions

  1. Build a simple two-column layout that stacks into one column on small screens.
  2. Use media queries to change font sizes based on screen width.
