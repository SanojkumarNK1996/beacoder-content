CSS Syntax

Why this exists

  • CSS syntax defines how we should write style rules so the browser can understand and apply them correctly.

What is this

  • CSS syntax is made up of selectors, properties, and values.
  • Each rule targets elements and assigns specific visual styles.

Syntax / Basic structure

selector {
  property-1: value-1;
  property-2: value-2;
}

h1 {
  color: red;
  text-align: center;
}

How it works

  1. The selector chooses which HTML elements to style.
  2. Each property describes what aspect to style (color, size, spacing).
  3. The value gives the specific setting (red, 16px, etc.).

When to use

  • Any time you want consistent styling for a group of elements.

Minimum you must remember

  • Each property ends with a semicolon.
  • Selectors can be tags, classes, IDs, or more complex forms.

Common mistakes

  • Missing semicolon at the end of a property.
  • Using invalid property names or values.

Interviewer perspective

  • Explain the structure of a CSS rule.
  • What happens if you forget a semicolon?

Practice questions

  1. Write a CSS rule that makes all paragraphs blue and 18px in size.
  2. Write a rule that centers all h2 elements and makes them bold.
