What is CSS?

Why this exists (Indian real-life example)

  • Government portals, IRCTC, and food delivery apps like Swiggy and Zomato all use the same HTML structure but look completely different because of CSS.
  • Without CSS, every page would look like plain black text on a white background.

What is this

  • CSS (Cascading Style Sheets) is the language used to style HTML pages.
  • It controls colors, fonts, spacing, layout, and overall visual design.

Syntax / Basic structure

selector {
  property: value;
}

p {
  color: blue;
  font-size: 16px;
}

How it works (step-by-step)

  1. Browser loads the HTML document.
  2. Browser then loads CSS (inline, internal, or external).
  3. CSS rules are matched with HTML elements using selectors.
  4. Final styles are applied based on cascade and specificity.

When to use

  • Any time you want to control the appearance of a web page.
  • To build consistent design systems across multiple pages.

When not to use

  • Do not use CSS to store data or handle business logic.
  • Do not rely on CSS alone for important information (screen readers still need semantic HTML).

Minimum you must remember

  • CSS = styling language for HTML.
  • Styles are defined as selector + property + value.
  • CSS can be inline, internal (<style>), or external (.css file).

Common mistakes

  • Writing all styles inline instead of using a separate CSS file.
  • Not reusing classes, leading to duplicated code.

Interviewer perspective

  • What is CSS and why is it needed if we already have HTML?
  • Difference between inline, internal, and external CSS?

Practice questions

  1. Create a simple HTML page and add an external CSS file that changes background color and font style.
  2. Try styling the same element using inline, internal, and external CSS and see which one wins.
