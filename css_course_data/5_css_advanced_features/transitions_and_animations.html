Transitions and Animations

Why this exists

  • Subtle animations improve user experience in navbars, buttons, modals, and loaders on modern Indian apps and websites.

What is this

  • Transitions smoothly change property values over time.
  • Animations use keyframes to define complex multi-step changes.

Syntax / Basic usage

.button {
  background-color: blue;
  transition: background-color 0.3s ease;
}

.button:hover {
  background-color: darkblue;
}

@keyframes fade-in {
  from { opacity: 0; }
  to { opacity: 1; }
}

.box {
  animation: fade-in 1s ease-in-out;
}

How it works

  1. transition defines which properties should animate and how long they take.
  2. When the property changes (like hover), the browser interpolates values over time.
  3. @keyframes defines steps in an animation; animation applies it to an element.

Minimum you must remember

  • transition: property duration timing-function.
  • animation: name duration timing-function other-options.

Common mistakes

  • Overusing animations causing distraction or performance issues.
  • Animating expensive properties like width and height instead of transform/opacity.
