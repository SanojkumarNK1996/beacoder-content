CSS Preprocessors

Why this exists

  • Large projects (like e-commerce or banking portals) need organized, reusable, and maintainable CSS.

What is this

  • CSS preprocessors like Sass and LESS extend CSS with variables, nesting, mixins, and functions.
  • You write code in .scss or .less, which is compiled into normal CSS.

Syntax / Basic usage (Sass example)

$primary-color: #3498db;

.button {
  background-color: $primary-color;
  padding: 10px 20px;

  &:hover {
    background-color: darken($primary-color, 10%);
  }
}

How it works

  1. Write styles using preprocessor features (variables, nesting, etc.).
  2. Run a build step (CLI, build tool, or bundler) to compile into plain CSS.
  3. Browser only sees and uses the compiled CSS.

Minimum you must remember

  • Preprocessors improve developer productivity but are optional.
  • Final output is still standard CSS.

Common mistakes

  • Over-nesting selectors leading to very specific and hard-to-override CSS.
  • Using too many nested levels instead of keeping styles flat and modular.

Interviewer perspective

  • Why use Sass instead of plain CSS?
  • What are variables and mixins in Sass?

Practice questions

  1. Install Sass and convert a small CSS file into SCSS using variables.
  2. Create a mixin for a button style and reuse it for multiple buttons.
